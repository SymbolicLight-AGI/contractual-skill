# PRD Framing Response: "AI Insights for All Users"

## Immediate Gap Assessment

The request contains **four critical unknowns** that prevent PRD creation:

| Missing Element | Why It Blocks Progress |
|-----------------|------------------------|
| **User Segment** | Cannot design UX, define access controls, or prioritize feature scope |
| **Workflow** | Cannot map to existing user journeys or identify integration points |
| **Data Source** | Cannot scope technical complexity, data availability, or API dependencies |
| **Success Metric** | Cannot measure launch readiness, validate hypothesis, or declare ROI |

---

## Recommended Next Steps

Rather than produce a hollow PRD, I propose a **3-step framing process** to resolve these gaps before detailed specification.

### Step 1: Discovery Sprint (1 week)

| Activity | Output |
|----------|--------|
| Identify 3 candidate user segments | Confirmed segment with size estimate |
| Map current workflow pain points | User journey map (as-is state) |
| Audit available data sources | Data availability matrix |
| Define initial success hypothesis | Hypothesis statement with 2-3 candidate KPIs |

**Framework:** Opportunity Solution Tree (OST) to map from problem → solution hypothesis

---

### Step 2: Validation Gate (Week 2)

Before full PRD, run a lightweight validation:

- **5 user interviews** against candidate segment
- **Data feasibility check** with Engineering
- **Metric alignment** with stakeholder on 1-2 leading indicators

---

### Step 3: PRD Kickoff (Week 3)

With validated inputs, PRD follows standard structure:

- Problem statement & hypothesis
- User segment definition
- Core workflow (day-in-the-life framing)
- Data source specification
- Success metrics (primary + guardrail)
- Scope / exclusions

---

## What I Need to Proceed

| Information | Format | Source |
