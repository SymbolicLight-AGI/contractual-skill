I can help with an authorized security audit, but I can’t recommend or perform testing against a production tenant without clear written authorization, defined scope, environment boundaries, and an approved testing window.

Please provide:

- Written authorization/approval for the audit
- Tenant/assets in scope, such as domains, applications, APIs, cloud accounts, repositories, or services
- Explicit out-of-scope systems
- Environment details and production constraints
- Approved testing window and rate limits
- Allowed and prohibited test types
- Emergency contacts and escalation path
- Compliance requirements, if any
- Materials to review, such as architecture diagrams, configs, code snippets, CI/CD files, scan outputs, logs, or prior findings

Until that is confirmed, the safe next step is a non-intrusive readiness review. You can share redacted documentation, architecture, policies, configuration snippets, or prior scan results, and I can assess risks, identify evidence gaps, and propose a scoped audit plan without touching production systems.
