## Objection Playbook

### Known objection: "Integration effort may be too high."

| Element | Response |
|---|---|
| Acknowledge | "That's a fair concern — integration cost is one of the top reasons API projects stall." |
| Clarify | Ask: which system are we integrating with? In-house dispatch, TMS, mobile app? Who would own the work? What's their current capacity? |
| Reframe | Frame against the cost of *not* integrating: continued manual routing, fuel/time waste, missed SLAs. |
| Evidence to offer (only if true) | Reference docs, sandbox/API keys, sample code, time-to-first-call benchmarks, and any similar customer integration timelines — **only cite specifics that are factually supported.** |
| De-risk | Propose a scoped technical validation (e.g., a time-boxed proof of integration) with clear success criteria. |
| Human gate | Any offer of paid POC, free trial, professional services, or implementation commitment requires sales manager + delivery owner approval. |

### Anticipated objections to prepare for

- "We can build this in-house." → Compare maintenance, edge cases, ongoing optimization research.
- "Wrong time / no budget this quarter." → Identify a compelling event; offer phased start.
- "Security / data review needed." → Route to security owner; do **not** pre-approve any security claim.
- "Pricing is unclear / too high." → Cannot respond without pricing model — **Human gate** before any concession.

## Closing Strategy (Stage: Post-Demo)

Post-demo with an unresolved technical objection and a single contact is **not** a closing stage — it's a multi-threading + technical validation stage. Treat the next steps as advancing, not closing.

1. **Confirm or build a champion.** Test Jordan: will she introduce the economic buyer and the engineering lead?
2. **Engage the technical evaluator** to convert the integration objection into a scoped plan.
3. **Surface the economic buyer** and the business case (cost/time saved per route, SLA impact).
4. **Establish a compelling event** to anchor a real close date.
5. **Then** move to proposal, procurement, and mutual close plan.
