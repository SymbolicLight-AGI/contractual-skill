## 4. Objection Playbook

### Known Objection: “Integration effort may be too high”

**Why they say this:** They likely lack internal dev capacity, fear disrupting existing operations, or have had a bad integration experience before. There may also be a technical blocker who hasn’t been heard.

**Response framework:**
1. **Acknowledge:** “Totally fair – integration is often the part that makes or breaks a tool’s success. I want to make sure this is as smooth as possible.”
2. **Reframe:** “The heaviest lift is usually custom map data and route constraints. We’ve found that the most time-consuming piece isn’t our API – it’s cleaning up legacy address data, which your existing system already handles. What’s your current stack?”
3. **Evidence:** “One logistics company similar to yours integrated our API in 12 working days using our pre-built SDK for their TMS. Here’s the implementation timeline they followed.” (Prepare a referenceable case study.)
4. **Confirm:** “If we can show you a proof-of-concept integration in under two weeks with a dedicated engineer, would that address your concern? Then we can map out the full timeline together.”

**If they push back:** Propose a no-cost, limited-scope sandbox trial where they call the API with a subset of routes. Offer a technical support engineer for the duration.

### Anticipated Objections

- **Price/Budget:** “We don’t have budget this year.” → Respond by linking to cost savings: “Our pricing is $X/month. If we can show that the routing savings alone cover that 3x in the first quarter, does that change the budget conversation?”  
- **Timing:** “Can we revisit next quarter?” → “What’s the cost of waiting? For example, during the holiday peak, every percentage point of route efficiency will save you X. Delaying means you run the old routes for another cycle.”  
- **Status Quo:** “We already have a routing system / manual process.” → “What’s it costing you in overtime and missed delivery windows? Let’s benchmark together.”  
- **Competition:** (Unknown competitors) → Without intel, generic line: “Different tools optimize routes, but our API is built for real-time re-optimization and dynamic constraints, which matters if you’re doing same-day or B2B deliveries.” Avoid direct competitor attack until you know who they are.

- **Authority:** “I need to check with my boss.” → “Of course. Let’s schedule a brief joint call so I can hear their perspective and address any concerns early. When works for both of you?”
