## 风险清单（按严重度排序）

### 🔴 严重 - 权限风险（Authorization）
- **后端 API 未校验角色**：仅靠前端按钮隐藏做管理员判断，等同于无防护。任何已登录用户（甚至通过抓包伪造请求的非管理员）都可调用导出接口，造成**水平/垂直越权**。
- **直接对象引用风险**：若接口支持按用户 ID/范围导出，未校验调用者权限可导致全量用户数据泄露。

### 🔴 严重 - 隐私与合规风险（Privacy / Compliance）
- **导出字段包含 PII**：姓名、手机号、邮箱属于个人敏感信息；最近订单金额属于交易数据。批量导出明文 PII 可能违反 GDPR / 个人信息保护法 / 等保要求。
- **缺少脱敏策略**：未说明是否对手机号/邮箱做掩码处理，或是否区分"查看"与"导出"权限。
- **缺少用户同意/告知与数据出境评估**（如涉及）。

### 🟠 高 - 审计与可追溯性
- **无审计日志**：无法回答"谁在何时导出了哪些用户的哪些字段"，事后无法溯源、无法满足合规审计要求。
- **缺少导出操作的双人复核 / 审批流**（敏感数据导出通常需要）。

### 🟠 高 - 滥用与性能风险
- **无导出数量限制 / 无频控**：可被用于一次性拖库、构造爬虫、或制造 DoS（大查询拖垮 DB）。
- **无分页 / 流式处理说明**：大数据量导出可能 OOM。
- **导出文件存储位置与生命周期未定义**：临时文件是否清理、是否签名 URL、是否走对象存储 ACL。

### 🟡 中 - 可维护性
- 前后端权限判断不一致是典型的"信任前端"反模式，应统一在后端 + 网关层做权限校验。
- 字段集合硬编码会随业务变化，建议白名单 + 字段级权限配置。

### 🟡 中 - 测试缺口
不确定 PR 是否包含测试，但基于摘要列出**必须补齐**的测试：
1. **权限测试**：非管理员调用接口应返回 403；未登录返回 401。
2. **越权测试**：管理员能否导出超出其租户/范围的数据。
3. **审计日志写入测试**：成功/失败导出均应记录。
4. **限额测试**：超过单次导出阈值应拒绝或异步化。
5. **频控测试**：短时间多次调用应被限流。
6. **大数据量测试**：性能/内存基线。
7. **PII 字段脱敏测试**（若实现脱敏）。
8. **失败/中断测试**：导出中断后文件是否清理、状态是否一致。

## 复核建议
**不建议在以下三项闭环前合并**：① 后端鉴权落地；② 审计日志补齐；③ 导出数量/频率限制。其余隐私脱敏与审批流可作为后续迭代项，但需安全与产品给出明确结论。

> 不确定性：未见实际代码，无法确认是否已存在网关层鉴权、是否复用既有审计组件、字段是否已有脱敏。以上结论以摘要为准，需对照实现复核。
