- 报错类型：`IndexError: list index out of range`

### 可能原因（按可能性，均待验证）
1. 请求体合法但 `items` 字段为空列表（最常见）。
2. 请求体结构不符合预期，`parse_payload` 返回了带空 `items` 的默认结构。

4. `parse_payload` 内部对异常输入做了"返回空列表"的兜底，掩盖了上游问题。
5. **仅在数据库为某些数据源时** 才有可能间接相关——但当前 trace 无证据。

### 代码可维护性/健壮性问题（severity: Medium）
- `service.py:42` 单行链式访问 `[...]["items"][0]`，没有对键缺失、类型、空列表做防御。
- 缺少输入校验与结构化错误响应；当前会向调用方暴露 `IndexError` 或 500。

- **误诊风险**：在缺失信息时跳到"数据库损坏"会导致错误的运维动作（如重启 DB、回滚），加重事故。
