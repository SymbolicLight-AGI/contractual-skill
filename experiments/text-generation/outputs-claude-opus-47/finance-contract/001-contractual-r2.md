# 售后知识库 Agent 试点报价 — 财务与合同风险评估

## 结论摘要

报价草案在金额、付款比例与基本范围上已有初步约定，但**试运行通过标准、数据清洗边界、第三方接口费用归属、变更计费方式**等关键条款均未明确，存在显著的验收争议、成本超支与回款风险。建议在签署前由财务、法务、交付 PM 共同补全条款，**当前不可作为最终合同文本提交**。

## 已知事实

- Known term：项目为"售后知识库 Agent 试点"。
- Known term：首期报价 120,000 元。
- Known term：付款节奏为签署后 50% + 试运行通过后 50%。
- Known term：交付范围包含知识库初始化、1 个客服 Agent、2 次远程培训。

## 缺失信息

- Missing term：数据清洗边界（数据源数量、格式、清洗深度、由谁提供原始数据、脏数据返工责任）。
- Missing term：试运行通过标准（试运行时长、评估指标、回答准确率门槛、由谁判定、争议处理机制）。
- Missing term：第三方系统接口费用归属（工单系统、CRM、知识源 API 的接入与授权费用是否含在 120,000 元内）。
- Missing term：变更需求计费方式（变更触发条件、计价单位、最小计费单元、审批流程）。
- Missing term：客户信用与历史回款情况（影响"先做后付"风险判断）。
- Missing term：成本基线与毛利测算（无法判断 120,000 元是否覆盖人力+第三方费用）。
- Missing term：知识库初始化的数据量上限、Agent 调用量上限、培训人数与时长上限。
- Missing term：试点结束后的转正式、续费、知识产权归属条款。
- Missing term：SLA、保密、终止与退款条款。

## 财务与合同风险

- Risk（验收风险）：试运行通过标准缺失，第二笔 50% 回款触发条件不可量化，可能导致尾款长期挂账或被客户单方面拒收。
- Risk（范围蔓延）：数据清洗边界未定，易演变为"无限清洗"，直接吞噬毛利。
- Risk（成本穿透）：第三方接口费用未界定归属，若由我方承担可能使 120,000 元报价跌破成本线。
- Risk（变更失控）：无变更计费机制，客户可在试点期间持续追加需求而不增费。
- Risk（培训范围模糊）：仅写"2 次远程培训"，未限定时长、人数、内容深度、是否含答疑期。
- Risk（Agent 用量未封顶）："1 个客服 Agent"未约定并发量、调用量、知识条目上限，可能引发后期资源争议。
- Risk（现金流风险）：50/50 付款结构在缺乏验收标准时，实际回款周期不确定。
- Risk（信用风险）：未做客户资信检查即承诺签署后启动交付。
- Risk（知识产权与数据合规）：未约定客户提供数据的合法性承诺、清洗后语料归属、模型微调产物归属。
- Risk（试点延伸）：未写明试点与正式合同的衔接，可能被默认延续为长期免费服务。

## 不得直接承诺事项

以下事项**本评估不予批准**，需走对应审批流程：

- 不得承诺 50/50 付款结构最终生效（待财务+信用评估）。
- 不得承诺"试运行通过"以客户主观判断为准。
- 不得承诺包含第三方接口费用。
- 不得承诺无限次数据清洗或无限轮培训答疑。
- 不得承诺变更需求免费。
- 不得承诺 SLA、可用性、回答准确率等量化指标。
- 不得承诺试点结果不达标时的全额退款或免费返工。
- 不得签署任何具有法律约束力的最终条款。

## 待确认问题

提交给客户/内部前需澄清：

1. 数据清洗：数据源有几个？总量多少？由谁提供？清洗到什么程度算完成？
2. 试运行通过标准：试运行多久？以什么指标判定（如准确率 ≥X%、覆盖问题数 ≥N）？谁有最终判定权？
3. 第三方接口：涉及哪些外部系统？接入与授权费由谁承担？是否需要客户协助开通？
4. 变更计费：以人天 / 功能点 / 固定包计价？最小计费单位？变更审批走谁？
5. 培训：每次培训时长、人数上限、是否提供材料、是否含课后答疑期？
6. Agent 范围：并发数、月调用量、知识条目上限是多少？超量如何计费？
7. 验收与尾款触发：是否引入"自动验收"条款（如试运行结束 N 个工作日内未书面反对视为通过）？
8. 知识产权：客户数据、清洗后语料、Prompt 与 Agent 配置的归属？
9. 客户信用与开票：客户主体、付款账期、发票类型？
10. 试点退出机制：未通过验收时双方权利义务？

## 交接建议

- **Finance**：复核 50/50 付款节奏、客户信用与回款历史、毛利测算、开票与税务安排。
- **Legal**：补全验收、SLA、知识产权、保密、终止、违约与退款条款；审查"试运行通过"的法律可执行性。
- **Delivery PM**：量化数据清洗边界、试运行通过指标、培训范围、Agent 用量上限、第三方接口工作量与外购成本。
- **Risk Review**：评估范围蔓延、变更失控、试点延伸为长期免费服务的风险，决定是否需要总价封顶或工时上限条款。
