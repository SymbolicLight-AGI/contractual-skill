# 合同评估结论

## 是否可直接走合同
**不可以直接走合同。** 当前材料存在多项重大风险，必须经多角色复核后方可推进。

---

## 主要风险识别

### 1. 范围风险
- "5 个业务 Agent" 未定义功能边界、复杂度、模型选型、训练数据来源。
- "对接 OA、CRM、售后工单系统" 未列出具体系统厂商、接口规范、是否已有 API、是否需要定制开发。
- "试点后推广到 8 家分公司" 推广阶段范围模糊，各分公司是否同构、是否需二次实施未明。

### 2. 合同结构风险
- 客户要求"一份合同覆盖全部阶段"，将试点与推广合并，导致前期不确定性被打包计价，承接方风险敞口过大。
- 缺少阶段性里程碑和阶段退出机制。

### 3. 报价风险
- "报价包含所有系统接口"为开放式承诺，属典型的**无限服务条款**，无法估算成本。
- 未区分一次性实施费、订阅费、运维费、接口扩展费。

### 4. 验收风险
- "验收标准后续再定"——**缺失验收标准**意味着付款触发条件不明，存在严重的应收账款回收风险。

### 5. 付款风险
- 材料中未见付款里程碑、预付款比例、质保金安排。

---

## 必须复核的角色与事项

### 售前 / 解决方案
- 拆解 5 个 Agent 的功能清单、复杂度分级。
- 与客户确认 OA / CRM / 售后工单的系统版本、接口现状、对接工作量。
- 明确试点范围与推广范围的差异化方案。

### 交付
- 评估试点与 8 家分公司推广的实施工时与资源占用。
- 给出分阶段交付计划与里程碑。
- 提出可量化的验收标准草案（功能、性能、可用性、接口联调）。

### 财务
- 复核报价结构是否覆盖所有成本项；接口对接是否单独计费。
- 复核付款节奏、现金流与回款风险。
- 评估"一份合同覆盖全部阶段"对收入确认的影响。

### 法务
- 审查"报价包含所有系统接口"等开放式承诺的措辞，建议改为枚举式或变更控制条款。
- 设计验收条款、变更管理条款、阶段退出 / 中止条款。
- 数据安全、知识库数据权属、模型输出责任条款。

### 风险审查 / 合规
- 集团级知识库涉及的数据合规、权限隔离、跨分公司数据流转。
- 推广阶段的合规风险（如分公司所在地法规差异）。

---

## 需向客户确认的问题
1. 是否接受将合同拆分为「试点合同 + 推广框架协议 + 分公司订单」结构？
2. 5 个 Agent 的具体业务场景与功能清单？
3. OA / CRM / 售后工单的厂商、版本、接口文档是否可提供？
4. 是否同意在签约前共同制定验收标准？
5. 接口范围是否限定在已列明的三类系统，新增接口是否走变更流程？

---

## 建议下一步
1. **暂缓出合同**，先完成售前 + 交付的范围拆解。
2. 输出《范围与假设清单》《验收标准草案》《分阶段报价》给客户确认。
3. 法务、财务、风险审查三方会签后，再决定合同结构。
4. 任何折扣、付款条款、范围承诺**本评估不予批准**，须由相应授权角色决定。

> 说明：本判断基于所提供合成材料；客户内部决策流程、历史合作记录、集团采购政策等信息缺失，相关结论存在不确定性。
