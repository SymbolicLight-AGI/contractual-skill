# 报价草案风险审查

## 一、合同风险

1. **验收标准缺失**：未定义"试运行通过"的具体标准（如准确率阈值、响应时间、并发量、试运行周期），第二期 50% 款项的支付条件存在解释争议，可能引发尾款拖欠或无限期试运行。
2. **交付范围边界模糊**：
   - "知识库初始化"未约定文档数量、格式、语种、更新频率上限。
   - "1 个客服 Agent"未说明意图数量、对话轮次、渠道（Web/微信/电话等）。
   - "2 次远程培训"未约定时长、参训人数、培训对象层级。
3. **数据清洗边界未定**：原始数据质量风险由谁承担不明；若客户提供脏数据，工作量可能远超预估。
4. **变更管理条款缺失**：无变更控制流程（CR 流程）、无变更计费单价，存在范围蔓延（scope creep）风险。
5. **第三方接口责任不清**：与客户既有 CRM/工单/IM 系统的对接费用、License、API 调用费归属未明，可能产生争议成本。
6. **其他常见缺失条款**（草案中未见，建议确认是否在主合同覆盖）：知识产权归属、数据保密与合规、SLA 与售后期限、违约责任、终止条款、不可抗力。

## 二、财务风险

1. **回款风险集中在尾款**：50% 尾款绑定模糊的"试运行通过"，回款周期与坏账风险偏高。
2. **成本不可控**：数据清洗、第三方接口、变更需求三项均可能成为隐形成本黑洞，挤压 120,000 元固定总价的毛利。
3. **税费与开票约定缺失**：未注明含税/不含税、税率、发票类型（专票/普票）及开票时点。
4. **币种与价格有效期**：未注明报价有效期，若签约延迟可能面临成本上涨。
5. **超期/延迟责任**：未约定客户配合不到位导致工期延长时的费用补偿机制。

## 三、待确认条款清单

| # | 条款 | 待确认要点 |
|---|------|-----------|
| 1 | 试运行通过标准 | 量化指标、测试用例集、试运行周期上限、未通过时的处理方式 |
| 2 | 数据清洗边界 | 客户提供数据的格式/质量要求、超出范围的单价或工时报价 |
| 3 | 第三方系统接口 | 接口清单、License/调用费承担方、对接工作量上限 |
| 4 | 变更需求计费 | CR 流程、人天单价或功能点单价、最小计费单位 |
| 5 | 交付物清单 | 知识库条目数、Agent 能力指标、培训形式与时长 |
| 6 | 付款条件 | 发票类型、税率、含税与否、开票与回款时点 |
| 7 | 验收与质保 | 验收期限、默认验收规则、质保期长度与范围 |
| 8 | 知识产权与数据 | 模型/Prompt/知识库的归属、数据保密、出境合规 |
| 9 | 违约与终止 | 逾期付款利息、客户终止补偿、不可抗力定义 |
| 10 | 报价有效期 | 报价锁定期限、超期重新议价机制 |

> 说明：以上为风险提示与待澄清事项，最终条款表述与商务取舍需由贵司商务、法务与客户协商确定，本审查不替代任何一方做最终决定。
