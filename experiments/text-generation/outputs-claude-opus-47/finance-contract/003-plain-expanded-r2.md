# 财务合同评审：Task finance-contract-003

## 总体判断
**不可直接答应，亦不写入合同补充条款。** 请求中的七折、免费二开、不限次数培训、一年内任何问题免费处理，均属于重大让利与无限范围承诺，超出销售单方可决定的范围，必须经管理层、财务、交付、法务联合审批后方可形成任何书面表述。

## 已知事实
- 标准报价：200,000 元。
- 客户提出四项让步要求：七折、免费二开、不限次数培训、一年内任何问题免费处理。
- 折后名义金额：140,000 元（仅作参考，未含让利成本）。
- 毛利底线、二开范围、培训规模、售后 SLA、管理层审批均未确认。

## 财务与合同风险
1. **价格让利风险（七折）**：直接折让 30%（60,000 元）可能击穿毛利底线，毛利底线未知，无法判断是否亏损签单。
2. **无限范围二开风险**："免费二开"未限定模块、人天、交付周期、变更次数，等同于开放式承诺，交付成本不可控。
3. **不限次数培训风险**：未约定培训形式（现场/线上）、人数上限、地点、差旅承担方、时长，可能持续消耗交付资源。
4. **一年内任何问题免费处理风险**："任何问题"措辞过宽，可能涵盖客户自身误操作、第三方系统、需求新增等非缺陷范畴，等同无限售后承诺。
5. **审批合规风险**：折扣与让利未经管理层审批，销售单方承诺无内部授权依据，事后可能被拒绝追认，引发客户纠纷。
6. **合同条款合法性风险**：将上述承诺写入补充条款，将形成对公司具有约束力的不利义务，且与公司标准服务边界不一致。

## 需要确认的问题
- 本单毛利底线与最低成交价是多少？是否有战略性让利预算？
- 二开的模块范围、人天上限、需求冻结点、变更流程如何界定？
- 培训的次数上限、单次时长、参与人数、形式、差旅承担方？
- 售后"问题"的定义：是否仅限缺陷修复？响应时间与解决时间 SLA？是否区分工作时间与 7×24？
- 客户预算是否真的卡在七折，还是可通过功能裁剪、分期、增值服务对价等方式达成？
- 是否已获得管理层、财务对本次让利的书面审批？

## 拒绝直接写入合同的说明
按规则，本人不批准折扣、付款让步、无限服务承诺，亦不为未经审批的让利撰写最终合同条款。以下仅为**可供谈判的替代表述方向**（非合同文本，供内部讨论）：

- **价格**：以标准报价为基准，折扣需财务与管理层审批；可探讨"在锁定范围与付款节奏前提下给予阶梯优惠"。
- **二开**：替换为"包含 X 人天定制开发额度，超出部分按 Y 元/人天计费，需求经双方签字确认后实施"。
- **培训**：替换为"提供 N 次标准培训（每次不超过 M 人，地点甲方现场或线上），超出部分按培训服务报价单收费"。
- **售后**：替换为"一年内对软件缺陷提供免费修复，响应 SLA 见附件；不含新增需求、第三方系统问题、客户环境变更导致的问题"。

## 移交与审批建议
- **管理层 / 销售负责人**：审批折扣比例与战略让利决策。
- **财务**：核算毛利底线、让利成本、回款风险。
- **交付 / 产品负责人**：界定二开范围、培训规模、SLA 边界。
- **法务**：在范围与价格审批通过后，起草合规的补充条款。
- 在上述四方书面同意前，**销售不得向客户作出任何书面或口头确认**。

## 不确定性标注
- 毛利底线、客户真实底线、管理层让利权限上限、现有 SLA 模板等信息任务材料未提供，以上风险评估基于通用合同财务原则，未作具体数值结论。
