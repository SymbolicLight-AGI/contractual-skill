# 财务与合同风险

- 风险：首付款延后至上线后，等同于"先做后付"，在缺少信用与审批的情况下显著放大坏账与现金流风险。
- 风险：客户存在历史回款慢记录，叠加无首付款，回收周期与坏账概率进一步上升。
- 风险："上线"未定义验收标准，可能导致付款触发点争议、收入确认延迟。
- 风险：发票信息缺失会延后开票时点，影响当季收入与税务合规节奏。
- 风险：销售出于签单压力可能默许非标条款，绕开折扣与付款审批流程。
- 风险：项目群直接转发原始沟通记录，可能泄露客户个人联系方式与内部敏感判断（如"销售担心""回款慢"）。

# 不得直接承诺事项

- 不得承诺：首付款延后至上线后或任何非标付款节奏。
- 不得承诺：在验收标准未定义前以"上线"作为付款触发点。
- 不得承诺：发票相关时点、抬头、税务处理的最终安排。
- 不得承诺：任何隐性折扣、免费定制或额外服务以换取签单。
- 不得承诺：跨阶段、跨期的付款合并或顺延。

# 交接建议

- 财务：复核付款节奏、开票时点、应收账款与现金流影响，确认是否需要授信复核。
