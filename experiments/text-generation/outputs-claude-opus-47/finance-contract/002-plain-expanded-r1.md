# 财务合同审阅：先做后付请求

## 摘要判断
**不支持直接回复客户"可以先做后付"。** 在客户信用、项目范围、合同与成本均未明确的情况下，承诺先做后付将使我方承担全部前置成本与回款风险，属于不可由销售或单点角色直接批准的付款让步。

## 已知事实
- 客户为某中型制造企业，信用状况未知。
- 项目范围尚未形成正式方案。
- 合同尚未起草。
- 我方需投入知识库实施人员，可能涉及现场调研与第三方工具采购成本。
- 客户提出"先做后付"的付款方式。

## 财务与合同风险
1. **回款风险**：客户信用未经评估，先投入后回款可能形成坏账或长期应收。
2. **成本敞口**：实施人员、差旅、第三方工具均为刚性支出，若客户中途变更或拒付，损失无法收回。
3. **范围风险**：项目方案未定，"做"的边界不清，易出现无限交付、反复返工、验收无标准。
4. **合同空白风险**：无合同即开工，缺乏付款节点、违约条款、知识产权与责任界定，争议时无法主张权利。
5. **第三方成本传导风险**：第三方工具费用若先行垫付，客户后续不认账将由我方承担。
6. **越权承诺风险**：付款条款属于需财务与法务审批的事项，销售/项目侧不得直接答复客户。

## 可替代付款方案（供内部讨论，非对客承诺）
- **标准分期**：例如签约预付 30%–50%，里程碑验收付 30%–40%，终验付尾款（具体比例需财务确认）。
- **小额试点 + 正式合同**：先以付费 POC / 诊断阶段验证范围与价值，再签订正式实施合同。
- **担保或预付保证**：在客户确有先做后付诉求时，要求银行保函、母公司担保或更高比例尾款。
- **里程碑挂钩验收**：每阶段交付物对应付款，避免一次性回款集中在末端。

> 以上方案仅为可讨论选项，最终条款须由财务与法务审批后方可对客。

## 必须向客户/内部确认的问题
- 客户工商、信用、过往付款记录及付款周期？
- 客户提出先做后付的真实原因（预算流程？审批节奏？信任问题？）？
- 项目范围、交付物清单、验收标准是什么？
- 实施人天、现场调研次数、第三方工具清单与费用估算？
- 客户可接受的付款节点与比例区间？
- 是否可提供担保、保函或预付款？
- 我方对该客户的授信额度是否已评估？

## 移交与审批建议
- **财务**：评估客户信用、授信额度、回款风险与最低可接受付款结构。
- **法务**：审阅付款条款、违约责任、第三方成本承担与合同范本适配。
- **交付/实施负责人**：核算人天、现场与第三方成本，明确范围与验收口径。
- **销售负责人 / 商务审批人**：付款让步属于需上级审批事项，不得由对接人单独承诺。
- **风险/管理层**：若客户坚持完全先做后付，需提交风险评审。

## 对客回复建议（话术方向，非最终稿）
在内部信息补齐前，建议先以"需要根据项目范围与合作模式共同确定付款安排"为由暂缓承诺，引导客户先进入范围沟通与方案确认阶段，而非直接答复"可以先做后付"。

---
**不确定性标注**：客户信用、项目范围、成本估算与公司既有付款政策均未在材料中提供，上述比例与方案仅为通用参考，具体数字须由财务与法务确认。
