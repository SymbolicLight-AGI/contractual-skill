# 报价草案财务与合同风险检查

## 一、识别到的风险

### 1. 范围风险（Scope Risk）
- **数据清洗边界未定义**：客户原始数据质量未知，若清洗工作量超预期，将直接挤压交付成本与周期。
- **"知识库初始化"定义模糊**：未说明初始化的数据条目数量上限、文档格式范围、语料来源。
- **"1 个客服 Agent"功能边界未写**：未明确意图数量、对话轮次、支持渠道、并发能力等。
- **"2 次远程培训"颗粒度不足**：未说明每次时长、参与人数、培训对象（管理员/坐席）、是否提供材料。
- **第三方系统接口未界定**：工单系统、CRM、IM 等接口费用归属不清，存在隐性成本。

### 2. 财务风险（Financial Risk）
- **试运行通过标准缺失**：50% 尾款挂钩"试运行通过"，但通过标准未定义，可能导致尾款被无限期拖延（应收账款风险）。
- **变更需求计费方式空白**：客户后续提变更时无计价依据，易陷入"免费加需求"或争议。
- **未约定税费、发票类型、开票节点**（材料未提及，需确认）。
- **未约定逾期付款责任**（违约金/利息条款缺失）。
- **未约定项目终止/中止时的费用结算方式**。

### 3. 付款条款风险（Payment Risk）
- **50/50 分期，无中期里程碑**：项目周期内现金流压力集中在尾款，若试运行拖期，回款风险显著。
- **缺乏验收时限条款**：未规定客户在试运行后多少工作日内必须确认通过或提出书面异议，存在"沉默不验收"风险。

### 4. 合同与审批风险（Contract/Approval Risk）
- **无验收/异议处理流程**。
- **无知识产权归属条款**（知识库内容、Agent 配置、定制开发成果归属未明）。
- **无保密条款、数据安全与个人信息合规条款**（售后数据可能涉及客户个人信息）。
- **无服务范围外（OOS）请求处理机制**。
- **无 SLA / 售后维保期约定**（试运行后是否含运维、运维时长与费用）。

## 三、建议的下一步

1. **暂不签署当前草案**：关键条款（试运行标准、变更计费、接口费用归属）未明确前不建议出具最终报价。
2. **由产品/交付团队**补齐量化的范围说明与试运行验收标准（SOW 或附件形式）。
3. **由财务团队**评估变更计费模型、付款节奏（建议引入中期里程碑，如 30/40/30）与开票安排。
4. **由法务团队**审阅验收、知识产权、保密、数据合规、违约条款。
5. 汇总澄清后，再行内部审批并向客户提交修订版报价。

---

> ⚠️ 说明：本检查仅识别风险与待确认项，**不代表已批准任何折扣、付款条款、范围或合同语言**。最终条款须经财务、法务及交付负责人审批后方可对外确认。
>
> 不确定性标注：材料未提供税费、发票、违约、知识产权、保密、维保等条款现状，以上相关风险基于"草案未写明"的假设给出，需进一步核对完整合同模板。
