# 任务结果：finance-contract-005

## 判断结论

**不可直接走合同。** 当前材料存在多项重大风险，必须先经多角色复核后再决定签约方式（建议拆分为框架合同 + 分阶段子合同/SOW，而非一份大合同覆盖全部）。

---

## 主要风险识别

| 类别 | 风险点 |
|---|---|
| 范围风险 | "5 个业务 Agent"功能边界未定义；OA/CRM/售后工单接口数量、版本、改造责任方未明；"推广到 8 家分公司"无范围上限和分公司差异化需求说明 |
| 合同结构风险 | 一份合同覆盖总部试点 + 8 家分公司推广，阶段未拆分，易形成无限服务和无限责任 |
| 报价风险 | 报价"包含所有系统接口"——接口清单未冻结，属于开口承诺 |
| 验收风险 | 验收标准"后续再定"——无可量化交付物，付款里程碑无触发条件，应收账款风险高 |
| 付款风险 | 未见付款节点、预付款、阶段款比例、滞纳条款 |
| 审批风险 | 集团级 + 多分公司推广涉及主体多，签约主体与结算主体未明确 |

---

## 必须复核的角色与事项

### 售前
- 冻结"5 个业务 Agent"功能清单与边界
- 出具 OA、CRM、售后工单的接口清单、改造范围及客户侧配合事项
- 明确总部试点与分公司推广的差异化方案

### 交付
- 评估试点和 8 家分公司推广的工时、资源、周期可行性
- 定义每阶段交付物与可量化的验收标准
- 评估"一份合同覆盖全部阶段"的交付控制风险

### 财务
- 复核报价是否覆盖全部接口与推广阶段成本，识别开口承诺
- 设计付款里程碑（预付款 / 阶段款 / 验收款 / 质保金）
- 评估 8 家分公司结算主体与回款风险

### 法务
- 审核合同结构：建议主框架合同 + 分阶段 SOW
- 拟定范围变更、接口新增的变更与计费条款
- 明确验收标准缺失情况下的兜底条款与争议解决机制
- 明确签约主体（集团 vs 分公司）和授权链

### 风险审查
- 复核"验收标准后续再定"带来的应收账款与无限服务风险
- 评估推广阶段范围蔓延的整体敞口
- 评估是否需升级到更高审批层级

---

## 建议下一步

1. 暂不签署单一合同；先向客户回函说明拆分签约的必要性。
2. 召集售前 + 交付 + 财务 + 法务联合评审会，冻结范围、报价口径、验收原则。
3. 在范围与验收标准未冻结前，本人不批准任何折扣、付款条款或合同语言。
4. 法务与财务给出建议合同结构后，再提交风险审查与最终审批。

## 不确定性标注

- 客户预算、目标签约金额、期望签约时间未提供。
- 报价金额、毛利结构未提供，无法判断折扣空间。
- 客户内部审批与采购流程未提供，签约主体不明。
