# 任务结果：finance-contract-005

## 总体判断

**不可直接走合同。** 当前信息存在多项重大风险，必须经过多角色复核后才能进入合同流程。

## 已知事实

- 项目类型：集团级知识库 + 多 Agent 工作流。
- 范围：总部知识库、5 个业务 Agent、对接 OA/CRM/售后工单、试点后推广到 8 家分公司。
- 客户要求：
  - 单一合同覆盖全部阶段（试点 + 8 家推广）。
  - 报价包含所有系统接口。
  - 验收标准后续再定。

## 财务与合同风险

1. **范围模糊（高风险）**：一份合同覆盖"试点 + 全集团 8 家推广"，但 8 家分公司的现状、数据规模、定制需求未知，无法在前期锁定工作量。
2. **接口风险（高风险）**：OA/CRM/售后工单三套系统，报价"包含所有接口"，但接口数量、文档、改造范围、第三方配合度未确认，存在无限服务敞口。
3. **验收标准缺失（高风险）**：验收"后续再定" 等于无验收基线，回款节点和结案均无依据。
4. **付款里程碑缺失**：未见付款节奏，单合同一次性覆盖长周期，存在应收款迟滞与坏账风险。
5. **多 Agent 交付边界不清**：5 个业务 Agent 的功能颗粒、数据源、评估指标未定义，易出现免费迭代/无限调优。
6. **推广阶段责任划分不清**：8 家分公司的实施主体、培训范围、运维归属未约定。

## 需澄清的问题

- 试点与推广是否拆分为分阶段合同或主合同 + SOW？
- 5 个 Agent 的功能边界、数据来源、性能/准确率指标？
- OA、CRM、售后工单的接口清单、提供方、是否需第三方付费配合？
- 验收标准能否在合同签署前以附件方式锁定？
- 付款里程碑、发票节奏、推广阶段定价机制？
- 培训范围、运维 SLA、保修期是否包含在报价内？

## 必须复核的角色与事项

| 角色 | 复核事项 |
|---|---|
| **售前** | 5 个 Agent 范围拆解、接口清单核对、推广阶段工作量重新评估 |
| **交付** | 试点与推广路径、8 家分公司实施模式、培训/运维边界、第三方接口可行性 |
| **财务** | 付款里程碑设计、单合同 vs 分阶段合同的回款风险、发票节奏、应收账款敞口 |
| **法务** | 单一合同覆盖长周期的合规性、验收条款、变更控制条款、违约与终止条款 |
| **风险审查** | 验收标准缺失风险、接口无限服务风险、推广阶段范围漂移风险、整体项目敞口 |

## 处置建议

- **不予直接出合同。** 建议改为"主协议 + 试点 SOW"结构，推广阶段以补充 SOW 方式签订。
- 在签约前必须补齐：Agent 功能清单、接口清单、验收标准附件、付款里程碑。
- 由售前牵头收敛范围，交付确认可行性，财务/法务/风险三方联审后再决定合同形式。

## 不确定性标注

- 客户行业、合同金额、交付周期、是否已有框架协议等信息未提供，以上判断基于现有材料；若存在框架协议或已有验收模板，部分风险可下调。
