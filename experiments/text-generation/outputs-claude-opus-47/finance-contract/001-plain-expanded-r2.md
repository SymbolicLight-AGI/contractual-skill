# 报价草案财务与合同风险审查

## 总体判断
当前报价草案具备基本结构，但存在多处范围与验收边界不清、潜在成本敞口未覆盖的问题，**不建议在现状下对外发出或签署**，需经财务、交付、法务联合澄清后再定稿。

## 已知事实
- 项目：售后知识库 Agent 试点
- 报价金额：首期 120,000 元
- 付款节点：签约后 50%，试运行通过后 50%
- 交付范围：知识库初始化、1 个客服 Agent、2 次远程培训
- 明确未写清事项：数据清洗边界、试运行通过标准、第三方接口费用归属、变更计费方式

## 财务与合同风险

1. **验收标准缺失（高风险）**
   "试运行通过"未定义指标（准确率？覆盖率？时长？由谁判定？），可能导致尾款 60,000 元长期挂账或被客户单方面拖延确认。

2. **数据清洗范围敞口（高风险）**
   未约定数据源数量、格式、体量、清洗轮次。客户可能要求无限次清洗或纳入脏数据治理，造成人力成本失控。

3. **第三方系统接口费用归属不清（高风险）**
   若试点涉及工单系统、CRM 等对接，接口开发费、第三方 License、调用量费用是否含在 120,000 元内未约定，存在被默认包含的合同解释风险。

4. **变更需求无计费机制（中高风险）**
   未约定变更评估流程与计费标准（按人天？按模块？），易演变为"免费定制"。

5. **培训范围模糊（中风险）**
   "2 次远程培训"未约定单次时长、人数上限、培训材料是否包含、是否需考核或补训。

6. **付款节奏风险（中风险）**
   50/50 结构在验收标准未定义的前提下，尾款回收风险显著；缺少试运行最长周期、超期视同通过条款。

7. **试点边界与后续商务衔接缺失（中风险）**
   未声明"试点"性质、是否含后续推广义务、知识产权归属、数据保密与退出条款。

## 待确认条款（需澄清后再定稿）

- 试运行通过标准：量化指标、判定方、判定时限、超期默认规则
- 数据清洗：数据源清单、数据量上限、清洗轮次、超出部分计费方式
- 第三方接口：是否含在报价内；若含，列明对接系统清单与上限；若不含，明确由谁承担
- 变更管理：变更评估流程、报价机制、最小计费单位
- 培训细则：单次时长、参训人数、地点、材料、补训规则
- 知识产权与数据安全：知识库内容、Agent 配置、客户数据归属与保密义务
- 试点期限与续约：试点周期、试点结束后的过渡安排
- 发票与付款：发票类型、开票时点、付款周期（如 T+N 天）、逾期利息

## 处理建议与交接

- **不直接批准**当前报价对外发送或作为最终合同基础。
- **交付/产品**：澄清范围边界、试运行验收指标、培训与变更口径。
- **财务**：核定第三方费用归属、发票与付款周期、尾款回收风险敞口。
- **法务**：补充验收、变更、知识产权、保密、试点退出等条款。
- **销售负责人**：与客户沟通澄清事项，避免在未澄清前作出口头承诺。

## 不确定性说明
- 报价 120,000 元的成本构成与毛利结构未提供，无法判断折扣或让步空间。
- 客户行业、数据规模、第三方系统清单未提供，相关风险等级可能随实际情况上调。
