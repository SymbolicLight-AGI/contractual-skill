# 关于"先做后付"请求的回复建议

## 一、风险判断（不建议直接答应"先做后付"）

当前信息不足，直接承诺"先做后付"存在以下风险：

1. **客户信用风险未知**：尚未对该制造企业进行信用背调，无法判断其付款能力与付款意愿，存在坏账风险。
2. **项目范围未定义**：尚未形成正式方案，"做完"的标准无法界定，后期容易产生"未达预期—拒绝付款"的争议。
3. **合同尚未起草**：缺乏书面约束，权利义务、验收标准、违约责任均无依据。
4. **我方成本前置且不可逆**：
   - 知识库实施人员人力成本需先期投入；
   - 现场调研产生差旅费用；
   - 第三方工具可能涉及采购或订阅费用，部分不可退。
5. **现金流压力**：所有成本由我方先行垫付，回款周期被动拉长。

> 结论：在信用、范围、成本三项关键信息缺失的情况下，不应直接支持"先做后付"。

---

## 二、可替代付款方案（建议向客户提出）

可根据客户接受度，从以下方案中协商选择：

| 方案 | 结构 | 适用场景 |
|---|---|---|
| **A. 分阶段里程碑付款** | 签约预付30% + 方案确认30% + 验收30% + 质保10% | 推荐主方案 |
| **B. 按月/按周期结算** | 实施期间按月结算工时与已发生成本 | 项目周期较长 |
| **C. 成本覆盖+尾款** | 预付款覆盖第三方工具+差旅等硬性成本，其余尾款验收后支付 | 客户对预付敏感时 |
| **D. POC先行** | 先做小范围付费POC，验证后再签整体合同 | 范围尚不清晰时优先建议 |
| **E. 信用担保** | 客户提供银行保函/母公司担保后，可放宽账期 | 客户坚持后付时 |

---

## 三、必须向客户/内部确认的问题

**对客户需确认：**
1. 项目范围与交付物的初步边界是什么？是否可先签署SOW或意向书？
2. 期望的项目周期与上线时间？
3. 付款主体是哪家法人？是否涉及集团统一付款流程？
4. 客户内部的付款审批与账期惯例（30天/60天/90天）？
5. 是否接受预付款、里程碑付款或POC先行模式？
6. 是否可提供财务报表、征信报告或担保措施？

**对内部需确认：**
1. 该客户的工商、涉诉、信用评级初步背调结果。
2. 项目预估成本区间（人力工时 + 差旅 + 第三方工具）。
3. 公司对该类客户的最长可接受账期与坏账风险敞口。
4. 第三方工具成本是否可由客户直接采购，避免我方垫付。
5. 法务对合同关键条款（验收标准、付款节点、违约条款）的要求。

---

## 四、建议的回复口径（给客户）

> "感谢您的信任。关于付款方式，我们理解贵司的考虑，也希望尽快推进合作。鉴于本项目涉及人员投入、现场调研以及第三方工具成本，为保障双方权责清晰，建议我们先共同明确项目范围并签署合同，付款方式可以采用**分阶段里程碑结算**或**POC先行**的方式，既能降低贵司的前期投入压力，也便于我们合理安排资源。具体节点可以基于项目方案进一步协商。"

---

**不确定性标注**：以上方案比例（如30/30/30/10）为行业常见参考，最终比例需结合公司财务政策与客户谈判结果确定；客户信用状况未提供，所有方案的最终选择应在背调完成后再敲定。
