# 关于"先做后付"请求的回复建议

## 一、风险判断（不建议直接答应）

在当前信息缺口下，直接同意"先做后付"存在以下风险：

1. **信用风险未评估**：客户为中型制造企业，但其付款记录、财务状况、过往合作信誉均未知，无法判断回款能力与意愿。
2. **范围未锁定**：正式方案尚未形成，工作量边界模糊。"先做"在缺少范围定义时，极易出现需求蔓延，做完后客户以"不符合预期"为由拒付或压价。
3. **成本敞口较大**：
   - 知识库实施人员属于人力刚性投入；
   - 现场调研涉及差旅与时间成本；
   - 第三方工具可能产生采购或订阅费用（且部分为预付/不可退）。
   一旦客户违约，上述成本难以追回。
4. **合同缺位**：合同尚未起草，没有违约责任、验收标准与付款触发条件的法律抓手，争议解决成本高。
5. **不确定性提示**：客户具体规模、行业、采购流程未明，是否属于"先验收后付款"的常规甲方文化也无法判断。

## 二、可替代付款方案（建议任选其一与客户协商）

| 方案 | 结构 | 适用场景 |
|---|---|---|
| **A. 分阶段里程碑付款** | 例如 30%（合同签订/启动）— 40%（方案交付或中期验收）— 30%（最终验收） | 标准首选，平衡双方风险 |
| **B. 启动金 + 尾款** | 收取覆盖第三方工具及调研成本的启动金（如 20–30%），余款验收后支付 | 客户接受度较高，至少覆盖硬成本 |
| **C. 小范围试点先付** | 拆出一个低成本 POC/诊断包，先做先付；通过后再进入主项目 | 客户希望"先看效果"时的折中 |
| **D. 信用背书条件下的后付** | 客户提供银行保函、母公司担保或预付发票贴现等增信措施 | 客户坚持后付且具备增信能力 |

## 三、回复客户前必须先确认的问题

**关于范围与方案：**
1. 项目目标、交付物清单与验收标准是什么？
2. 是否包含现场调研？频次与地点？
3. 是否需要第三方工具？由哪方采购与承担费用？

**关于客户与信用：**
4. 客户法人主体、注册资本、过往类似项目采购记录？
5. 客户内部的付款审批流程与账期惯例（30/60/90 天）？
6. 是否能提供征信报告或商业资信证明？

**关于合同与商务：**
7. 总预算区间与付款节奏的可接受范围？
8. 验收人/决策人是谁？验收标准如何书面化？
9. 违约责任、知识产权归属、终止条款的预期？

---

**建议话术方向**（供参考，不作为最终答复）：
> "先做后付我们原则上可以探讨，但为了双方都能把项目稳妥推进，需要先把范围、验收标准和付款节点一起确认清楚。我们更建议采用分阶段里程碑付款的方式，既能让您在每一步验证我们的交付质量，也便于我们合理安排资源投入。具体方案我整理后与您当面对齐。"

> ⚠️ 不确定性提示：以上建议基于现有合成材料，客户实际信用等级、行业惯例与议价空间未知，最终方案需在补全信息后再行确定。
