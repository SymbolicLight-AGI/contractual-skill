# 报价草案财务与合同风险审查

## 摘要判断
当前报价草案在金额与付款节奏层面结构基本合理，但**范围边界、验收标准、接口费用与变更计费均未写清**，存在较高的财务暴露与履约争议风险，**不建议以现状对外签署**，需在补齐关键条款并经相关角色复核后再推进。

## 已知事实
- 项目：售后知识库 Agent 试点。
- 首期报价：120,000 元。
- 付款节奏：签署后 50%，试运行通过后 50%。
- 交付范围：知识库初始化、1 个客服 Agent、2 次远程培训。
- 明确未写清事项：数据清洗边界、试运行通过标准、第三方系统接口费用归属、变更需求计费方式。

## 财务与合同风险
1. **验收标准缺失（高风险）**：未定义"试运行通过"的客观判定标准，可能导致尾款 60,000 元长期挂账，形成应收账款风险。
2. **范围边界模糊（高风险）**：
   - "知识库初始化"未约定数据来源、数量、清洗深度、是否含历史工单整理，存在被无限拉长工作量的可能。
   - "1 个客服 Agent"未约定意图数、对话流复杂度、上线渠道，可能被解读为无上限定制。
3. **第三方接口费用归属不清（中-高风险）**：若客户工单/CRM/IM 系统对接产生授权费、接口开发费、调用费，未明确由谁承担，可能侵蚀项目毛利。
4. **变更计费机制缺失（高风险）**：无变更单（CR）流程与单价/人天费率，后续需求增减无法计费，易产生免费定制压力。
5. **培训范围风险（中风险）**："2 次远程培训"未约定每次时长、人数、对象、是否含答疑期，可能被要求无限追加。
6. **付款节奏风险（中风险）**：仅两段 50/50，未设置里程碑或启动费/质保金安排，若试运行被拖延，尾款风险集中。
7. **质保与售后责任未提（中风险）**：未约定试运行后是否含质保期、运维范围与边界，可能延伸为无偿长期服务。
8. **知识产权与数据条款缺失（中风险，依据不足，标注不确定）**：草案未涉及客户数据使用、模型产出归属、保密条款，需法务确认。

## 待确认条款（建议补齐后再外发）
- 数据清洗边界：数据源清单、数据量上限、清洗规则、超出部分计费方式。
- 试运行通过标准：可量化指标（如意图识别准确率、回答采纳率、稳定运行天数、问题修复 SLA）、判定主体、判定时限、超时视为通过的兜底条款。
- 第三方系统接口：涉及系统清单、接口费用与授权费由哪方承担、配合方义务与时限。
- 变更需求计费：CR 流程、人天单价或包干价、最小计费单位、审批人。
- 客服 Agent 规格：意图数量上限、对话流条数、接入渠道范围、超出部分计费。
- 培训范围：每场时长、参与人数、培训对象、答疑期长度。
- 质保与运维：是否含质保期、时长、范围、超出部分报价方式。
- 付款节奏可选优化点：是否引入启动预付款、里程碑款或质保金（需商务与财务定）。
- 法务条款：保密、数据合规、知识产权归属、违约与终止条款。

## 不予直接确认事项
本审查**不替公司或客户确认最终条款**，包括但不限于：报价金额、折扣、付款比例与时点、验收标准的最终判定口径、第三方费用承担方、变更单价、最终合同文本。以上均需对应角色审批后定稿。

## 建议交接复核
- **商务/销售负责人**：确认范围边界、客户预期、报价与付款节奏。
- **交付/项目经理**：定义试运行通过标准、培训与变更流程、工作量评估。
- **财务**：核对付款节奏、应收账款风险、税费与开票安排。
- **法务**：审阅验收、变更、质保、保密、数据合规、IP 与违约条款。
- **技术负责人**：评估第三方接口可行性与费用、数据清洗工作量。
- **风险/合规复核（如适用）**：客户数据处理合规性确认。

## 不确定性标注
- 客户行业、第三方系统清单、客户数据规模、公司既有 CR 与质保模板均未提供，相关风险量化与条款建议仅基于草案信息，**不构成最终条款建议**。
