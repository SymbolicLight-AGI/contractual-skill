# 报价草案风险审查

## 一、财务风险

1. **收款节奏与成本不匹配**
   - 仅"签署 50% + 试运行通过 50%"两阶段，若试运行周期被拉长或迟迟无法验收，后 50% 回款风险较高。
   - 未约定试运行通过的时限及"逾期视同通过"条款，存在尾款长期挂账风险。

2. **范围外成本可能由我方承担**
   - 第三方系统接口费用（如工单系统、CRM、客服平台 API 调用费、授权费）归属未明，可能被默认包含在 120000 元内。
   - 数据清洗工作量未设上限，存在工时无限扩张但报价固定的亏损风险。

3. **变更需求无计费机制**
   - 未约定变更单（Change Request）流程及计费标准，客户中途追加需求时缺乏额外收费依据。

4. **税费与开票条款缺失**
   - 未注明 120000 元是否含税、税率、发票类型与开具时点，可能影响实际到账金额。

5. **差旅与培训成本**
   - "2 次远程培训"虽为远程，但未约定单次时长、人数上限、超出后的计费方式；若后续要求现场培训，差旅成本归属不清。

## 二、合同风险

1. **验收标准缺失（核心风险）**
   - "试运行通过"无客观、可量化的验收指标（如准确率、响应时长、覆盖问题数、试运行周期），易引发尾款争议。

2. **工作边界不清**
   - "知识库初始化"未定义数据来源、数据量上限、清洗深度（去重/标注/分类到何种粒度）。
   - "1 个客服 Agent"未约定支持的渠道数、并发量、意图数量、知识库条目上限。

3. **责任划分缺失**
   - 客户配合义务未写明（数据提供时限、对接人、第三方系统权限开通等），客户延误导致工期顺延的处理方式不明。
   - 数据安全、隐私合规（如个人信息处理）、数据归属与销毁义务未约定。

4. **知识产权与使用范围**
   - Agent、模型微调产物、Prompt、知识库结构的知识产权归属未约定。
   - 是否允许客户在其他业务线复用、是否限制并发使用未说明。

5. **服务期限与售后**
   - 未写明交付后的保修期、Bug 修复 SLA、是否含运维与模型更新。
   - "首期"暗示后续仍有合作，但未约定后续报价机制或排他性。

6. **终止与违约**
   - 缺少单方解约、违约金、不可抗力、争议解决（管辖法院/仲裁）条款。

## 三、建议待确认条款清单

请与客户及内部商务/法务进一步明确以下事项（不代为决定具体数值）：

| 序号 | 待确认事项 |
|---|---|
| 1 | 报价是否含税、税率、发票类型与开具节点 |
| 2 | 试运行通过的量化验收标准（指标、样本、周期、超期默认机制） |
| 3 | 数据清洗的输入边界：来源、数据量上限、清洗粒度、超量计费规则 |
| 4 | 第三方系统接口费用（接口开发、第三方授权、调用量费用）的承担方 |
| 5 | 变更需求的提出流程、计价单位（人天/模块）、单价 |
| 6 | "1 个 Agent"的具体技术边界：渠道、并发、意图数、知识条目上限 |
| 7 | 2 次远程培训的单次时长、人数、对象、追加培训的报价 |
| 8 | 客户配合义务及其逾期的工期与费用影响 |
| 9 | 数据安全、隐私合规责任与数据归属/销毁约定 |
| 10 | 交付物及衍生成果（模型、Prompt、知识结构）的知识产权归属与使用范围 |
| 11 | 交付后保修期、SLA、运维与模型迭代是否含在 120000 元内 |
| 12 | 付款节奏（建议增加里程碑或封顶付款时限）与逾期付款违约金 |
| 13 | 单方解约、违约责任、不可抗力、争议解决机制 |

## 四、不确定性说明

- 草案中"首期"二字暗示存在二期/续约，但材料未提供后续合作信息，相关排他性、续约定价等无法判断，需向业务方确认。
- 行业惯例下"远程培训""知识库初始化"范围差异较大，未掌握贵司标准化交付定义，以上边界问题以草案文本为准提示。
