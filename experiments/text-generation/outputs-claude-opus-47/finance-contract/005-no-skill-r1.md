# 判断结论

**不可以直接走合同。** 当前信息存在范围模糊、验收缺失、报价不闭环等重大风险，必须经多角色复核后再签约。

---

## 一、核心问题清单

| 序号 | 问题 | 风险等级 |
|------|------|----------|
| 1 | 一份合同覆盖"试点 + 8 家分公司推广"全部阶段 | 高 |
| 2 | 报价需包含 OA / CRM / 售后工单全部接口，但接口规格未明确 | 高 |
| 3 | 验收标准"后续再定" | 极高 |
| 4 | 5 个业务 Agent 的边界、知识库范围未量化 | 高 |
| 5 | 分公司推广的触发条件、计价方式未约定 | 高 |

---

## 二、必须复核的角色及事项

### 售前
- 澄清 5 个业务 Agent 的功能边界、知识库数据范围与来源。
- 摸排 OA / CRM / 售后工单的接口形态（是否有标准 API、是否需定制）。
- 明确试点范围（哪家分公司、哪些业务场景）。
- 确认分公司推广的标准化程度（是否每家差异化实施）。

### 交付
- 评估"一份合同覆盖全部阶段"的可行性，建议拆分为试点期 + 推广期分阶段交付/分阶段验收。
- 提出可量化的验收标准模板（功能项、性能项、知识库覆盖率、Agent 准确率等）。
- 评估 3 个外部系统对接的工作量与依赖客户配合事项。
- 评估 8 家分公司推广的人力、周期、现场支持需求。

### 财务
- 拒绝"包含所有接口"的一口价模式，要求按试点 / 推广分期计价或设置接口数量上限。
- 明确收款节点（签约款、试点验收款、推广分期款、质保金）。
- 评估推广阶段未确定情况下的收入确认风险与回款周期。
- 测算成本构成中接口定制、分公司差旅、运维的占比。

### 法务
- 反对"验收标准后续再定"，必须在合同或附件中先行约定验收原则与方法，未达成不得签约。
- 设计"分阶段签署 + 推广 SOW 另签"的合同结构，或在主合同中加入推广启动的前置条件条款。
- 明确范围变更（接口新增、Agent 新增、分公司增加）的变更管理与计价条款。
- 约定第三方系统（OA / CRM / 工单）接口不开放或客户配合不到位的免责与顺延条款。
- 知识产权归属（知识库内容、Agent 模型、Prompt 资产）需明确。
- 数据安全与保密条款（集团知识库涉及敏感数据）。

### 风险审查
- 范围蔓延风险：5 Agent + 3 系统 + 8 分公司，缺乏边界控制。
- 验收风险：验收标准缺失 = 合同款项无法闭环。
- 报价风险：接口工作量未知却要求全包，存在严重亏损可能。
- 推广不确定性风险：试点失败或客户不推广时的退出机制。
- 合规与数据风险：集团知识库涉及跨公司数据流转，需评估数据合规。

---

## 三、建议处置路径

1. **暂缓签署单一总包合同。**
2. 建议合同结构调整为：**主框架协议 + 试点 SOW + 推广 SOW（后签）**。
3. 在签约前完成：验收标准附件、接口清单附件、变更管理条款。
4. 上述五个角色全部出具复核意见后，方可进入签约流程。

> 不确定性说明：客户对"一份合同"的坚持程度、3 个外部系统的接口开放情况、分公司推广的预算归属（总部 or 分公司）等信息缺失，需销售进一步澄清后再最终定稿。
