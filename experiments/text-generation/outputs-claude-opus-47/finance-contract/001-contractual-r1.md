# 财务与合同风险

- 风险（范围）：数据清洗边界未定义，可能被客户要求无限次清洗或扩展数据源，导致工作量失控。
- 风险（验收）：试运行通过标准缺失，第二期 50% 回款触发条件不可量化，存在"客户主观不通过"导致尾款拖欠的可能。
- 风险（成本）：第三方接口费用归属不清，若由我方承担可能直接侵蚀毛利甚至亏损。
- 风险（变更）：未约定变更计费方式，易发生"小步快跑"式无偿增量需求。
- 风险（现金流）：尾款 50% 与不明确验收标准绑定，回款周期不可控。
- 风险（信用）：未见客户信用核查，先签后付模式下存在坏账可能。
- 风险（范围扩张）：1 个 Agent、2 次培训未定义边界（如调用量、培训时长/人数），易被理解为无限使用。
- 风险（合规/隐私）：未明确客户数据使用、存储、删除责任，可能引发合规问题。

# 待确认问题

1. 数据清洗：客户提供数据的格式、数量上限、清洗深度（去重/结构化/标注）、客户侧配合责任？
2. 试运行通过标准：采用哪些可量化指标（如准确率、解决率、响应时长）？样本量与测试周期？由谁判定？争议如何解决？
3. 第三方接口费用：是否包含在 120,000 元内？若包含，上限是多少？超出由谁承担？
4. 模型调用成本（token/API）：归属与上限？
5. 变更管理：变更如何定义？计费单位（人天/模块）？审批流？
6. Agent 使用边界：调用量/并发/坐席数上限？
7. 培训边界：每场时长、参训人数、是否含培训材料、是否限同一地点？
8. 交付周期与里程碑、SLA、质保期、终止与退款条款？
9. 客户信用状况与发票/税务安排？
10. 报价是否已通过内部定价与折扣审批？

# 交接建议

- 财务：复核报价是否符合定价政策；测算 120,000 元在含/不含第三方费用情景下的毛利；评估 50/50 付款节奏的现金流风险；核查客户信用。
- 法务：起草试运行验收条款、变更条款、SLA、质保与终止条款；审查数据合规与责任划分。
- 交付 PM：明确数据清洗边界、Agent 与培训边界、交付里程碑及工作量估算；提出可量化的验收指标建议。
- 风险评审：对验收标准模糊、第三方成本不明、变更未定价等高风险点进行联合评审，确定是否需要追加条款或调整报价结构。
