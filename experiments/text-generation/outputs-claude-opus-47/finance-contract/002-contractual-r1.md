# 结论摘要

**不建议直接回复客户"可以先做后付"。** 当前缺少客户信用评估、项目正式范围、合同草案及成本核算等关键信息，直接承诺"先做后付"将使我方承担全部前期投入风险、收款风险与范围扩张风险。建议先以"待内部评估付款方案"为口径回应客户，同步推动信用核查、范围确认与报价审批流程。

# 已知事实

- **Known term**：客户为某中型制造企业。
- **Known term**：客户主动提出"先做后付"的付款诉求。
- **Known term**：项目尚未形成正式方案。
- **Known term**：合同尚未起草。
- **Known term**：我方预计成本项包含知识库实施人员投入、可能的现场调研、可能的第三方工具费用。

# 缺失信息

- **Missing term**：客户信用记录、回款历史、付款能力评估。
- **Missing term**：项目正式范围说明（功能边界、交付物清单、阶段划分）。
- **Missing term**：验收标准与验收触发条件。
- **Missing term**：报价金额与折扣审批依据。
- **Missing term**：成本基线（人员工时、现场调研次数与差旅、第三方工具具体清单与单价）。
- **Missing term**：付款里程碑设计（是否分阶段、预付比例、尾款条件）。
- **Missing term**：客户主体合同签署人、法务对接方、采购流程。
- **Missing term**：违约与终止条款、SLA、知识产权归属。
- **Missing term**：内部审批层级（折扣矩阵、特殊付款条件审批人）。

# 财务与合同风险

- **Risk – 现金流风险**：在范围与成本均未确定时承诺先做后付，前期人员、调研、第三方工具支出全部由我方垫付，回款周期不可控。
- **Risk – 坏账风险**：客户信用未知，中型制造企业存在资金紧张或拖欠可能，无信用核查即先行交付等同无担保授信。
- **Risk – 范围蔓延风险**：无正式方案即开工，客户可能持续追加需求，导致"做完才发现做不完"，付款触发条件无依据。
- **Risk – 验收争议风险**：无验收标准 + 后付模式 = 客户可借"不满意"延迟或拒付。
- **Risk – 成本失控风险**：现场调研与第三方工具均为"可能"，未量化即承诺总价或后付，会侵蚀毛利。
- **Risk – 合同与法律风险**：无合同先交付，争议时缺乏书面依据，追偿成本高。
- **Risk – 审批合规风险**：销售单方答复客户付款条件可能绕过折扣与付款条款审批矩阵。

# 不得直接承诺事项

- 不得承诺"先做后付"、"先实施后签合同"、"验收后再付款"等延后付款安排。
- 不得承诺无预付款启动项目。
- 不得承诺包含现场调研、第三方工具费用的固定总价。
- 不得承诺验收标准、SLA、无限范围或无限次修改。
- 不得承诺折扣或减免任何费用。
- 不得在未签署合同前安排实施人员进场。

# 待确认问题

1. 客户为何提出先做后付？是流程惯例、预算审批节奏，还是对我方交付能力存疑？
2. 客户内部采购与付款流程是怎样的？是否有标准合同模板？
3. 客户能否提供工商、征信或过往合作方付款记录以支持信用评估？
4. 项目范围是否可先以付费 POC 或诊断咨询形式启动，再过渡到正式实施？
5. 我方成本中现场调研与第三方工具的量级如何估算？是否可由客户承担第三方工具直采？
6. 是否可接受分阶段付款（如签约预付 + 里程碑 + 验收尾款）替代全额后付？
7. 该单的折扣与付款条件特批权限在谁？是否已进入审批矩阵？

# 交接建议

- **Finance（财务）**：评估客户信用、设计付款里程碑、核定预付比例与坏账敞口、审批任何延期付款方案。
- **Legal（法务）**：起草合同框架，明确验收、SLA、违约、知识产权、终止条款；审定任何"先做后付"相关条款。
- **Delivery PM（交付）**：在签约前完成范围澄清、工作量估算、现场调研必要性判断、第三方工具清单与成本。
- **Risk Review（风险评审）**：审议无预付/低预付方案、跨阶段授信、未签约先实施等高风险安排。
- **Sales（销售本人）**：先以"我需要走内部评估，给您一个更稳的付款方案"口径回应客户，避免口头承诺先做后付。

**可向客户回复的草拟话术（非最终承诺，仅供内部参考草稿）：**
> "关于付款方式，我们理解贵司的考虑。为了让方案和报价更贴合贵司需求，我建议我们先用一到两周时间做一次范围澄清（必要时含一次现场调研），输出正式方案与分阶段实施计划，并据此商定一个对双方都稳妥的分期付款安排。这样既能让贵司在每个阶段看到成果再付款，也能让我们的投入有据可依。"
