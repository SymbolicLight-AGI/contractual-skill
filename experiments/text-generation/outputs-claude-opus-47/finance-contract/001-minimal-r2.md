# 财务与合同风险检查：售后知识库 Agent 试点报价

## 一、识别的风险

### 1. 范围风险（Scope Risks）
- **数据清洗边界未定义**：知识库初始化通常涉及大量原始数据处理，未约定清洗范围（数据来源、数据量上限、清洗深度、是否包含历史数据迁移）极易导致工作量失控。
- **"1 个客服 Agent" 定义模糊**：未说明 Agent 的能力边界（意图数量、问答覆盖率、多轮对话深度、知识库条目上限、并发量等）。
- **"2 次远程培训" 颗粒度不足**：未约定单次时长、培训对象人数、培训内容深度、是否提供材料、是否含答疑。
- **第三方系统接口费用归属不明**：若客户的工单/CRM/IM 系统需要对接，接口开发、第三方授权费、调用费由谁承担未明确，可能产生大额隐性成本。

### 2. 财务风险（Financial Risks）
- **付款节奏偏后**：50% 尾款绑定"试运行通过"，但试运行标准缺失，可能导致**尾款无限期拖延**甚至无法回款。
- **缺少里程碑分期**：120000 元仅分两期，中间无阶段性回款，现金流压力集中在尾款。
- **无逾期付款条款**：未见付款延期的违约责任、滞纳金或停工条款。
- **无发票/税务约定**：未说明开票类型（专票/普票）、税率、开票时间节点。

### 3. 合同/验收风险（Acceptance & Change Risks）
- **试运行通过标准缺失**：这是最关键的风险点——无客观验收指标（如准确率、响应时长、试运行周期），尾款回收完全由客户主观判断。
- **变更计费方式缺失**：客户提出新需求（新增意图、扩充知识库、追加培训）时无计费规则，极易陷入"免费加需求"循环。
- **未约定服务期限与运维责任**：试点期后是否含质保、Bug 修复期、运维责任范围均未提及。
- **未约定知识产权归属**：Agent 配置、Prompt、知识库结构、训练数据的 IP 归属未明确。
- **未约定保密与数据安全**：涉及客户售后数据，未见数据使用、留存、销毁条款。

## 二、待确认条款清单（建议与客户/内部对齐）

| # | 待确认项 | 建议确认方 |
|---|---------|-----------|
| 1 | 数据清洗范围：数据源数量、条目上限、清洗深度 | 交付 + 客户 |
| 2 | 1 个 Agent 的能力指标（意图数、覆盖率、并发等） | 产品 + 交付 |
| 3 | 2 次培训的单次时长、人数、内容大纲 | 交付 |
| 4 | 第三方接口费用归属（开发费 + 第三方授权/调用费） | 商务 + 客户 |
| 5 | 试运行通过的客观标准（准确率、响应时间、试运行周期、样本量） | 产品 + 客户 |
| 6 | 变更请求的计费规则（人天单价、最小计费单位、审批流程） | 商务 |
| 7 | 付款节奏（是否拆分启动款、交付款、验收款） | 财务 |
| 8 | 尾款触发的兜底条款（最长验收期限、视同通过条件） | 法务 + 财务 |
| 9 | 开票类型、税率、开票时点 | 财务 |
| 10 | 质保期、Bug 响应 SLA、试点后运维报价 | 交付 |
| 11 | 知识产权归属（Prompt、配置、数据） | 法务 |
| 12 | 数据保密、留存、销毁条款 | 法务 + 安全 |

---

## 三、建议的下一步

1. **暂不发出当前报价草案**——核心验收标准与变更规则缺失，回款风险较高。
2. 由**交付/产品**先与客户对齐范围与试运行通过标准，量化为可验收指标。
3. 由**商务/财务**评估是否引入第三期里程碑款，并补充逾期与兜底条款。
4. 由**法务**补齐 IP、保密、数据安全、变更管理条款。
5. 第三方接口费用建议在报价中**单独列示或明确排除**，避免混入总价。
6. 待上述条款齐备后，再走内部审批流程，由有权限方确认最终条款。

---

> 说明：本检查仅识别风险并列出待确认事项，**未对任何折扣、付款条件、范围或最终条款进行批准**。最终条款需由商务、财务、法务及交付负责人按内部流程审批确认。
