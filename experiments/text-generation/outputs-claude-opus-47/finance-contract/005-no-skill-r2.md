# 合同可行性判断

**结论：不可直接走合同。** 当前材料存在范围未冻结、验收标准缺失、接口口径未确认等关键缺陷，须经多角色复核后方可签署。

---

## 必须复核的事项分配

### 一、售前（方案 & 报价）
1. 拆分项目阶段：知识库搭建、5 个 Agent 开发、3 系统对接、8 家分公司推广，逐项明确工作量与里程碑。
2. 确认"报价包含所有系统接口"的口径：OA / CRM / 售后工单的接口是否已具备、是否含定制开发、是否含第三方授权费。
3. 8 家分公司推广的报价模型：按公司数计价还是包干，是否含本地化适配与培训。
4. 多 Agent 工作流的算力 / 模型调用成本测算（Token、并发、SLA）。

### 二、交付
1. 集团总部 + 8 家分公司的部署架构、数据隔离与权限模型可行性。
2. 5 个业务 Agent 的范围边界、与现有系统的依赖关系。
3. 试点验收转推广的准入条件（gate 条件）。
4. 接口对接所需的甲方配合资源（账号、测试环境、接口文档提供方）。
5. 项目排期与人力投入估算。

### 三、财务
1. "一份合同覆盖全部阶段"下的收入确认节奏与回款节点设计（避免长周期 + 模糊验收导致挂账）。
2. 试点与推广阶段的计价拆分、发票节奏、税务处理。
3. 成本预估与毛利测算，重点关注接口定制与推广阶段成本不确定性。
4. 预收 / 里程碑款比例建议。

### 四、法务
1. **"验收标准后续再定"为重大风险**，必须要求在合同中以附件形式约定验收标准制定机制、时限及未达成时的处置条款。
2. "一份合同覆盖全部阶段"需设置分阶段终止 / 暂停条款，避免试点失败仍被绑定推广义务。
3. "报价包含所有系统接口"条款的范围界定，明确接口清单、变更管理与超范围收费机制。
4. 知识产权归属（知识库内容、Agent 模型、二次开发成果）。
5. 数据安全、保密、个人信息保护条款（涉及 CRM、工单数据）。
6. 违约责任、SLA、责任上限。

### 五、风险审查
1. 验收标准缺失 → 收入无法确认、烂尾风险。
2. 范围蔓延风险：8 家分公司推广未冻结需求，易演变为无限期服务。
3. 第三方系统（OA / CRM / 售后工单）接口不可控风险，建议设置甲方配合义务与免责条款。
4. 试点失败的退出机制。
5. 多 Agent 长期运行的模型成本波动风险（如外部 LLM 调价）。

---

## 建议处置

- **方案 A（推荐）**：签订框架合同 + 分阶段 SOW，先锁定试点范围与价格，推广阶段以补充协议形式落地。
- **方案 B**：若客户坚持单合同，则必须在签署前补齐《验收标准附件》《接口清单附件》《推广阶段单价表》，否则不予用印。

## 不确定性标注
- 客户对"接口"的具体定义（标准 API / 定制开发 / 数据打通深度）未知，报价完备性无法判断。
- 8 家分公司的 IT 环境一致性未知，推广成本浮动区间无法量化。
- 试点范围（哪家先试点、试点时长）未提供，里程碑设计待补充。
