# 财务与合同风险

- Risk（范围）：一份合同覆盖试点 + 8 家分公司推广，属于跨阶段、范围未冻结合同，极易导致后续无限扩展。
- Risk（验收）：验收标准"后续再定"将使收款触发条件不可执行，存在长期挂账风险。
- Risk（接口）：报价包含"所有系统接口"但接口清单未定，等同于无限接口承诺，成本不可控。
- Risk（推广）：8 家分公司推广可能涉及不同系统版本与本地化，按单一报价覆盖会侵蚀毛利。
- Risk（现金流）：缺少里程碑与付款触发条件，回款节奏不可预测。
- Risk（信用）：未提供客户信用与回款历史，无法评估"先做后付"类风险。
- Risk（法律）：跨阶段单一合同若无变更与终止条款，难以在范围失控时止损。
- Risk（成本）：成本基线缺失，无法判断报价是否覆盖真实交付成本。

# 不得直接承诺事项

- 不得承诺"一份合同覆盖试点 + 全部 8 家分公司推广"。
- 不得承诺"报价包含所有系统接口"。
- 不得承诺"验收标准后续再定"作为合同条款。
- 不得承诺无明确触发条件的付款安排。
- 不得承诺未审批的折扣、让步或免费定制。
- 不得承诺无上限的培训、运维或 SLA。

- 售前：澄清客户对"一份合同覆盖全阶段"的真实诉求，回推"主合同 + 分阶段 SOW"或"试点合同 + 推广合同"备选方案（草案语言，非最终承诺）。
