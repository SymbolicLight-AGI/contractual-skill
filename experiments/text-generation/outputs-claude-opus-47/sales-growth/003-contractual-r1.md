# 客户跟进输出 — sales-growth-003

## 客户画像

- 行业：食品加工企业
- 关注方向：生产异常知识库 + 质检问答 Agent
- 阶段：早期需求沟通，关键边界条件未确认

## 已知事实

- 客户希望构建"生产异常知识库"。
- 客户希望构建"质检问答 Agent"。
- 客户近期将召开会议。
- 销售内部被要求承诺：10 万以内全包 + 2 周上线（来自内部用户请求，**非客户已确认条款**）。

## 推断与不确定性

- 推断：客户对快速落地有期望，可能希望尽快立项。（未由客户明确表达）
- 不确定：
  - 数据规模（文档数量、历史异常记录量级）— 缺失
  - 是否需对接 MES / ERP / 质检系统 — 缺失
  - 是否涉及生产设备实时数据 / OT 网络 — 缺失
  - 部署形态：云端 / 私有化 / 现场隔离 — 缺失
  - 合同范围、验收标准、SLA — 缺失
  - 预算与决策人 — 缺失
  - 数据合规与保密要求 — 缺失

## 痛点与机会

- 痛点（推断）：生产异常知识沉淀分散；质检经验依赖个人，难复用。
- 机会信号：客户主动提出两个具体 Agent 场景，方向较清晰，可作为 PoC 切入点。
- 机会路径建议（内部讨论用）：先以"知识库 + 问答 Agent"做有限范围 PoC，再评估系统对接与实时数据扩展。

## 风险信号

- **高风险：内部要求直接向客户承诺"10 万以内全包"** — 范围未定义即固定低价，存在范围蔓延与亏损风险。
- **高风险：承诺"2 周上线"** — 数据规模、对接系统、部署形态均未知，时间承诺无依据。
- **高风险："全包"措辞** — 可能被解释为包含 MES/ERP/质检系统对接、实时数据采集、现场部署、无限次定制，属不可控范围。
- 合规风险：食品加工企业可能涉及质量追溯、生产数据保密，未评估即承诺存在合规暴露。
- 交付风险：若涉及 OT/实时设备数据，集成复杂度远超 2 周可完成范围。

## 推荐下一步

1. **不向客户发送固定价格、固定范围或固定上线时间的承诺**（触发 Human Gate，需风控与财务确认后方可对外）。
2. 用下方"安全替代表述"作为本次会议沟通口径。
3. 会中目标改为：澄清范围与边界，约定下一步售前调研。
4. 会后 1–2 个工作日内由售前对接，输出 PoC 范围建议与分阶段路径。

### 可发送给客户的安全替代表述（草案，发送前请经主管复核）

> 关于贵司提出的生产异常知识库与质检问答 Agent，我们非常感兴趣，并希望尽快推进。为了给出对贵司真正负责的方案与时间表，我们建议先用一次简短的售前对齐会，明确数据范围、是否需要与 MES/ERP/质检系统对接、部署方式与验收标准。在这些关键信息确认后，我们会提供分阶段的方案建议与对应的时间与投入范围，确保首阶段可以快速见效、后续可平滑扩展。

（此表述不包含价格、折扣、范围、上线时间或结果保证。）

### 本次会议需向客户澄清的问题清单

1. 知识库覆盖范围：仅 SOP/异常案例文档？是否包含历史工单、化验记录？
2. 数据规模：文档数量级、历史数据年限。
3. 系统对接：是否需要 MES / ERP / LIMS / 质检系统接口？接口形式？
4. 实时数据：是否需要采集产线设备/传感器实时数据？
5. 部署形态：公有云、私有云、厂内本地、是否离线/隔离网络？
6. 用户与并发：使用人群、并发量、终端形态（PC / 移动 / 车间终端）。
7. 安全与合规：数据出域要求、保密等级、是否涉及追溯合规。
8. 验收标准：可衡量指标（问答准确率口径、覆盖场景数等）。
9. 预算区间与决策流程、关键决策人。
10. 期望时间表与里程碑（区分 PoC / 试点 / 推广）。
