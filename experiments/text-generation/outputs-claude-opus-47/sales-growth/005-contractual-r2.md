# 客户跟进与交接建议

## 客户画像

- 行业：装备制造集团（多层级组织：集团、分公司、服务商）。
- 业务诉求方向：售后服务数字化（知识库、工单分类、系统打通）。
- 关注层级：管理层关注试点 ROI，存在自上而下的可见度需求。
- 采购成熟度：尚不明确（决策人、预算、采购流程均未提供）。

## 已知事实

- 客户提出四项需求：售后知识库、工单自动分类、与现有 CRM 和备件系统打通、管理层希望看到试点 ROI。
- 客户要求"先把所有系统都接上，再看效果"。
- 客户希望报价里包含后续所有二开。
- 数据权限涉及集团、分公司、服务商三方。

## 推断与不确定性

- 推断：客户内部可能尚未对"试点范围"形成共识，管理层要 ROI 与一线要"全接入"之间存在张力。
- 推断：CRM 与备件系统可能是异构、多版本或多分公司部署，集成复杂度未知。
- 不确定性（缺失信息）：
  - 预算区间未提供。
  - 决策人/签约主体未提供（集团 vs 分公司）。
  - 期望试点时间窗与上线时间未提供。
  - 现有 CRM、备件系统的厂商、接口能力、数据口径未提供。
  - 服务商是否为合同相对方、是否参与数据使用未明确。
  - ROI 衡量口径（成本节约？响应时长？一次解决率？）未提供。

## 痛点与机会

- 痛点：
  - 售后知识沉淀分散，工单分派依赖人工。
  - 集团-分公司-服务商之间的数据可见度与权限边界不清。
- 机会：
  - 知识库 + 工单分类是相对独立、可短期出效果的切入点，适合做"小范围可量化试点"。
  - 管理层对 ROI 的关注，可作为推动分阶段交付而非一次性大集成的抓手。

## 风险信号

- 范围风险：客户要求"先全接，再看效果"——典型的低价固定范围 + 无限集成期望，存在范围蔓延与交付不可控风险。**需风险审查**。
- 商务风险：客户希望报价包含"后续所有二开"——无限定制条款，价格与工作量无法封顶。**需风险审查 + 财务合同**。
- 合规/数据风险：集团、分公司、服务商三方数据权限——涉及数据归属、跨主体授权、个人信息处理边界，可能需要 DPA 或分级授权方案。**需风险审查 + 售前方案**。
- 集成可行性风险：CRM 与备件系统接口未知，"全部打通"不可在未评估前承诺。**需售前方案**。
- 期望管理风险：管理层 ROI 期望与"先全接再看"路径互相矛盾，若不澄清将影响验收。**需交付经理介入**。

## 推荐下一步

销售可继续推进的事项：
1. 与客户对齐"试点"定义：建议以知识库 + 工单分类为最小可行范围，先不绑定全量集成。
2. 收集缺失信息：决策人、预算区间、期望时间窗、CRM/备件系统的厂商与版本、ROI 衡量指标、三方数据权限主张。
3. 安排联合调研会（销售 + 售前），不在会上做价格、范围、上线时间、集成保证的承诺。
4. 准备非承诺性的"试点价值假设"材料（仅作为讨论稿，不作为报价）。

必须交由其他角色确认的事项（不可由销售单独承诺）：
- 是否能"全系统打通" → 售前方案。
- 二开范围与边界 → 售前方案 + 财务合同。
- 报价、是否包含后续二开、付款条件 → 财务合同。
- "先全接再看效果"的范围/价格条款 → 风险审查。
- 三方数据权限与合规边界 → 风险审查 + 售前方案。
- 试点时间表、上线节点、验收标准、ROI 度量口径 → 交付经理。

## 交接建议

**交售前方案**
- 评估 CRM、备件系统集成可行性与工作量分级。
- 给出知识库 + 工单分类的最小可行试点方案（不承诺全量集成）。
- 提出三方数据权限的技术方案选项（如分级账号、数据隔离、审计）。

**交风险审查**
- "全部系统集成"承诺的范围风险。
- "报价包含所有后续二开"的无限定制条款风险。
- 集团/分公司/服务商三方数据使用授权的合规风险。
- 是否需要 DPA、数据处理附录、跨主体合同结构。

**交交付经理**
- 试点 ROI 度量指标定义与基线数据来源。
- 试点周期、里程碑、验收标准草案（不对客户承诺，仅内部对齐）。
- 数据准备度评估（知识来源、工单历史数据可用性）。

**交财务合同**
- 试点与后续阶段的报价结构建议（分阶段、不含无限二开）。
- 二开的计价方式（如按工作量、按变更单）。
- 付款节点与验收挂钩条款。

**个人联系信息**：本材料未涉及个人联系方式；若后续生成群内可见摘要，需对接人姓名/手机/邮箱做脱敏。
