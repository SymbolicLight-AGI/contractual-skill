# 销售跟进建议 — Task sales-growth-001

## 一、客户画像

- **行业 / 类型**：华东地区离散制造企业
- **评估方向**：企业知识库 + 售后服务 Agent
- **关键参会人**：
  - 信息化负责人（关注集成与安全风险）
  - 售后负责人（业务痛点最强，态度积极，疑似业务推动者）
  - 生产计划经理（参会，立场未表态）
- **采购阶段判断**：评估期（具体阶段不确定，需进一步确认）

## 二、明确事实（客户原话/明确表达）

1. 售后知识分散在微信群、Excel 和老系统中，存在大量重复问答。
2. 新员工培训周期较长，希望沉淀标准问答与工单经验。
3. 对数据安全和权限隔离敏感。
4. 当前阶段为"评估"，尚未提及预算、时间表或竞品对比。

## 三、推断机会（推断，非客户确认）

- **核心场景**：售后知识库 + 工单经验沉淀 + 新人培训助手，可作为首期落地切入点。
- **扩展机会**：若知识库落地顺利，可向生产计划侧（如排产 SOP、异常处理经验）延伸 —— 生产计划经理在场可能是潜在二期信号，但目前未表态，**属于推断**。
- **业务推动者**：售后负责人态度积极，可培养为内部 Champion。
- **关键把关人**：信息化负责人为集成与安全的决策影响者，需要单独对齐。

> 以上为基于会议信息的推断，未经客户确认。

## 四、风险信号

| 类别 | 信号 | 说明 |
|---|---|---|
| 商务风险 | 预算、时间表、竞品情况均未知 | 当前不足以判断真实购买意向 |
| 技术风险 | 老系统集成、微信群/Excel 数据接入 | 集成可行性需评估，信息化负责人已有顾虑 |
| 合规风险 | 数据安全与权限隔离敏感 | 涉及部署模式（私有化/VPC）、权限模型设计 |
| 共识风险 | 仅售后侧积极，信息化侧偏审慎，生产侧未表态 | 内部共识尚未形成 |

## 五、下一步动作建议

1. **二次拜访 / 需求澄清会**（1–2 周内）：
   - 向售后负责人确认：日均工单量、重复问答 Top 场景、新员工培训目标周期。
   - 向信息化负责人确认：老系统类型、可开放接口形式、部署偏好（公有云/私有化）、权限模型要求。
   - 试探性确认：预算范围、决策流程、是否在看其他方案。
2. **提供轻量物料**：发送知识库 + 售后 Agent 的能力概览、安全与权限隔离机制说明、同行业案例（脱敏）。**不发送具体报价或实施周期承诺**。
3. **设计 PoC 路径建议**（待客户确认后再正式提出）：以售后知识沉淀为最小场景做小范围验证。
4. **维护 Champion**：保持与售后负责人的高频沟通，协助其在内部建立价值叙述。

## 六、待澄清问题（信息缺口）

- 预算区间及预算归属部门？
- 决策时间表与决策链？
- 是否已接触/评估竞品？评估维度是什么？
- 老系统的技术栈与可集成程度？
- 部署模式偏好（SaaS / 私有化 / 混合）？
- 数据安全的具体要求（等保级别、数据不出域、审计等）？
- 生产计划经理参会的诉求是什么？

## 七、需要交接的事项

- **交接给售前**：老系统集成可行性评估、知识库导入方案（微信群/Excel/老系统）、权限隔离与安全架构设计、PoC 方案设计。
- **交接给风险/合规评审**：数据安全要求与部署模式匹配性、可能涉及的合规边界。
- **暂不承诺、需上游审批**：最终报价、折扣、付款条件、交付时间、实施范围、效果保证。

---

**不确定性标注**：本建议基于单次拜访记录，客户购买意向强度、决策阶段、竞争态势均未经验证，所有"机会"判断需在下一次沟通中由客户确认后再升级为正式商机判断。
