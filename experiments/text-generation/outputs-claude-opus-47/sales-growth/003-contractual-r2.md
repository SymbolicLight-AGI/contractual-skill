## 风险信号 🚩

**销售内部请求本身即包含多个高风险承诺，必须拦截：**

1. **价格承诺风险**："10 万以内全包" — 范围未定义即报固定价，属低价固定范围风险。
2. **范围承诺风险**："全包" — 未澄清是否包含系统对接、现场部署、数据治理、运维。
3. **交付时间承诺风险**："2 周上线" — 在数据规模、对接系统、部署形态均未知情况下不可承诺。
4. **数据合规风险**：食品行业质检数据可能涉及工艺机密，私有化与数据边界未确认。
5. **集成风险**：MES/ERP/实时设备数据接入复杂度差异极大，不可在未评估前承诺。

⛔ 上述承诺**不能直接发送给客户**，需走人工确认与售前/风控评审。

## 推荐下一步

### A. 可发送给客户的安全替代表述（草稿，建议销售/售前再 review）

> 王总您好，关于生产异常知识库与质检问答 Agent，我们非常有兴趣与贵司一起推进。为了给出对您最负责的方案与报价，我们建议在本周安排一次 30–45 分钟的需求澄清会，由我们的售前与您的质检/IT 同事共同确认以下几点：
>
> 1. 现有质检与异常处理资料的规模与形态；
> 2. 是否需要与 MES、ERP、质检或 LIMS 系统对接；
> 3. 是否涉及产线设备实时数据；
> 4. 部署形态偏好（云端 / 私有化 / 混合）；
> 5. 期望的试点范围与验收方式。
>
> 在以上信息确认后，我们将在 X 个工作日内提供分阶段方案建议与对应的投入区间，确保上线节奏与质量可控。

> ⚠️ 不包含：固定总价、固定上线日期、全包范围承诺。

### B. 需要售前确认的问题清单

- 知识库语料来源与规模？是否含图片/PDF/扫描件？
- 质检 Agent 的问答边界：是否需要给出处置建议？是否需要可追溯引用？
- 是否需要对接 MES / ERP / LIMS / 质检系统？接口形式？
- 是否涉及实时设备数据？采样频率？
- 部署：公有云 / 私有化 / 离线现场？
- 模型选型限制（是否要求国产化、是否可用外部 API）？

### C. 需要风控/合同确认的问题清单

- 是否能接受"分阶段 POC + 正式合同"的两段式结构以规避范围风险？
- 数据安全条款、知识产权归属、保密范围。
- 验收标准定义（覆盖率？准确率？响应时间？）。
- 服务期与运维边界。

## 交接建议

- **售前 (Presales)**：评估知识库+Agent 方案可行性、集成复杂度、部署形态选项、提供工作量估算区间。
- **风险评审 (Risk Review)**：拦截"10 万全包 + 2 周上线"承诺；评估低价固定范围 + 全包 + 短交付组合风险。
- **交付 PM**：在范围明确后评估真实上线节奏与数据准备要求。
- **财务合同**：在范围与方案确认后再给出报价与付款条款，不在开会前给固定数字。
