# 销售跟进建议 — sales-growth-001

## 一、客户画像
- 行业与定位：华东某离散制造企业。
- 当前阶段：方案评估期（评估"企业知识库 + 售后服务 Agent"两类能力）。
- 关键干系人：
  - 信息化负责人：技术守门人，关注集成与安全风险。
  - 售后负责人：业务推动者，态度积极，疑似潜在 Champion。
  - 生产计划经理：参会但立场不明，可能是横向影响人或后续场景拓展点。

## 二、明确事实（客户已表达）
1. 售后知识分散在微信群、Excel、老系统，存在重复问答问题。
2. 希望沉淀标准问答与工单经验，缩短新员工培训周期。
3. 对数据安全与权限隔离敏感。
4. 售后负责人态度积极；信息化负责人关注系统集成风险（我方观察）。

## 三、推断机会（带不确定性标注）
- [较高置信] 售后场景是首个落地切入点：痛点具体、推动人明确。
- [中等置信] 新员工培训提效可作为可量化 ROI 抓手（培训周期、首次解决率、重复问答比例）。
- [中等置信] 生产计划经理在场，暗示后续可能扩展至生产侧知识沉淀或跨部门协同（需进一步确认）。
- [较低置信] 客户可能尚未深度比较竞品，存在引导评估标准的窗口期 —— 待验证。

## 四、风险信号
1. **集成风险**：信息化负责人对老系统对接持谨慎态度，若不正面回应可能成为否决点。
2. **安全合规**：权限隔离、数据出域、模型调用方式需明确说明，否则可能进入冗长的安全评审流程。
3. **关键信息缺失**：预算、决策时间表、竞品比较情况均未知，当前无法判断商机优先级与成单概率。
4. **单一 Champion 风险**：仅售后负责人积极，若其推动力不足或角色变动，项目易停滞。
5. **范围蔓延风险**：知识库 + Agent 两类能力同时上，若不分阶段，交付周期与预期容易失控。

## 五、下一步动作（建议 2 周内推进）
1. **48 小时内**：发送会议纪要 + 痛点确认清单，请客户书面确认，巩固 Champion。
2. **1 周内**：
   - 向信息化负责人单独输出《集成与安全方案说明》（接口方式、部署形态、权限模型、数据流向）。
   - 向售后负责人提供《售后知识沉淀场景 POC 建议书》，建议小范围试点（如 1 个产品线 / 1 个团队）。
3. **2 周内**：
   - 安排第二次会议，目标获取：预算区间、决策流程与时间表、是否在比较其他方案。
   - 邀请生产计划经理参加，探查二期扩展可能。
4. 准备同行业（离散制造 + 售后知识库）参考案例素材，用于建立信任。

## 六、需要交接的事项
- **售前 / 解决方案团队**：
  - 老系统集成可行性评估（需客户提供系统清单与接口现状）。
  - 权限隔离与数据安全方案（部署形态选型：私有化 / 专有云 / SaaS）。
- **产品团队**：
  - 售后场景 POC 范围与验收指标定义。
  - 知识沉淀工作流（微信群 / Excel / 工单系统）的导入方案确认。
- **商务 / 销售管理**：
  - 商机阶段登记、预算与决策时间表的二次摸排计划。
- **法务 / 安全**：
  - 提前准备数据处理协议、安全白皮书模板。

## 不确定性声明
- 预算、决策时间表、竞品比较情况未知，以上动作优先级基于"售后场景为高确信切入点"假设，需在下一次会议中验证。
- 生产计划经理的角色与影响力仅基于其参会事实推断，未经证实。
