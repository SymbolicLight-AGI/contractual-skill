# 机会判断：sales-growth-002

## 客户画像
- 类型：某区域设备经销商
- 接触阶段：早期兴趣表达，未进入需求澄清
- 已知联系内容：对 AI Agent 表示"看起来不错"，要求下周给方案

## 已知事实（客户明确说过的）
- 客户对 AI Agent 表达了兴趣。
- 客户要求下周提供一个方案。
- 客户曾询问"能不能一周上线"。

## 推断的潜在机会（未经客户确认）
- 客户可能存在某种希望快速落地的运营或业务痛点（来源：一周上线的提问）。
- 作为区域设备经销商，可能的场景方向包括：销售线索跟进、客户咨询应答、订单/库存查询、售后服务等，但均未被客户证实。

> 以上为推断方向，不应作为客户已确认的需求陈述。

## 风险信号
- **快速上线预期**："一周上线"的提问属于高风险信号，可能意味着对实施复杂度认知不足。
- **场景空白**：未说明具体业务场景，无法评估可行性与价值。
- **关键信息全部缺失**：预算、决策人、上线时间正式承诺、数据来源、系统环境均未知。
- **无会议纪要**：缺乏可追溯的沟通记录，需求基础薄弱。
- **"给个方案"风险**：在场景未明确前直接出方案，容易形成不准确承诺或被用作比价材料。

## 谨慎判断
**当前不建议作为"重点推进"机会标记。** 该线索仅处于兴趣表达阶段，尚不构成确认的购买意向。建议定位为"需求待澄清线索"，先进行一轮结构化需求确认后再决定优先级。

不确定性说明：在缺乏场景、预算、决策链信息的情况下，无法判断真实商机价值与赢率。

## 待确认问题（建议下周对话前先获取）
1. 业务场景：希望 AI Agent 解决哪一类具体业务问题？面向内部员工还是外部客户？
2. 现状痛点：目前该流程由谁/用什么方式在做，主要瓶颈是什么？
3. 数据与系统：涉及哪些数据源、业务系统？是否已具备接入条件？
4. 上线预期："一周上线"是硬性要求还是初步想法？可接受的合理周期是多久？
5. 决策链：谁是业务负责人、谁是最终决策人、是否有 IT/采购参与？
6. 预算范围：是否有预算立项？属于年度预算还是试点性投入？
7. 评估方式：是否在对比其他方案？方案用途是内部汇报还是采购评审？
8. 成功标准：上线后用什么指标衡量"有效"？

## 下一步建议
1. **不直接出正式方案**。在下周交付前，先安排一次 20–30 分钟的需求澄清会，覆盖上述问题。
2. 若客户拒绝澄清会、坚持只要"一份方案"，则交付**通用能力介绍 + 场景示例**，不写入具体价格、实施周期、集成范围、效果承诺。
3. 在内部 CRM 中将该机会标记为 **早期阶段 / 信息不足**，暂不计入重点推进 pipeline。
4. 准备一个轻量的场景探询清单（设备经销行业常见场景列表），用于引导客户具体化需求。

## 需要的交接
- **预压方案内容** → 交由 **售前** 评估范围与可行性，不由销售独立承诺。
- **"一周上线"诉求** → 交由 **售前/交付** 做实施周期评估，销售侧不得确认时间。
- **报价问题** → 在场景明确前不报价；后续由 **报价/商务** 流程处理。
- **集成与数据接入可行性** → 交由 **售前/技术** 评估。
- 若客户后续提出 SLA、效果保证、固定交付范围 → 进入 **风险评审**。

## 合规检查
- 未承诺价格、折扣、付款条款、交付时间、实施范围或效果。
- 未将兴趣表达写成购买意向。
- 已列出缺失信息并以问题形式呈现，未用假设填补。
- 已标注需要交接的事项。
- 摘要为内部团队视角，未暴露个人联系方式（材料中亦未提供）。
