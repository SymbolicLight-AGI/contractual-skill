# 机会判断与下一步建议

## 一、谨慎判断

**结论：不建议在当前信息基础上将其列为"重点推进"机会，建议先定级为"待验证线索（Lead to Qualify）"。**

理由：
- 客户仅表达了**初步兴趣**（"看起来不错"），未构成明确购买意向。
- 关键资格要素（BANT / MEDDIC 维度）几乎全部缺失：业务场景、预算、决策人、上线时间、数据来源、系统环境均未知。
- "下周给我一个方案"是**模糊请求**，在没有需求边界的情况下输出方案，存在两类风险：
  1. 方案脱靶，浪费售前资源；
  2. 用错误锚点（如"一周上线"）形成不切实际的预期。
- "能不能一周上线"的提问需警惕：可能反映客户对 AI Agent 实施复杂度认知不足，也可能是临时性应急需求，需要进一步澄清，不宜直接当作商机紧迫性信号。

不确定性标注：以上判断基于材料中"未说明 / 没有纪要"的描述；若销售侧另有未记录的口头信息，结论可能调整。

## 二、待确认问题清单（建议在出方案前澄清）

**1. 业务场景与价值**
- 希望 AI Agent 解决哪类具体问题？（如：经销商内部询价、售后工单、库存查询、客户线索跟进？）
- 当前该业务由谁/什么系统承担？痛点是什么？
- 期望的成功标准是什么？（效率提升、人力替代、响应速度？）

**2. 决策与商务**
- 本次由谁发起？最终拍板人是谁？是否涉及总部？
- 是否已有预算范围或采购流程预期？
- 期望的采购形式：一次性项目、订阅、POC 试点？

**3. 时间与交付**
- "一周上线"是针对哪部分功能？是 demo、POC 还是正式投产？
- 实际期望的上线节点和业务驱动事件是什么？

**4. 数据与系统**
- 数据来源：ERP / CRM / 微信 / Excel / 设备数据？
- 是否允许数据出域？部署形态偏好（SaaS / 私有化）？
- 现有 IT 能力与对接资源？

## 三、下一步建议（按顺序）

1. **不要直接出方案**。先用 1 封简短邮件或 1 通 15–30 分钟的需求澄清电话回应客户，明确："为了下周给到匹配的方案，需要先对齐几个关键点"。
2. **附上 3–5 个最关键的问题**（建议从上方清单中选：场景、决策人、时间含义、数据来源），降低客户回复门槛。
3. **预约一次正式需求会议**，要求决策人或业务负责人到场，并产出会议纪要（弥补当前无纪要的盲区）。
4. **若客户拒绝澄清、坚持"先看方案"**：提供**通用能力介绍 + 同行业参考案例**，而非定制方案，避免售前资源前置投入。
5. **内部定级**：在 CRM 中标记为 "Qualifying"，设置 1–2 周的资格确认窗口；窗口内若关键信息仍缺失，则降级或暂缓。
6. **针对"一周上线"预期**：在下一次沟通中主动管理预期，区分 "demo 可一周" 与 "生产可用需更长"，避免后续交付冲突。

## 四、提示

当前材料不足以支撑商机金额、赢率、优先级的量化判断；以上结论与建议会随澄清结果显著变化。
