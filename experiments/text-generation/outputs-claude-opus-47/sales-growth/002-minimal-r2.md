## 风险信号 ⚠️
1. **"一周上线"询问** —— 交付时间预期不合理，属于交付时间承诺类风险，需移交售前与风险评审。
2. **"下周给方案"** —— 在零需求输入下被要求出方案，存在方案范围与定价被客户单方面框定的风险。
3. **无会议纪要、无书面需求** —— 信息可追溯性差，后续易出现范围/承诺争议。
4. **未识别决策人** —— 方案可能交付给非决策角色，资源投入回报不确定。
5. 隐含风险：若按"通用 AI Agent 方案 + 低价 + 快速上线"组合推进，将触发**低价固定范围 + 快速上线 + 全场景对接**多重风险标记。

## 判断结论
**不建议作为重点机会推进。** 当前属于需进一步资格确认的早期线索。在场景、预算、决策人、时间表四项中至少确认两项之前，不应投入售前方案资源，**也不得对价格、折扣、付款条件、交付时间、实施范围做任何承诺**。

## 待确认问题（建议下次沟通清单）
1. 业务场景：希望 AI Agent 解决哪个具体环节的问题？目前如何处理？
2. 成功标准：上线后期望看到的可量化结果是什么？
3. 决策链：本次评估由谁主导？最终拍板人是谁？是否涉及总部？
4. 预算区间：本年度是否已立项？预算量级？
5. 时间预期："一周上线"是硬性要求还是初步想法？背后有什么时间压力？
6. 数据与系统：现有 ERP/CRM/进销存系统是什么？数据是否可对接？
7. 历史尝试：之前是否评估过类似产品？为什么没有继续？

## 下一步建议
1. **不出方案**，改为申请一次 30–45 分钟的需求澄清会，议程聚焦上述 7 个问题。
2. 会前发送简短议程邮件并请客户邀请相关业务/IT 干系人参与（验证决策链）。
3. 沟通中明确表达：方案与报价需要在场景与环境确认后由售前共同输出，**避免口头承诺时间或价格**。
4. 会后形成书面会议纪要并请客户确认，作为后续判断是否升级为重点机会的依据。
5. 将"一周上线""下周给方案"两项标注为风险，提交售前与风险评审备案。
6. 资格确认完成前，该机会在管线中保持为"线索/待资格确认"状态，不占用重点资源。
