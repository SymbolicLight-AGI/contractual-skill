# 销售跟进建议 — Task sales-growth-001

## 客户画像

- 行业与定位：华东某离散制造企业（具体规模、产品线未提供）。
- 评估方向：企业知识库 + 售后服务 Agent。
- 关键参与者：信息化负责人、售后负责人、生产计划经理。
- 立场倾向（推断）：
  - 售后负责人：业务推动方，态度积极。
  - 信息化负责人：把关方，关注集成与安全风险。
  - 生产计划经理：参会角色未明确表态，影响范围待确认。

## 已知事实

- 客户售后知识分散在微信群、Excel、老系统中，存在重复问答。
- 新员工培训周期长，客户希望沉淀标准问答和工单经验。
- 客户对数据安全和权限隔离敏感。
- 售后负责人态度积极。
- 信息化负责人关注系统集成风险。
- 客户**未**披露：预算、决策时间表、是否在比较竞品。

## 推断与不确定性

- 推断：售后场景是当前最有可能的首期落地点（依据：售后负责人积极 + 痛点集中在售后知识与工单）。
- 推断：生产计划经理参会，可能预示未来向生产侧（计划、异常处理）扩展，但当前未表达需求。
- 不确定性：
  - 预算区间未知。
  - 决策流程与时间窗口未知。
  - 是否已接触竞品、处于评估哪个阶段未知。
  - 数据源现状（老系统是哪些、是否有 API、微信群/Excel 数据量级与质量）未知。
  - 权限模型颗粒度要求（按角色 / 按工厂 / 按客户）未知。
  - 是否要求私有化部署或本地化大模型未知。

## 痛点与机会

- 痛点：
  - 知识分散、重复问答消耗售后人力。
  - 新人上手慢，培训成本高。
  - 安全与权限隔离合规压力。
- 机会信号：
  - 痛点清晰且业务方主动。
  - 售后是边界相对可控的首期场景，便于做小步快跑试点。
  - 知识沉淀诉求与 Agent + 知识库产品定位高度吻合。
  - 生产计划经理在场，存在二期向生产侧扩展的潜在空间（待验证）。

## 风险信号

- 集成风险：信息化负责人明确关注，老系统接入方式、接口可用性未评估。
- 安全合规风险：数据安全与权限隔离敏感，可能涉及部署模式（私有化 / 专有云 / 数据不出域）。
- 范围蔓延风险：若不锁定首期边界，易被要求"全系统打通 + 全量知识接入"，存在低价固定范围或无限定制风险。
- 决策不确定风险：预算、时间表、竞品情况均未知，不能据此承诺时间或价格。
- 数据质量风险：微信群/Excel 沉淀数据噪声大，需评估清洗与标注成本，不应承诺"开箱即用"。

## 推荐下一步

1. 由销售牵头，安排一次需求澄清会，目标补齐：预算量级、决策流程与时间表、是否在评估竞品、部署偏好（云 / 私有化）。
2. 邀请**预售/解决方案**与信息化负责人单独对齐：老系统清单、接口现状、权限模型颗粒度、数据出域要求。
3. 与售后负责人共同梳理首期试点边界（建议：聚焦售后高频问答 + 工单沉淀，先不承诺打通生产侧）。
4. 提供产品能力与同行业（脱敏）参考说明，不涉及报价、不承诺上线时间。
5. 内部评估：是否需要做小范围 POC，POC 范围、数据样本、评估指标在内部对齐后再与客户沟通。
6. 在所有客户面材料中，避免出现最终价格、折扣、交付时间、集成保证、效果保证字样。

## 交接建议

- 预售 / 解决方案：评估老系统集成可行性、知识库数据接入路径、权限模型方案、部署模式建议。
- 风险评审：客户对"安全与权限"敏感 + 信息化方关注集成，需评估是否存在全系统集成承诺、无限范围或数据合规风险。
- 交付 PM：在 POC 或一期范围确定后，介入评估时间表、数据准备、验收标准。
- 财务 / 合同：在客户披露预算并完成方案对齐后再介入；目前**不**提供报价或折扣口径。
- 人工确认门槛：任何涉及最终价格、折扣、合同范围、上线时间、系统集成保证、业务结果保证的表述，须经上级与对应职能确认后才能对客户表达。
