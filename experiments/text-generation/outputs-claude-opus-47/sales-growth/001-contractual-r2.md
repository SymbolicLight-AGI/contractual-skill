# 销售跟进建议 — Task sales-growth-001

## 客户画像

- 行业与区域：华东地区离散制造企业。
- 评估方向：企业知识库 + 售后服务 Agent。
- 参会角色覆盖：信息化负责人、售后负责人、生产计划经理（IT + 业务双线参与）。
- 客户成熟度信号：已意识到知识分散问题，并主动评估解决方案，属于"问题已显性化"阶段。
- 内部立场分布（观察）：售后负责人=潜在拥护者；信息化负责人=谨慎方/把关人；生产计划经理=立场未明。

## 已知事实

- 售后知识分散在微信群、Excel、老系统中，存在重复问答。
- 新员工培训周期长，客户希望沉淀标准问答与工单经验。
- 客户对数据安全和权限隔离敏感。
- 参会方包含信息化负责人、售后负责人、生产计划经理。
- 售后负责人在会上态度积极。
- 信息化负责人关注系统集成风险。

## 推断与不确定性

- 推断：售后场景可能是首个落地切入点，因售后负责人积极且痛点最具体。（推断，非客户明确表态）
- 推断：生产计划经理在场，未来可能延伸到生产排程/异常处理知识场景，但当前未表态。（推断）
- 推断：客户对权限隔离敏感，可能涉及多工厂/多部门数据边界，但具体模型未知。（推断）
- 不确定性 — 缺失信息（不可臆造）：
  - 预算：未知。
  - 决策时间表/采购流程：未知。
  - 是否已接触竞品、评估阶段位置：未知。
  - 老系统类型、接口能力、数据导出可行性：未知。
  - 微信群/Excel 数据规模、质量、敏感等级：未知。
  - 最终决策人与签批链：未知（信息化负责人是否为决策人未确认）。
  - 部署形态偏好（私有化 / 专有云 / SaaS）：未知。

## 痛点与机会

- 痛点1：知识分散在 IM、表格、老系统 → 重复问答消耗售后人力。
- 痛点2：新人培训周期长 → 服务一致性与人员爬坡成本高。
- 痛点3：知识沉淀缺乏结构化机制。
- 机会1：售后 Agent + 标准问答库 可作为最小可验证场景（MVP）。
- 机会2：售后负责人是潜在 Champion，可借其推动内部对齐。
- 机会3：若首期落地良好，存在向培训、生产异常处理、工单分析等场景扩展的二次机会。（推断）

## 风险信号

- 集成风险：客户已有"老系统"，信息化负责人关注集成 → 若承诺"全系统打通"或"快速对接"将构成 Risk signal。
- 数据安全/权限风险：客户敏感 → 任何关于"权限可全自定义""绝对隔离""零泄漏"的表述均需风控审视。
- 范围蔓延风险：知识来源涉及微信群、Excel、老系统，若一次性纳入容易演变为无限定制。
- 决策链不清风险：未确认决策人，可能出现"售后推动、信息化否决"的拉锯。
- 竞争态势不明风险：是否在比稿、是否已有偏好供应商未知。
- 商务信息缺失风险：预算与时间表未知，过早投入大型 POC 成本不可控。

## 推荐下一步

以下均为内部建议，不构成对客户的承诺：

1. 由销售在 3–5 个工作日内发送会议纪要确认函，向客户复核痛点与诉求，并请客户补充：预算区间、期望上线节奏、决策流程。（信息补全）
2. 安排一次 30 分钟的售后负责人 1:1，巩固 Champion 关系，了解其内部推动路径。
3. 与信息化负责人单独沟通，了解老系统类型、接口情况、数据安全合规要求（是否等保/行业规范）。
4. 内部召集预售评估会，初步判断售后 Agent MVP 范围（仅聚焦售后场景，不承诺全域知识打通）。
5. 准备一份"分阶段落地思路"草案（阶段一：售后标准问答；阶段二：工单经验沉淀；后续：扩展场景）——仅作为讨论框架，不含价格、不含上线时间、不含集成承诺。
6. 暂不报价、不承诺折扣、不承诺上线时间、不承诺集成范围，直至预算、决策链、集成边界三项信息补全。

## 交接建议

- 预售（Presales）：评估售后 Agent MVP 的方案范围、与老系统集成可行性、权限模型与数据隔离技术方案、知识接入（微信群/Excel/老系统）的技术假设。
- 风控审查（Risk Review）：客户对数据安全与权限隔离敏感，任何涉及"隔离保证""数据不出域""集成无风险"的对外表述须经风控确认；同时审视范围蔓延与无限定制风险。
- 交付 PM（Delivery PM）：在客户提供上线期望后，评估阶段化落地路径、数据准备要求、验收标准草案。
- 财务/合同（Finance Contract）：在客户提供预算与采购流程后，再介入定价与合同条款；当前阶段不出价。
- 人工确认（Human Gate）：任何面向客户的最终价格、折扣、上线时间、集成范围、效果保证均须销售负责人 + 相关职能确认后方可发出。
