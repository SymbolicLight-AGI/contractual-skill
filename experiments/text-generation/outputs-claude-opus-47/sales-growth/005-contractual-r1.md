# Sales Growth 跟进与交接建议

## 客户画像

- 行业：装备制造集团（多层级组织：集团 / 分公司 / 服务商）。
- 业务关注：售后服务智能化（知识库、工单分类）与系统集成。
- 关注角色：管理层（关注试点 ROI）。
- 决策结构：未明确，存在跨组织数据权限协调需求。

## 已知事实

- 客户提出四项需求：售后知识库、工单自动分类、与 CRM 和备件系统打通、试点 ROI 展示。
- 客户要求"先把所有系统都接上，再看效果"。
- 客户希望报价中包含后续所有二开。
- 数据权限涉及集团、分公司、服务商三方。

## 推断与不确定性

- 推断：客户对售后效率有较强提升意愿，可能存在工单积压或知识沉淀不足问题（未明确）。
- 推断：管理层关注 ROI，表明决策层已介入，但具体决策人、预算、立项阶段均未提供。
- 不确定性：
  - 预算规模与审批流程：未知。
  - 决策人 / 决策委员会构成：未知。
  - 期望试点与上线时间：未知。
  - 现有 CRM、备件系统的版本、接口能力、数据治理现状：未知。
  - 三方数据权限的现行制度与合规要求：未知。
  - "所有二开"的范围边界：未知。

## 痛点与机会

- 痛点：售后知识分散、工单分类依赖人工、跨系统数据割裂、管理层缺少量化效果视图。
- 机会：
  - 售后知识库 + 工单分类可作为有边界的试点切入。
  - ROI 诉求可推动以"小范围试点 → 度量 → 扩展"的路径达成共识。
  - 跨系统集成是中长期扩展空间。

## 风险信号

- **范围失控风险**：客户要求"先全部打通再看效果"，与试点 ROI 诉求自相矛盾，可能导致投入大、验证晚、效果难归因。
- **商务风险**：报价包含"后续所有二开" = 无限范围 + 固定价格，属高风险条款。
- **数据合规与权限风险**：集团 / 分公司 / 服务商三方数据权限边界不清，涉及授权、隔离、审计。
- **集成可行性风险**：CRM 与备件系统的接口能力未评估，"全部打通"在技术与时间上均不可承诺。
- **ROI 度量风险**：缺乏基线数据，ROI 口径若未提前对齐，将难以验收。

## 推荐下一步

销售可继续推进的事项：

1. 与客户对齐试点目标与口径（知识库 + 工单分类作为第一阶段候选范围）。
2. 收集缺失信息：预算区间、决策人 / 立项阶段、期望时间、现有系统版本与接口资料、三方数据权限制度。
3. 推动客户提供 CRM、备件系统的技术资料与样例数据需求清单（不承诺集成范围）。
4. 安排一次联合会议，邀请售前方案与客户业务 / IT / 合规对接人参加。

销售**不得**自行承诺的事项（需上游确认）：

- 集成范围与可行性。
- 二次开发是否纳入报价、纳入边界。
- 试点与上线时间。
- 任何 ROI 数值或效果保证。
- 三方数据权限方案。

## 交接建议

- **售前方案（Presales）**：
  - 评估与 CRM、备件系统集成的技术可行性与假设条件。
  - 设计"试点先行、分阶段扩展"的方案，拆解"全系统打通"诉求。
  - 输出知识库 + 工单分类试点的功能边界与依赖项。

- **风险审查（Risk Review）**：
  - 审查"报价包含后续所有二开"条款 —— 属无限范围固定价，需明确拒绝或以变更管理机制替代。
  - 审查"先全部打通再看效果"对交付与验收的风险。
  - 审查集团 / 分公司 / 服务商三方数据权限带来的合规与责任划分问题。

- **交付经理（Delivery PM）**：
  - 设计试点里程碑、验收标准、ROI 度量口径与基线数据采集方式。
  - 评估数据就绪度与三方协作机制。

- **财务合同（Finance Contract）**：
  - 待方案与范围确定后介入定价、付款节奏、二开计费模型（建议按变更或工时计费，避免无限范围）。
