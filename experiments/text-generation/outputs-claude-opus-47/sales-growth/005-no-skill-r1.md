# 客户跟进交接建议 — 某装备制造集团

## 一、销售可继续推进的事项

| 事项 | 动作 |
|---|---|
| 关系维护 | 继续维系决策层与业务对接人，明确管理层 ROI 关注点的具体口径（指标、周期、考核人）。 |
| 需求确认 | 与客户确认售后知识库、工单分类的业务范围、用户数、工单量级。 |
| 试点共识 | 推动客户接受"先试点、后扩展"的合作节奏，铺垫与"全量接入"诉求相反的预期。 |
| 商务节奏 | 同步预算区间、采购流程、决策链与签约时间窗。 |
| 竞争情报 | 了解是否有在选型的其他供应商、过往同类项目经验。 |

---

## 二、必须交售前方案确认的事项

1. **集成可行性评估**
   - 现有 CRM 厂商、版本、接口开放程度。
   - 备件系统的数据结构、接口类型、调用限制。
2. **工单自动分类方案**
   - 分类体系来源（客户已有 / 共建 / 模型推断）。
   - 模型冷启动数据量与标注责任方。
3. **售后知识库**
   - 知识来源、更新机制、权限模型。
4. **试点 ROI 度量设计**
   - 试点范围、基线指标、衡量周期、汇报形式。
   - 提出"分阶段交付 + 阶段验收"建议，应对客户"先全接再看效果"的诉求。

---

## 三、必须交风险审查确认的事项

1. **报价范围风险**
   - 客户要求"报价含后续所有二开" → 属开放式范围承诺，需明确：
     - 二开的定义、计费方式（包干 / 工时池 / 按需报价）。
     - 合同中需写入变更控制条款。
2. **数据权限三方结构**
   - 集团、分公司、服务商三方的数据可见性、隔离要求、责任边界。
   - 是否涉及个人信息、设备运行数据出境/跨主体共享的合规问题。
   - 需法务/合规审阅数据处理协议（DPA）模板适用性。
3. **范围蔓延风险**
   - "先全接再看效果"模式下的实施成本与回款风险，需评估是否设最低分阶段付款节点。

---

## 四、必须交交付经理确认的事项

1. **实施路径**
   - 在客户坚持"全量接入"诉求下，交付侧是否具备并行集成能力与资源。
   - 建议的 MVP 范围与里程碑。
2. **集成排期与依赖**
   - CRM、备件系统接口对接所需的客户侧配合资源（接口人、测试环境、数据样本）。
3. **试点验收标准**
   - 与售前方案对齐 ROI 指标后，确认交付侧可量化、可验收的口径。
4. **三方权限落地**
   - 权限模型在产品/实施层面的可实现性，是否需要定制开发。

---

## 五、不确定性标注

- 客户预算、决策链、采购时间表、CRM 与备件系统的具体厂商均未在材料中提供，以上建议基于通用 B2B 复杂集成项目经验，落地前需销售补充事实信息。
- "后续所有二开"是否为客户初步表态或硬性条款，未明确，需销售在下一次沟通中澄清。
