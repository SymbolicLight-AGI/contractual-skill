# 客户跟进交接建议——某装备制造集团

## 一、下一步主跟进人

**售前方案经理（主），销售（辅）**，并同步引入 **风险审查** 与 **交付经理** 进行专项确认。

原因：客户需求已超出标准产品范畴（多系统集成 + 三方数据权限 + 二开范围未封口 + ROI 论证），销售单独推进会产生范围与商务风险。

---

## 二、销售可继续推进的事项

1. **客户关系与决策链梳理**
   - 确认管理层 ROI 汇报对象、决策人、预算归属（集团 / 分公司）。
   - 维护日常沟通节奏，安排售前方案经理拜访。
2. **基础需求与商务信息收集**
   - 当前售后工单量级、坐席数、知识库现状、CRM/备件系统厂商与版本。
   - 项目预算区间、采购流程、签约主体。
3. **标准产品价值传递**
   - 售后知识库、工单自动分类的标准功能演示与成功案例。
4. **试点方案的销售引导**
   - 引导客户接受"先试点—再扩展"的路径，弱化"全系统一次性打通"的预期（最终方案由售前定）。

---

## 三、必须交给 **售前方案经理** 确认的事项

1. **集成方案可行性**
   - CRM、备件系统的接口类型、字段映射、调用频率、是否需要中间件。
2. **工单自动分类的落地路径**
   - 训练数据来源、分类体系、冷启动策略、准确率验收口径。
3. **试点 ROI 模型设计**
   - 试点范围（建议：1 个分公司 / 1 类产品线）、量化指标（响应时长、一次解决率、坐席工时节省）、基线数据采集方式。
4. **分阶段实施蓝图**
   - 将"全系统打通"拆解为：试点 → 扩展集成 → 全量上线，并形成客户可接受的叙事。

---

## 四、必须交给 **风险审查** 确认的事项

1. **商务范围风险**
   - 客户要求"报价包含后续所有二开"——需明确二开口径、工时上限、超出部分计费规则，**严禁在报价中承诺无限二开**。
2. **验收风险**
   - 客户"先全部接上再看效果"的预期，可能导致试点无法验收、回款延迟，需要在合同中锁定分阶段验收节点。
3. **数据合规与权限风险**
   - 集团 / 分公司 / 服务商三方数据权限边界：
     - 数据所有权归属
     - 跨主体数据访问授权文件
     - 服务商作为外部方的数据隔离与脱敏要求
   - 是否涉及个人信息、是否需要签署 DPA。
4. **合同主体风险**
   - 签约方是集团还是分公司？付款方与受益方是否一致？

---

## 五、必须交给 **交付经理** 确认的事项

1. **集成工作量与排期评估**
   - CRM + 备件系统对接的人天估算、客户侧配合资源、联调窗口。
2. **试点交付边界**
   - 试点期内的功能清单、不做项清单（避免范围蔓延）。
3. **三方协作机制**
   - 集团 IT、分公司业务、外部服务商的对接人、决策机制、变更流程。
4. **二开承接能力**
   - 后续二开是否由交付团队承接、是否需要产品/研发介入、SLA 设定。
