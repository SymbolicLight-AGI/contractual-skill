# 销售跟进建议 — Task sales-growth-001

## 一、客户画像

- **行业 / 区域**：华东地区，离散制造业
- **评估方向**：企业知识库 + 售后服务 Agent
- **关键干系人**：
  - 信息化负责人（关注系统集成与数据安全，偏谨慎）
  - 售后负责人（业务痛点持有者，态度积极，潜在内部推动者 / Champion）
  - 生产计划经理（参会，立场未明，需进一步识别）
- **组织成熟度推断**：现有知识沉淀依赖微信群 / Excel / 老系统，数字化基础较弱，但已有明确改进意愿。

## 二、明确事实（客户已表达）

1. 售后知识分散在微信群、Excel、老系统中，存在大量重复问答。
2. 新员工培训周期长，希望沉淀标准问答与工单经验。
3. 对数据安全和权限隔离敏感。
4. 售后负责人态度积极。
5. 信息化负责人关注系统集成风险。

## 三、推断机会（待验证，非承诺）

- **首选切入场景**：售后服务 Agent + 售后知识库（业务方痛点最明确，ROI 故事最清晰）。
- **价值主张方向**（待商务/售前确认后落地）：
  - 减少重复问答工作量
  - 缩短新员工上手周期
  - 工单经验结构化沉淀
- **扩展潜力**：若售后场景跑通，存在向生产计划、全员知识库延展的二次机会（生产计划经理在场是信号，但需验证）。
- **Champion 候选**：售后负责人。

> 注：预算、决策时间表、竞品比较情况均未知，机会规模无法判断，标注为**信息缺口**。

## 四、风险信号

| 风险 | 说明 | 处置 |
|---|---|---|
| 系统集成风险 | 信息化负责人明确关注；老系统 + 微信 + Excel 混合，集成复杂度未知 | 交接售前做集成评估，**不承诺**对接范围 |
| 数据安全与权限隔离 | 客户敏感点，涉及合规与架构 | 交接安全/架构团队，**不承诺**具体方案细节 |
| 预算未知 | 无法判断项目体量 | 下次拜访挖掘 |
| 决策路径不清 | 决策人、决策时间表、采购流程均未确认 | 下次拜访绘制决策地图 |
| 竞品情况不明 | 是否在比价、对标谁未知 | 侧面了解 |
| 生产计划经理立场不明 | 参会但未表达，可能是观察者也可能是隐性决策影响人 | 单独沟通确认 |
| **需主动规避的销售陷阱** | 客户未来可能提出"全系统打通 / 快速上线 / 打包固定价 / 效果保证"等诉求 | 一律不当场承诺，转入售前与风险评审 |

## 五、下一步动作

**短期（本周内）**
1. 向售后负责人发送会议纪要，确认理解一致，巩固 Champion 关系。
2. 提供售后知识库 / 服务 Agent 的**通用能力介绍材料**（不含价格、不含交付承诺）。
3. 与信息化负责人单独沟通，了解：
   - 现有系统清单与对接偏好
   - 数据安全与权限的具体要求（本地化？分级？审计？）
4. 约访生产计划经理，识别其诉求与立场。

**中期（2 周内）**
5. 推动一次售前主导的**需求澄清会**，目标产出：场景边界、集成范围、安全要求清单。
6. 引导客户披露：预算区间、期望时间点、是否在评估其他厂商。
7. 评估是否设计**小范围 PoC** 作为低风险切入（具体范围与条件由售前定义，不在销售侧承诺）。

| **风险评审 / 商务** | 定价策略、交付周期、商务条款，待信息补齐后启动 |
| **客户成功（预沟通）** | 新员工培训场景落地路径，作为价值故事素材 |
