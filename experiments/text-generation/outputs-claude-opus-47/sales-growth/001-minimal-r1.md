# 销售跟进建议 — Task sales-growth-001

## 一、客户画像
- **行业与定位**：华东离散制造企业，处于售后服务与知识管理数字化探索阶段。
- **评估对象**：企业知识库 + 售后服务 Agent。
- **关键干系人**：
  - 信息化负责人（关注集成与安全，倾向谨慎）
  - 售后负责人（业务痛点直接相关，态度积极，潜在 Champion）
  - 生产计划经理（边缘相关，作用待确认）
- **组织态度**：业务侧推动力较强，IT 侧持风险审视态度，典型"业务热-IT冷"格局。

## 二、明确事实（客户已表达）
1. 售后知识分散于微信群、Excel、老系统，存在重复问答。
2. 新员工培训周期长，期望沉淀标准问答与工单经验。
3. 对数据安全与权限隔离敏感。
4. 售后负责人态度积极。
5. 信息化负责人关注系统集成风险。

## 三、推断机会（含不确定性标注）
- **核心场景**：售后知识沉淀 + Agent 辅助问答 + 新员工赋能（高置信）。
- **潜在扩展**：工单经验结构化、与老系统对接形成统一入口（中置信，需信息化确认）。
- **Champion 候选**：售后负责人（高置信）。
- **决策结构**：信息化负责人可能为技术否决方（高置信）；最终决策人未知（不确定）。
- **生产计划经理参与原因**：不明确，可能为未来场景延伸（如生产异常知识联动），需澄清（不确定）。

## 四、风险信号
1. **集成范围不清**：老系统接入复杂度未知，存在"全系统集成"预期风险。
2. **数据安全与权限隔离**：是硬性门槛，需安全方案预先准备。
3. **客户未表达预算、时间表、竞品情况**：销售路径信息缺口大，存在被比价或陪跑风险。
4. **业务侧热情可能导致过度承诺**：警惕在范围、上线时间、效果上做出超出可交付的口头承诺。
5. **新员工培训周期缩短**属于"结果保证"类期望，须避免给出保证性承诺。

> 按 Skill 要求：不在本次跟进中承诺价格、折扣、付款条件、交付时间或实施范围。

## 五、下一步动作
1. **二次拜访（业务深挖）**：与售后负责人对齐高频问答类目、工单典型场景、知识更新流程，输出"价值场景清单"。
2. **IT 对齐会**：与信息化负责人单独沟通，明确：
   - 老系统类型与接口现状
   - 数据安全合规要求（本地化 / 权限模型 / 审计）
   - 可接受的集成边界
3. **澄清生产计划经理角色**：确认其诉求是否涉及生产侧知识联动场景。
4. **信息缺口补全**：通过 Champion 间接了解预算区间、决策流程、是否在评估竞品。
5. **准备价值材料**：知识沉淀与 Agent 在同类离散制造售后场景的能力说明（不含定价、不含交付承诺）。
6. **内部立项评审**：判断是否进入正式商机阶段。

## 六、需交接事项
- **交接给售前**：
  - 老系统集成范围与接口可行性评估
  - 知识库与 Agent 的方案蓝图（范围、架构、权限模型）
  - 安全与权限隔离方案设计
- **交接给风险/合规评审**：
  - 数据安全、权限隔离、数据驻留要求确认
  - 集成范围与上线节奏的可承诺边界
- **交接给方案/定价团队（后续阶段）**：
  - 在范围明确后再启动定价与交付时间评估
- **保留在销售侧**：
  - 关系推进、Champion 培育、决策链摸排、竞品情报收集
