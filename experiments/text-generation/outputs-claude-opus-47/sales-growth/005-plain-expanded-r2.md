# 客户跟进与交接建议 — 某装备制造集团

## 一、客户画像
- 行业：装备制造集团，多层级组织（集团 / 分公司 / 服务商）。
- 业务关注点：售后服务数字化（知识库、工单分类）、系统打通（CRM + 备件）、试点 ROI 可视化。
- 决策层：管理层关注 ROI，存在自上而下的推动力。
- 组织复杂度：三方数据权限交叉，治理难度较高。

## 二、已知事实（客户明确表达）
1. 需要售后知识库能力。
2. 需要工单自动分类能力。
3. 需要与现有 CRM 系统、备件系统打通。
4. 管理层要求看到试点阶段的 ROI。
5. 客户提出"先把所有系统都接上，再看效果"。
6. 客户希望报价中包含后续所有二开。
7. 数据权限涉及集团、分公司、服务商三方。

## 三、可推断的机会点（推断，非客户原话）
- 售后服务体系存在工单分发效率与知识沉淀问题，存在标准产品切入空间。
- 管理层关注 ROI，意味着存在以"小范围试点 + 数据回报"为路径的销售机会。
- 多系统打通需求暗示未来可能延展到服务运营、备件预测等场景。

> 以上为推断，需由客户或售前进一步确认。

## 四、风险信号
| 风险 | 说明 | 处置建议 |
|------|------|----------|
| 范围膨胀 | "先全部接上再看效果"违反试点逻辑，易导致项目失控 | 转售前方案重新定义试点边界 |
| 报价范围不可控 | 要求报价包含"后续所有二开"，无法量化 | 转风险审查 + 售前共同界定 |
| 数据权限复杂 | 集团/分公司/服务商三方权限，涉及合规与架构 | 转售前方案 + 交付经理评估 |
| ROI 口径未定义 | 管理层 ROI 指标尚未明确 | 销售牵头与客户对齐口径 |

## 五、销售可继续推进的事项
1. 与客户对齐**试点目标与 ROI 衡量口径**（例如：工单分类准确率、平均处理时长、知识库调用率等指标方向，需客户确认）。
2. 摸排客户**决策链与关键干系人**（集团 IT、售后业务负责人、分公司代表、管理层 Sponsor）。
3. 收集**现有 CRM、备件系统的基本信息**（厂商、版本、是否有开放接口），用于交付售前判断。
4. 推动客户提供**售后业务现状数据**（工单量、分类现状、服务商协作模式），为试点设计提供输入。
5. 推动客户认可"**先试点、再扩展**"的合作节奏（作为销售立场，但不承诺范围与价格）。

## 六、必须交接的事项

### → 售前方案团队
- 重新定义试点范围：哪些系统在试点期接入、哪些放在后续阶段。
- CRM 与备件系统的集成可行性评估。
- 售后知识库 + 工单自动分类的方案匹配与试点指标设计。
- 三方数据权限的架构方案（集团 / 分公司 / 服务商分层权限模型）。

### → 风险审查
- "报价包含后续所有二开"的商务条款风险评估，明确二开范围与计费模型。
- "全部系统先接上再看效果"带来的范围与交付风险。
- 三方数据权限涉及的合规、数据归属、责任边界问题。

### → 交付经理
- 试点阶段的交付节奏与里程碑设计。
- 多方协作（集团 / 分公司 / 服务商）的实施可行性与资源评估。
- 与客户既有系统对接的实施风险点初判。

- 工单量级、分类现状、知识库现有沉淀情况？
- 数据权限三方的归属与授权机制？
- 预算量级与采购周期？（未知）
