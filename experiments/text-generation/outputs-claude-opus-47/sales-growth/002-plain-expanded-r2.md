# 机会评估：区域设备经销商 - AI Agent

## 客户画像
- **类型**：区域设备经销商
- **接触阶段**：早期兴趣表达，未进入需求澄清
- **接触质量**：低（无会议纪要，无结构化沟通记录）

## 已知事实（客户明确说过）
- 客户表示 "AI Agent 看起来不错"
- 客户要求 "下周给一个方案"
- 客户曾询问 "能不能一周上线"

## 推断（非客户确认）
- 客户可能对快速见效有期待（基于"一周上线"的提问，但不能视为承诺需求）
- 客户可能处于自行调研阶段，尚未形成内部立项

> 注意：以上为推断，不能作为客户确认的购买意向。

## 风险信号
1. **信息严重不足**：无业务场景、无预算、无决策人、无系统环境、无数据来源。
2. **快速上线诉求**：客户曾问"一周上线"，属于交付期望风险，需由presales/交付评估，不可在销售侧承诺。
3. **方案请求过早**：在场景未澄清前给出方案，可能导致方案与需求错配，或被误认为已承诺范围与价格。
4. **无会议纪要**：沟通可追溯性差，后续容易出现理解偏差。

## 判断
**当前不建议作为"重点推进"机会对待**，应先归为"需澄清"阶段。客户仅表达了兴趣，未表现出确定的购买意向、预算或决策路径。在没有进一步信息前投入重资源（如定制方案、报价、POC 承诺）风险较高。

建议定位为：**值得跟进、但需先完成需求澄清的早期机会**。

## 待确认问题（下次沟通清单）
**业务场景**
- 希望 AI Agent 解决的具体业务问题是什么？（销售助理、客服、订单处理、设备售后…）
- 目标用户是内部员工还是终端客户？

**决策与组织**
- 谁是项目发起人？是否还有其他决策方？
- 公司内部是否已立项？

**预算与时间**
- 是否有预算范围或预算来源？
- "下周要方案"和"一周上线"是硬性要求还是初步设想？

**技术与数据**
- 现有业务系统是什么（ERP/CRM/进销存）？
- AI Agent 需要接入哪些数据？数据是否结构化、是否可用？

**成功标准**
- 客户如何衡量这件事做成了？

## 下一步建议
1. **不要在下周直接交付正式方案**。建议先安排一次 30 分钟的需求澄清会，覆盖上述问题；会后形成纪要并与客户确认。
2. 在澄清会前，可向客户发送一份简短的能力介绍 + 信息收集清单，引导客户提供场景信息，而非直接出方案。
3. 在销售侧不就**价格、上线时间（含"一周上线"）、实施范围、集成可行性、效果保证**做任何承诺。
4. 完成澄清后再决定是否升级为重点机会并启动presales介入。

## 需要交接的事项
- **Presales**：在场景澄清后，评估方案可行性与上线周期（特别是"一周上线"的可行性回应）。
- **风险/交付评审**：若客户坚持快速上线或要求范围承诺，需提前进入风险评审。
- **商务**：在客户提出报价请求前，不进入商务环节；后续报价由商务/presales统一出口。

## 不确定性标注
- 客户业务规模、当前数字化水平、对 AI Agent 的真实理解程度均未知。
- "下周给方案"是客户的硬性截止还是随口要求，无法从现有材料判断。
- 客户是否同时在接触其他供应商，未知。
