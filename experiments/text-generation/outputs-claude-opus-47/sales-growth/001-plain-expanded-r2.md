# 销售跟进建议 — Task sales-growth-001

## 一、客户画像
- **行业与定位**：华东某离散制造企业，处于售后服务与知识管理数字化的评估阶段。
- **关键干系人**：
  - 信息化负责人：关注系统集成风险与数据安全，偏谨慎。
  - 售后负责人：业务痛点最直接，态度积极，可作为内部推动者（Champion 候选）。
  - 生产计划经理：参会但立场未明，需进一步识别其诉求。
- **当前阶段**：需求探索 / 方案评估早期，尚未进入商务阶段。

## 二、明确事实（客户原话或会议确认）
1. 售后知识当前分散在微信群、Excel、老系统中，存在重复问答问题。
2. 新员工培训周期长，客户希望沉淀标准问答和工单经验。
3. 客户对数据安全与权限隔离敏感。
4. 参会人员包括信息化负责人、售后负责人、生产计划经理。

## 三、推断机会（基于观察，非客户明示）
- **企业知识库 Agent** 可对接微信群、Excel、老系统的知识资产，解决重复问答。
- **售后服务 Agent** 可结合工单经验沉淀，缩短新员工上手时间。
- 售后负责人可能成为内部推动者，有助于推进 POC。
- 生产计划经理参会，可能存在向生产侧知识管理延伸的二次机会（待验证）。

> 以上为推断，需在下一次沟通中向客户确认，不可作为已确认需求。

## 四、风险信号
- **预算未知**：无法判断项目成立性。
- **决策时间表未知**：无法评估推进节奏与资源投入。
- **是否在比稿未知**：竞争态势不明，存在陪标风险。
- **数据安全与权限隔离敏感**：合规与部署模式（私有化 / 专有云）可能成为门槛。
- **集成风险**：信息化负责人对老系统集成存疑，需presales评估可行性。
- **干系人结构不完整**：未见采购、财务、最终决策人（如 CIO / 总经理），决策链不清。

## 五、下一步动作建议
1. **48 小时内**发送会议纪要与感谢邮件，逐条复述客户痛点以建立信任；不报价、不承诺方案细节。
2. **二次沟通约访**，目标澄清以下信息（建议以提问清单形式提出）：
   - 项目预算范围与立项状态？
   - 期望的上线节奏与决策时间窗？
   - 是否同时在评估其他厂商或自研方案？
   - 数据安全的具体要求（是否要求私有化部署、等保级别等）？
   - 老系统的类型、接口情况、集成预期边界？
   - 生产计划经理本次参会的关注点？
3. **培养内部 Champion**：与售后负责人单独沟通，了解其 KPI 与推动动力，协助其在内部立项。
4. **安排 presales 介入**：针对集成与权限隔离做一次技术对齐会，但不在本阶段给出实施承诺。
5. **准备同行业脱敏案例**（仅作参考分享，不承诺同等结果），降低信息化负责人对风险的顾虑。

## 六、需要交接的事项
- **Presales**：老系统集成可行性评估、权限隔离与数据安全方案、知识库与售后 Agent 的技术对齐。
- **风险 / 合规审查**：数据安全要求（是否涉及等保、私有化部署、数据出域限制）。
- **方案 / 商务团队**：在客户预算与时间表明确后，再启动报价与 SOW 设计，本阶段不报价、不承诺折扣或交付周期。
- **管理层**：若客户后续提出快速上线或保证效果，需上报评估。

## 七、信息缺口（明确标注不确定性）
- 预算、决策时间表、竞品情况均未知。
- 生产计划经理的具体诉求未知。
- 客户当前知识资产规模、工单量级、用户并发数未知，影响方案体量判断。
- 客户是否有明确的项目编号或立项流程未确认。

> 注：本建议基于所提供拜访记录生成；客户未明确表达的内容均以"推断"或"待确认"标注，不作为承诺或既定事实。
