# 机会评估：sales-growth-002

## 一、谨慎判断

**结论：暂不建议列为"重点推进"机会，建议归类为"待验证线索（Lead，未达 Qualified Opportunity）"。**

依据：
1. 客户仅表达了**初步兴趣**（"看起来不错"），未形成任何明确的采购信号；"下周给我一个方案"属于礼貌性推进，不等同于购买意向。
2. BANT / MEDDIC 关键要素**全部缺失**：
   - **预算**：未知
   - **决策人 / 决策链**：未知
   - **业务场景与痛点**：未知
   - **时间线**：仅有一句"能不能一周上线"，性质不明（可能是真实诉求，也可能是随口试探，甚至是不切实际的预期信号——属于风险点而非确定性）
   - **数据来源 / 系统环境**：未知
3. 无会议纪要，信息可追溯性差，存在销售一方主观解读的风险。

**风险提示**：在场景、数据、系统均未澄清的前提下贸然产出方案，存在三类风险——方案脱靶、消耗售前资源、以及若客户对"一周上线"有刚性预期则可能在交付期望上提前埋雷。

## 二、下周方案交付前必须确认的问题清单

**A. 业务与场景**
1. 希望 AI Agent 解决哪个具体环节的问题？（售前咨询 / 订单处理 / 售后 / 库存调度 / 经销商内部协同？）
2. 当前该环节由谁/什么系统在做？主要痛点是什么？
3. 期望的成功标准是什么？（效率提升？人力替代？响应时长？）

**B. 数据与系统**
4. 涉及哪些数据源？（ERP / CRM / 进销存 / 微信生态 / 电话录音？）
5. 数据是否可对外接口/导出？是否涉及敏感信息合规？
6. 现有 IT 环境：本地化部署还是云端？是否有 IT 团队配合？

**C. 决策与商务**
7. 本次评估方案的决策人是谁？是否需要老板/股东最终拍板？
8. 预算区间或采购方式（试点 POC / 年度订阅 / 一次性买断）？
9. "一周上线"是真实截止时间，还是泛指"希望快"？背后是否有外部触发事件（如展会、季度目标）？

**D. 竞品与替代**
10. 是否在对比其他厂商或自研方案？

## 三、下一步建议

1. **不直接交付完整方案**。本周内主动约 30–45 分钟需求澄清会（线上即可），目标是回答上述 10 个问题中的至少 A、B、C 三组核心项。
2. **会议前发一份简短的"信息对齐清单"**（即上面的问题精简版），让客户内部先准备，提高会议效率，也借此测试客户的真实投入意愿——若客户不愿配合澄清，则该机会优先级应进一步下调。
3. **下周交付物降级为"场景假设 + 方向性建议 + POC 框架"**，而非详细实施方案；同时附一页"为出具精准方案所需的信息清单"，把球合理地传回客户侧。
4. **针对"一周上线"预期**：在澄清会中明确管理预期，区分"轻量 POC 可快速跑通"与"生产级落地周期"，避免后续交付冲突。
5. **CRM 中标记**：阶段 = Lead / 信息收集中；优先级 = 中（待澄清后重评）；不占用重点售前资源，直至 A、B、C 三组问题得到回答。
