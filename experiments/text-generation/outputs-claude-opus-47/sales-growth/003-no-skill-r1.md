原始话术存在以下高风险承诺，建议**不要直接发送**：

3. **范围承诺（生产异常知识库 + 质检问答 Agent）**：知识库的数据源、Agent 的能力边界、验收标准均未定义，易产生范围蔓延和验收争议。

## 三、需售前与风控确认的问题清单

### A. 范围与数据（售前主导）
1. **知识库数据源**：异常知识来自哪些载体（SOP 文档、历史工单、巡检记录、人工经验）？现有结构化/非结构化数据规模（条数、GB）？
2. **质检问答 Agent 的使用场景**：面向质检员、产线工人还是管理层？问答范围是 SOP 查询、异常诊断建议，还是包含决策类输出？
3. **数据更新频率**：知识库是一次性导入还是需要持续接入新数据？

### B. 系统对接（售前 + 客户 IT）
4. 是否需要对接 **MES**？接口形式（API/数据库/文件）？
5. 是否需要对接 **ERP**？涉及哪些模块（物料、批次、追溯）？
6. 是否需要对接 **质检系统 / LIMS**？
7. 是否需要接入 **生产设备实时数据**（PLC、IoT 网关、时序数据库）？若需要，将显著影响架构与周期。

### C. 部署与合规（售前 + 风控）
8. **部署方式**：公有云、私有云、客户机房本地化部署？是否要求纯内网、无外网调用大模型？
9. 若现场部署，是否需提供硬件？GPU 规格要求？
10. **数据合规**：是否涉及配方、客户、追溯数据等敏感信息？数据出域限制？

### D. 商务与验收（风控主导）
11. **预算口径**："10 万"是客户的预期还是已定预算？是否含硬件、年度服务费、模型 API 费用？
12. **验收标准**：知识库召回率、Agent 回答准确率的量化指标如何定义？由谁、以什么样本集验收？
13. **服务期与 SLA**：上线后的运维、知识更新、故障响应是否在本期合同内？
14. **付款节奏与违约条款**：分期比例、延期责任边界。
15. **"上线"的定义**：是 Demo 演示、试运行、还是通过正式验收？
