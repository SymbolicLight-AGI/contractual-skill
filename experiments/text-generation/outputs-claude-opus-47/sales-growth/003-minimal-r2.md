# 销售增长分析输出

## 一、客户画像
- 行业：食品加工企业
- 业务诉求：构建生产异常知识库 + 质检问答 Agent
- 关键不确定性：数据规模、系统对接范围（MES/ERP/质检）、实时数据采集、部署形态、合同范围与验收标准均未确认

## 二、痛点识别
- 生产异常知识沉淀分散，缺乏统一可查询载体
- 质检环节依赖人工经验，问答响应效率待提升
- （推测，待确认）跨系统数据未打通，影响知识闭环

## 三、成交路径
1. 当前阶段：早期需求沟通，范围与约束尚未澄清
2. 下一阶段：售前调研 → 范围界定 → 方案与商务建议 → 风控评审 → 报价与排期
3. 不应在本次会议给出价格、范围、上线时间的确定承诺

## 四、风险信号（高优先级）
⚠️ **用户原话存在三项高风险承诺**，均不可直接发给客户：
1. "10 万以内全包" —— 固定低价 + 范围未定，属于低价固定范围风险
2. "保证 2 周上线" —— 快速上线承诺，未评估数据/对接/部署复杂度
3. 隐含"全包" —— 全系统集成承诺，未确认 MES/ERP/质检/实时数据范围

按 Skill 要求：价格、范围、上线时间须移交售前与风控确认，不在销售侧直接承诺。

## 五、可发送给客户的安全替代表述

> 张总，您好。关于生产异常知识库与质检问答 Agent，我们已初步理解贵司的方向。为了给到您真正可落地、可验收的方案，我们建议本次会议先聚焦在场景与边界的对齐上：包括知识来源范围、是否需要与 MES/ERP/质检系统对接、是否涉及现场部署等。
>
> 我们会在调研后的 [X] 个工作日内，提供一份包含分阶段范围、里程碑与商务建议的正式方案。我们的目标是让首个可用版本尽快为您创造价值，同时确保交付质量与验收标准清晰，避免后续返工。具体的价格区间与上线节奏，待范围确认后由方案组统一给出书面建议。

（该表述不承诺金额、范围、上线时间，仅承诺流程与响应。）

## 六、需售前与风控确认的问题清单

**售前确认（范围与技术）**
1. 知识库的数据规模：文档数量、类型（SOP、异常记录、工艺单等）、更新频率？
2. 是否需要对接 MES / ERP / 质检系统？接口形式与数据权限？
3. 是否涉及产线设备实时数据采集？采样频率与数据量级？
4. 部署形态：公有云 / 私有云 / 现场内网部署？是否有信创/等保要求？
5. 用户规模与并发：质检 Agent 的使用人数、峰值并发？
6. 知识更新与运营：由客户自维护，还是需我方提供运营服务？

**风控确认（商务与交付）**
7. 合同范围如何切分（POC / MVP / 全量交付）？
8. 验收标准如何定义（功能清单 / 回答准确率 / SLA）？
9. 上线节奏是否可分阶段（如 POC 2–4 周 + 正式交付分期）？
10. 报价模型：一次性 + 订阅 / 按模块 / 按席位？
11. 数据安全与责任边界（食品行业合规、数据脱敏要求）？
12. 二次开发与变更管理机制？
