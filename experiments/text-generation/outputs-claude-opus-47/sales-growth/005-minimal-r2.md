# 任务结果：sales-growth-005

## 一、客户画像

- **客户类型**：装备制造集团（多层级组织：集团 / 分公司 / 服务商）
- **业务场景**：售后服务数字化
- **决策关注点**：管理层关注试点 ROI（说明存在管理层 Sponsor，但需用试点验证价值）
- **组织复杂度**：高（三方数据权限）

## 二、痛点梳理

1. 售后知识沉淀薄弱 → 需知识库
2. 工单分派效率低 → 需自动分类
3. 数据孤岛 → CRM、备件系统未打通
4. 价值不可见 → 管理层缺乏 ROI 视图

## 三、成交路径判断

建议路径：**试点（知识库 + 工单分类）→ 验证 ROI → 分阶段集成 CRM/备件 → 评估二开**
（客户当前倾向"全接入再看效果"，与试点路径冲突，需引导。）

## 四、风险信号（必须上报）

| 风险 | 类型 | 说明 |
|---|---|---|
| R1：要求先全系统接入再看效果 | 范围/交付风险 | 与试点 ROI 验证路径矛盾，交付周期与成本不可控 |
| R2：报价包含后续所有二开 | 商务/范围风险 | 二开需求未定义，无法做固定范围报价 |
| R3：集团-分公司-服务商三方数据权限 | 方案/合规风险 | 涉及权限模型、数据隔离、合规边界，需方案设计 |

## 五、下一步跟进分工

### ✅ 销售可继续推进的事项

1. 维护客户关系，确认管理层 Sponsor 与决策链
2. 收集业务现状：工单量级、分类口径、知识库现有形态、CRM/备件系统厂商与版本
3. 明确试点范围期望（部门 / 产品线 / 服务商范围）
4. 推动客户接受"试点验证 ROI"思路，弱化"全系统先接入"诉求
5. 安排售前与客户技术对口人对接会议
6. 输出商机基础信息、决策地图、采购流程与预算窗口

### ⚠️ 必须交给 **售前方案** 确认的事项

1. 知识库 + 工单自动分类的方案边界与技术选型
2. CRM、备件系统集成方式（接口、字段、频率）评估
3. 集团 / 分公司 / 服务商三方数据权限模型设计
4. 试点 ROI 衡量指标定义（如响应时长、一次解决率、人力节省）
5. 二开需求的识别、归类与边界划分

### ⚠️ 必须交给 **风险审查** 的事项

1. 客户"先全接入再看效果"诉求 → 范围与验收风险
2. "报价包含所有二开" → 不可量化范围的固定报价风险
3. 三方数据权限合规与责任归属
4. 任何价格、折扣、付款条件——**销售不做承诺**

### ⚠️ 必须交给 **交付经理** 确认的事项

1. 试点交付周期与里程碑（销售不可承诺上线时间）
2. 集成实施所需客户侧资源与配合度
3. 二开工时与资源排期机制
4. 试点到全量推广的交付路径
