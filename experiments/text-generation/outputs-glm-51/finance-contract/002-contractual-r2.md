# 结论摘要
不得直接支持“先做后付”。当前缺失客户信用、项目范围及成本核算等关键信息，直接承诺将导致严重的现金流与坏账风险。建议转向分阶段付款的替代方案，并在获得内部审批前不作任何对外承诺。

# 财务与合同风险
- [Risk] 现金流风险：在范围未定且成本不确定的情况下“先做后付”，我方需全额垫资（人员、差旅、第三方工具），严重影响现金流。
- [Risk] 坏账与应收款风险：客户信用未知，交付后若客户拒付、拖延或破产，我方可能面临全额损失。
- [Risk] 范围蔓延风险：无正式方案与合同约束，“先做”极易导致无限定制化或需求不断膨胀，最终无法界定验收与付款基准。
- [Risk] 法律与履约风险：合同未起草，无约束性条款保护我方收款权利及限制责任。

# 不得直接承诺事项
- 不得直接承诺同意“先做后付”或任何未经信用与审批核查的延迟付款条件。
- 不得批准任何非标准的付款里程碑或合同条款。
- 不得在无合同与范围界定的情况下，承诺启动实施工作或免费定制。

# 待确认问题
1. 客户的信用评级与财务状况如何？是否具备履约后付款的能力？
2. 项目的确切范围、交付物及验收标准是什么？
3. 我方开展此项目的总成本估算是多少？
4. 内部对“先做后付”或分阶段付款的审批权限归属谁？
5. 客户是否接受替代方案，如按里程碑分期付款（如：启动付30%，中期验收付40%，终验付30%）？

# 交接建议
- [Recommendation] Finance：评估客户信用，测算垫资成本与现金流影响，审批任何替代付款里程碑方案及折扣让步。
- [Recommendation] Legal：起草合同条款，确保收款权利、界定交付范围与验收标准，设置违约终止与索赔保护。
- [Recommendation] Delivery PM：界定正式项目范围与交付物，估算实施工时与差旅成本，明确验收标准。
- [Recommendation] Risk Review：针对“范围未定+延迟付款+信用未知”的组合风险进行专项审查，决定是否继续推进及推进前提条件。
