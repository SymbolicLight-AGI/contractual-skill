**安全问题**
- **SQL 注入风险（严重）**：`customer_id` 直接来自 HTTP query 参数，通过 f-string 直接拼接进 SQL 语句。攻击者可构造恶意输入（如 `1 OR 1=1` 或 `1; DROP TABLE notes; --`），从而越权获取全表数据或破坏数据库。

**正确性问题**
- **`SELECT *` 查询**：提取了所有列，可能包含不需要的敏感字段，增加网络传输和内存开销，且在表结构变动时易引发隐性错误。
- **无分页/限制机制**：使用 `fetchall()` 一次性提取所有匹配行。若某客户存在大量备注，可能导致内存溢出（OOM）或接口响应超时。
