- 核心需求：生产异常知识库、质检问答 Agent

- 客户为某食品加工企业。
- 客户明确希望构建“生产异常知识库”和“质检问答 Agent”。
- 客户即将开会，要求提供项目报价和上线周期承诺。

# 推断与不确定性
- **推断**：客户可能面临生产异常响应慢、质检知识检索难、依赖人工经验的痛点，希望通过 AI 提升效率。
- **不确定性/缺失信息**：
  - 数据规模未知（直接影响知识库构建周期与算力成本）。
  - 是否需要对接 MES、ERP 或质检系统未知（直接影响集成复杂度与工期）。
  - 是否涉及生产设备实时数据未知（影响 Agent 架构设计）。
  - 是否需要现场部署未知（影响基础设施准备与安全合规要求）。
  - 合同范围和验收标准未确认（“全包”定义模糊，极易导致需求蔓延）。

# 痛点与机会
- **痛点**：生产异常处理与质检依赖人工，效率与一致性可能存在瓶颈（推断）。
- **机会**：以知识库和 Agent 为切入点，建立食品加工行业智能化标杆，后续可拓展至更多生产环节。

# 风险信号
- **高风险承诺**：用户要求的“10 万以内全包”及“保证 2 周上线”属于未授权的价格、范围与交付时间承诺，严重违反风控原则。
- **技术风险**：若需对接 MES/ERP 等系统或处理实时设备数据，2 周内完成集成测试与上线几乎不可能。
- **交付风险**：“全包”在范围未定的情况下，极易陷入无限定制或低价格固定范围的亏损陷阱。
- **业务风险**：保证上线时间可能牺牲数据验证与验收质量，导致项目交付后无法达到预期业务结果。

**内部下一步：**
1. 暂停向客户发送任何包含绝对承诺的话术。
2. 在会前或会上优先澄清缺失信息（数据规模、系统对接、部署环境）。
3. 根据澄清后的信息，拉齐内部资源出具合理评估。

# 交接建议
- **售前**：确认系统集成（MES/ERP/质检系统）的技术可行性、架构假设与数据接入方案；评估数据规模对算力和构建周期的影响。
- **风控**：评估“10 万全包+2 周上线”的潜在风险，明确合同范围边界与验收标准，防止无限定制和低价格固定范围陷阱；审核任何涉及最终报价的对外沟通。
- **交付 PM**：评估 2 周上线的实际数据准备度与交付条件，明确分期交付或 MVP（最小可行产品）方案的验收标准。
