# 销售跟进建议

## 客户画像
- **行业/类型**：华东地区离散制造企业
- **当前关注**：企业知识库建设、售后服务 Agent 应用
- **关键干系人**：
  - 信息化负责人（关注系统集成与架构风险）
  - 售后负责人（业务驱动者，态度积极，关注效率与知识沉淀）
  - 生产计划经理（参会角色，具体业务诉求暂不明确）

## 已知事实
- [Known fact] 客户正在评估企业知识库和售后服务 Agent。
- [Known fact] 售后知识目前分散在微信群、Excel 和老系统中，重复问答多。
- [Known fact] 新员工培训周期长，客户希望沉淀标准问答和工单经验。
- [Known fact] 客户对数据安全和权限隔离比较敏感。
- [Known fact] 售后负责人对解决方案态度积极。
- [Known fact] 信息化负责人重点关注系统集成风险。

## 推断与不确定性
- [Inference] 生产计划经理参会可能意味着知识库需求不仅限于售后，可能延伸至生产排程与工艺知识管理，但需进一步验证。
- [Inference] 信息化负责人对集成风险的担忧，暗示其老系统可能存在接口封闭或数据结构复杂的问题。
- [Missing] 预算来源与金额。
- [Missing] 决策时间表与采购流程。
- [Missing] 竞品评估情况（是否已在对比其他厂商）。
- [Missing] 最终决策人身份与权限。

## 痌点与机会
- **痛点**：
  - 知识碎片化导致售后响应效率低、重复劳动多。
  - 人员培训成本高、周期长，经验难以传承。
  - 数据安全与权限管控存在合规或管理焦虑。
- **机会**：
  - 售后 Agent 与知识库的组合能直击售后部门效率痛点，业务价值明确。
  - 售后负责人可作为内部 Champion 推动项目立项。
  - 若能安全、平滑地整合老系统数据，将直接消除信息化负责人的核心顾虑，形成差异化优势。

## 风险信号
- [Risk signal] 系统集成风险：老系统（微信群、Excel、遗留系统）的数据抽取与实时对接可能技术难度大，影响项目交付范围与周期。
- [Risk signal] 数据安全与权限隔离：客户的高敏感度可能导致对 Agent 数据访问范围提出严苛限制，影响产品效果展现。
- [Risk signal] 商务推进风险：预算、决策时间表与竞品情况完全缺失，项目可能停留在初步意向阶段，无法转化为确定性商机。
- [Risk signal] 跨部门诉求差异：业务端（售后）求快求效，技术端（信息化）求稳求安全，若未对齐可能导致内部僵局。

## 推荐下一步
1. **业务场景深挖**：与售后负责人单独沟通，收集3-5个典型高频问答场景与工单流转现状，为产品演示准备针对性素材。
2. **技术顾虑消除**：安排我方售前/架构师与信息化负责人进行技术预沟通，重点了解老系统现状，初步探讨非侵入式集成方案与权限隔离架构。
3. **商机信息补全**：在后续互动中，通过业务痛点量化（如计算当前重复问答耗费的人力成本）引导客户探讨预算与决策时间表；侧面询问是否接触过其他同类厂商。
4. **扩大需求范围**：向生产计划经理了解其在生产排程中的知识调用痛点，评估是否可将知识库价值延伸至生产环节，提升项目整体ROI。

## 交接建议
- **Presales（售前）**：需交接客户对老系统集成的担忧与数据权限隔离的敏感度，要求售前准备系统集成可行性分析及安全架构说明材料；准备售后 Agent 场景化 Demo。
- **Risk Review（风险评审）**：若后续客户提出“保证全部老系统无缝对接”或“要求极低价格下的固定交付范围”，需提交风险评审，避免无限定制或低价固定范围陷阱。
- **Delivery PM（交付项目经理）**：暂不需深度交接，但需提醒关注未来可能面临的微信群/Excel等非结构化数据清洗与老系统接口对接的交付挑战。
