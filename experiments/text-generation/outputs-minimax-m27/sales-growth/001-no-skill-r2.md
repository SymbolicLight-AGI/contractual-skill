# 销售跟进建议

## 客户画像
华东离散制造企业，正在评估企业知识库和售后服务 Agent，属于制造业中大型客户。客户有明确的知识管理痛点，决策链涉及信息化、售后、生产三个核心部门。

## 明确事实
1. 售后知识分散存储于微信群、Excel表格、老旧系统三处
2. 一线人员重复回答同类问题，效率损耗明显
3. 新员工培训周期长，依赖老员工口口相传
4. 客户明确关注数据安全和权限隔离功能
5. 参会三方：信息化负责人、售后负责人、生产计划经理

## 推断机会
1. **知识整合需求强烈**：多渠道分散信息说明客户已意识到管理瓶颈，有明确整合意愿
2. **培训效率痛点**：新员工上手慢是持续性成本，降低培训周期能快速量化ROI
3. **内部 champion 潜力**：售后负责人态度积极，可能成为项目推动者
4. **多角色参会**：说明企业对此次评估重视，三个部门均有业务场景支撑决策
5. **潜在扩展可能**：生产计划经理参会暗示知识库应用可能延伸至生产知识管理

## 风险信号
1. **信息化负责人风险偏好低**：关注系统集成风险，对变更持谨慎态度，需降低其顾虑
2. **预算未明**：无法判断项目优先级和推进力度
3. **决策时间不明**：无法制定销售周期规划
4. **竞品对比未知**：不清楚客户当前评估范围和竞品情况

## 下一步动作
1. **针对售后负责人**：安排产品演示，重点展示知识库构建便捷性和智能问答能力，强化"减少重复回答"价值
2. **针对信息化负责人**：提供数据安全白皮书和权限隔离方案文档，安排技术架构沟通，消除集成顾虑
3. **探查决策节奏**：下次沟通时询问评估时间表和决策流程
4. **探查预算**：了解客户对知识库项目的预期投入范围
5. **收集竞品信息**：了解客户是否已在对比其他供应商

## 需交接事项
1. 客户对数据安全和权限隔离有明确要求，需产品团队确认功能满足度
2. 售后负责人是关键突破点，但最终决策可能受信息化负责人影响，需平衡两边诉求
3. 客户未透露预算和决策时间，建议在下一次拜访前通过电话或邮件提前了解
4. 客户地理位置在华东，建议后续安排现场拜访或工厂参观，深化信任关系
