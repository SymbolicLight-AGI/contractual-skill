# 销售跟进建议

## 客户画像

| 属性 | 内容 |
|------|------|
| 客户类型 | 华东离散制造企业 |
| 评估产品 | 企业知识库 + 售后服务 Agent |
| 关键决策人 | 信息化负责人（关注系统集成风险）、售后负责人（积极）、生产计划经理 |
| 需求阶段 | 初步评估期 |

---

## 已知事实

1. **客户明确表达的需求**
   - 售后知识分散在微信群、Excel 和老系统中，重复问答多
   - 新员工培训周期长，期望沉淀标准问答和工单经验
   - 对数据安全和权限隔离比较敏感

2. **内部观察**
   - 售后负责人态度积极，可能是内部 champion
   - 信息化负责人关注系统集成风险

## 痛点与机会

### 痛点
- 售后知识分散导致重复劳动、响应效率低
- 新员工依赖人工带教，培训成本高
- 多数据源（微信/Excel/老系统）难以统一管理

### 机会信号
- 客户已有明确场景（知识沉淀 + 智能问答），适合知识库 + Agent 产品定位
- 售后负责人积极，存在内部推动力
- 客户主动提出需求，说明已有一定痛点认知

---

## 风险信号

| 风险类型 | 描述 |
|----------|------|
| **集成风险** | 信息化负责人关注系统集成，需在方案中明确接口方式和兼容性 |
| **数据安全** | 客户明确强调数据安全和权限隔离，需说明技术实现和部署方式 |
| **范围不清** | 当前数据分散，可能涉及历史数据清理或迁移，需评估额外工作量 |
| **决策链路不明** | 尚未确认最终决策人和决策流程 |

---

## 推荐下一步

1. **约见售后负责人**（champion）进行深度需求挖掘，明确知识库结构、权限层级、工单场景细节
2. **与信息化负责人单独沟通**，了解现有系统架构和集成顾虑，邀请售前介入评估
3. **获取关键信息**：
   - 预算范围（可从项目规模、预期价值角度切入）
   - 期望上线时间
   - 决策流程和决策人
   - 竞品评估情况
4. **安排轻量级产品演示**，结合客户场景（售后知识库 + Agent）进行展示
