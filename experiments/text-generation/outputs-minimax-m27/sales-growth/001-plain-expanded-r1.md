## 客户跟进建议

### 客户画像

| 项目 | 内容 |
|------|------|
| 客户类型 | 华东离散制造企业 |
| 评估产品 | 企业知识库 + 售后服务 Agent |
| 参与角色 | 信息化负责人、售后负责人、生产计划经理 |
| 决策倾向 | 初期评估阶段，尚未进入采购决策 |

---

### 明确事实

- 售后知识分散存储于微信群、Excel表格、老系统三处，存在重复问答
- 新员工培训周期长，标准问答和工单经验尚未结构化沉淀
- 客户对数据安全和权限隔离有明确敏感性
- 售后负责人态度积极，信息化负责人关注系统集成风险

---

### 推断机会

- **知识整合需求**：多系统并存、重复问答 → 有知识库整合空间
- **培训效率诉求**：新员工上手周期长 → 知识沉淀可缩短培训周期
- **Agent应用场景**：售后工单处理 → 存在 Agent 替代重复问答的可能
- **集成顾虑**：信息化负责人关注集成 → 需确认与现有 ERP/WMS 等系统的对接方式

---

### 风险信号

| 风险类型 | 具体表现 |
|----------|----------|
| 采购不确定性 | 预算未明、决策时间表未明，需求强度待验证 |
| 竞争风险 | 未确认客户是否已在评估竞品 |
| 技术风险 | 信息化负责人对集成风险有顾虑，需技术评估 |
| 承诺风险 | 客户提到“数据安全和权限隔离敏感”，若前期承诺过多可能引发后期验收纠纷 |
| 范围风险 | “标准问答和工单经验沉淀”若转化为固定交付承诺，可能导致后期范围蔓延 |

---

### 下一步动作

1. **针对售后负责人**（高意愿）
   - 安排小范围 demo，重点演示知识沉淀路径和 Agent 问答能力
   - 收集其工单量/问答频率数据，用于需求量化

2. **针对信息化负责人**（关注集成）
   - 安排技术/售前对接，提供现有系统清单，评估集成可行性
   - 明确数据权限模型，消除安全顾虑

3. **信息收集**
   - 通过侧面或直接询问确认：是否在评估竞品、预算框架、期望上线时间
   - 识别最终决策人

---

### 需要交接的事项

| 交接对象 | 交接内容 | 优先级 |
|----------|----------|--------|
| 售前/技术团队 | 系统集成可行性评估、权限模型设计 | 高 |
| 方案团队 | 知识库与 Agent 联动场景细化 | 中 |
| 风险评审 | 客户安全要求较高，需确认合规性和数据隔离方案 | 高 |
| 销售管理层 | 客户尚未明确预算和时间表，不宜过度承诺 | 中 |

---

### 待确认问题清单

- 预算范围或量级
- 期望的决策/上线时间节点
- 是否已在评估其他供应商
- 现有老系统的类型及数据接口情况
- 最终决策人是否为上述三位之一
