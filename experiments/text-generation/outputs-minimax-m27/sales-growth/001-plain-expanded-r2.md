# 销售跟进建议

## 客户画像

| 项目 | 内容 |
|------|------|
| **客户类型** | 华东离散制造企业 |
| **关注产品** | 企业知识库、售后服务 Agent |
| **决策层级** | 中层管理者（信息化、售后、生产计划） |
| **决策模式** | 多部门联合评估，信息化拍板技术，售后主导业务需求 |
| **紧迫程度** | 未明确，有明确痛点但无时间压力信号 |

---

## 明确事实

1. **现状痛点**
   - 售后知识分散于微信群、Excel、老系统，缺乏统一沉淀
   - 标准问答和工单经验未能结构化，导致重复劳动

2. **明确需求**
   - 建立统一售后知识库
   - 缩短新员工培训周期
   - 知识资产沉淀与复用

3. **关键约束**
   - 对数据安全和权限隔离有明确敏感性
   - 信息化负责人关注系统集成风险

4. **内部氛围**
   - 售后负责人态度积极，可能为内部推动者
   - 信息化负责人偏谨慎，关注技术可行性

---

## 推断机会

1. **潜在需求延伸**
   - 生产计划经理参与可能意味着工单/流程自动化需求（待确认）
   - 知识库成功案例可促进向其他部门（如研发、工艺）复制

2. **采购触发点**
   - 人员流动或扩张带来培训压力上升时，可能加速决策
   - 竞品动作或行业标杆案例可能影响其时间表

3. **增值切入点**
   - 若能解决权限隔离问题，可形成差异化优势
   - 快速见效的"知识沉淀"场景比全面集成更易切入

---

## 风险信号

| 风险类型 | 具体表现 | 优先级 |
|----------|----------|--------|
| **信息缺口** | 无预算、无时间表、竞品情况不明 | 高 |
| **技术顾虑** | 信息化负责人对系统集成风险存疑 | 高 |
| **需求宽泛** | "企业知识库"和"售后 Agent"跨度大，尚未收敛 | 中 |
| **决策链不完整** | 未涉及高层（副总/CIO），中层决策可能被推翻 | 中 |
| **承诺风险** | 客户若问及价格、上线周期，勿直接回应 | 高 |

---

## 下一步动作

1. **本周内**
   - 与售后负责人建立私联，了解其内部推动意愿和期望
   - 确认是否已有竞品接触，记录客户评估维度

2. **两周内**
   - 安排线上/现场 demo，重点展示权限隔离机制和数据安全方案
   - 准备轻量级 POC 方案（聚焦"知识沉淀+基础问答"场景）

3. **持续跟进**
   - 每月一次价值传递（行业案例、方案迭代）
   - 关注客户侧人员变动或业务事件（如员工大量入职、旺季前）

---

## 交接事项

以下事项需移交 **售前/解决方案团队** 评估：

| 移交事项 | 接收方 | 原因 |
|----------|--------|------|
| 权限隔离方案细化（多角色、数据分级） | 售前 | 客户明确敏感点，需技术方案支撑 |
| 系统集成可行性评估（与老系统对接） | 售前/架构 | 信息化负责人核心顾虑 |
| 知识库+Agent 组合方案 Scope 定义 | 解决方案 | 客户需求跨度大，需帮其收敛 |
| 数据安全合规性说明 | 法务/产品 | 制造业客户对数据驻留、权限要求可能高于标准 |

以下事项需 **内部讨论后决策**：

- 是否针对该客户启动免费 POC
- 是否需要高层（如 VP）出面加速决策

## 待确认问题（内部提问清单）

1. 客户当前是否有明确的竞争对手在评估？其关注点有何差异？
2. 客户的信息化预算周期是年度还是项目制？
3. 售后 Agent 是否需要与现有工单系统（如派单、CRM）集成？
4. "老系统"具体指什么系统？接口开放程度如何？
5. 是否有明确的决策人（如 CIO 或分管副总）尚未参与？
