## 报价草案风险检查报告

### 一、财务风险

| 风险项 | 风险说明 |
|--------|----------|
| **付款触发条件模糊** | “试运行通过”无明确定义，可能导致第二笔50%款项拖延或争议 |
| **潜在隐性成本** | 第三方系统接口费用未明确，可能产生额外支出 |
| **变更费用未约定** | 项目范围外变更的计费方式缺失，可能引发定价争议 |
| **价格合理性存疑** | 120,000元报价未提供工时或成本拆分依据 |

### 二、合同范围风险

| 风险项 | 风险说明 |
|--------|----------|
| **知识库初始化边界不清** | 数据清洗范围、初始数据量、数据迁移责任未定义 |
| **Agent功能范围未定义** | 1个客服Agent的能力范围、知识库规模、响应指标未明确 |
| **培训标准缺失** | 2次远程培训的时长、参与人数、内容大纲未约定 |
| **验收标准缺失** | 无功能验收清单或性能指标，影响交付确认 |

### 三、其他风险

- **知识产权归属**未约定
- **违约责任**未明确
- **保密条款**未提及
- **延期责任**未界定

---

### 四、待确认条款清单

1. **试运行通过的明确定义**
   - 具体功能测试项
   - 性能指标要求（如响应时间、准确率）
   - 双方签字确认流程

2. **数据清洗边界**
   - 初始数据量上限
   - 数据清洗范围（是否包含非结构化数据）
   - 数据迁移责任划分

3. **第三方系统接口费用**
   - 是否已含在报价内
   - 如不含，预估费用范围及支付方式

4. **变更计费机制**
   - 工时单价或固定费率
   - 变更审批流程

5. **客服Agent范围定义**
   - 知识库条目数上限
   - 支持的对话场景数量
   - 集成系统范围

6. **培训细节**
   - 每次培训时长
   - 参训人数上限
   - 培训内容大纲

7. **知识产权归属**
   - 系统代码所有权
   - 训练数据所有权

---

### 五、建议下一步

| 步骤 | 行动 |
|------|------|
| 1 | 要求销售方提供详细报价拆分（工时、费用明细） |
| 2 | 与销售方协商定义“试运行通过”的具体验收标准 |
| 3 | 明确数据清洗范围和第三方接口费用 |
| 4 | 补充变更管理条款和违约责任条款 |
| 5 | 法务审核合同文本后决定是否签署 |

> **提示**：以上仅为风险识别，不构成对报价条款的最终批准或确认。
