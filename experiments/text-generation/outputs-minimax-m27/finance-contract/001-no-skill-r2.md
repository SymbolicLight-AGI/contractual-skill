**财务风险**  
1. 固定报价 120,000 元 与实际工作量可能不匹配，若数据清洗或第三方接口产生额外费用，利润将被压缩。  
2. “试运行通过后 50%” 的付款节点缺少明确的验收标准，可能导致尾款拖延或无法收回。  
3. 未列明第三方系统接口费用，若后续需接入外部平台，额外成本将超出预算。  
4. 变更需求计费方式不明，若项目过程中出现需求变更，可能产生争议或额外计费。

**合同风险**  
1. **范围不清晰**：仅列出“知识库初始化、1 个客服 Agent、2 次远程培训”，缺少对数据清洗边界、Agent 功能深度的定义，容易导致范围蔓延。  
2. **验收标准缺失**：未定义“试运行通过”的具体指标（性能指标、用户满意度、上线时间等），可能导致项目方与客户对是否通过产生分歧。  
3. **变更管理机制缺失**：未约定变更需求的提出、评估和计费流程，增加合同纠纷风险。  
4. **责任与赔偿条款空白**：缺少因延期交付、质量缺陷或数据泄露导致的违约责任和赔偿上限，可能使公司承担过大风险。  
5. **知识产权归属不明确**：知识库内容、培训材料及 Agent 逻辑的版权归属未约定，后续使用或二次开发可能受限。  
6. **保密与合规条款缺失**：项目涉及客户内部数据和业务，未约定保密义务、数据安全措施以及合规要求，存在合规风险。  
7. **支付保障不足**：仅靠“合同签署后 50%”的预付款，无履约保函或第三方托管，支付安全性较低。  

**待确认条款**  
- 数据清洗的具体范围、清洗标准及费用归属  
- “试运行通过”的验收指标、验收流程及验收期限  
- 是否已包含第三方系统接口费用；若未包含，计费方式及预算上限  
- 变更需求的提出、评估、批准和计费流程  
- 违约责任、赔偿上限及履约保函或付款保障机制  
- 项目产出（知识库、培训材料、Agent 代码）的知识产权归属及使用授权  
- 保密义务、数据安全措施及合规要求（如 GDPR、个人信息保护法）  
- 付款里程碑的细化（如每个里程碑对应的工作产出、交付物及验收方式）  
- 售后支持、维护期限及费用（如有）  

上述风险点和待确认事项需在正式合同签署前与客户进一步协商、明确并形成书面约定，以降低财务和合同风险。
