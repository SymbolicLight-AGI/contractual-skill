## 财务与合同审查意见

### 审查结论

**不支持直接同意“先做后付”。**

在客户信用未知、项目范围未明确、成本尚未估算的情况下，同意此付款安排将构成未经授权的合同让步。

---

### 已知事实

| 项目 | 状态 |
|------|------|
| 项目范围 | 未形成正式方案 |
| 合同文件 | 尚未起草 |
| 客户信用评估 | 未完成 |
| 我方成本估算 | 尚未核算 |

---

### 财务与合同风险

1. **信用风险**：客户信用状况不明，无法评估违约概率
2. **范围模糊风险**：无正式方案意味着无法界定交付边界，一旦产生争议缺乏依据
3. **成本不可控风险**：实施人员投入、现场调研、第三方工具成本均未量化，“后付”金额无法确定
4. **法律约束缺失**：合同未起草阶段即承诺付款安排，不构成有效约束
5. **现金流风险**：我方先行垫付成本，资金占用时间及回收不确定性高

---

### 替代付款方案建议

| 方案 | 说明 |
|------|------|
| 里程碑付款 | 按阶段划分，每阶段验收后付款 |
| 预付款+尾款 | 启动前收取部分费用，交付后结清余额 |
| 信用评估后决策 | 完成客户信用调查再议 |
| 小范围试点 | 以最小投入验证合作可行性 |

---

### 必须确认的问题

在同意任何“先做后付”安排前，需由以下角色确认：

- [ ] **财务/信用审查**：客户提供信用资料，完成信用评估
- [ ] **项目经理**：明确项目范围，产出正式方案
- [ ] **财务**：核算我方成本，设定付款底线
- [ ] **法务/合同管理**：评估风险，确定可接受的付款条款
- [ ] **销售/业务负责人**：评估客户重要性与风险容忍度

---

### 建议回复方向

暂不承诺“先做后付”，向客户说明需完成内部评估流程。可表述为：

> “我们理解贵司的需求。我们将启动内部评审，评估合作方案后向您提供具体的付款安排建议，确保双方权益得到保障。”

---

**如用户坚持需要批准“先做后付”，须提交上级或财务/法务部门专项审批，未经授权不得承诺。**
