## 风险判断

**不支持直接同意“先做后付”的核心原因：**

1. **信用风险**：客户信用状况未知，无历史合作记录或信用评估数据，无法判断对方履约能力和付款意愿。

2. **范围风险**：项目范围尚未形成正式方案，交付边界不清晰。若采用先做后付，无法确定“完成”的定义和验收标准。

3. **成本风险**：涉及知识库实施人员投入、现场调研、第三方工具成本，资金占用和成本回收存在不确定性。

4. **合同保障缺失**：尚未起草正式合同，缺少违约责任、争议解决等保护条款，先做后付将使我方处于完全被动地位。

## 可替代付款方案

| 方案 | 说明 | 风险控制程度 |
|------|------|---------------|
| **里程碑付款** | 按项目阶段设置付款节点（如需求确认、开发完成、上线验收），每阶段完成后付款 | ★★★★★ |
| **预付款+尾款** | 启动前收取一定比例预付款（如30%-50%），项目结束时支付尾款 | ★★★★☆ |
| **分阶段试点** | 先以小范围试点项目启动，完成验收后签署正式合同，再扩大范围 | ★★★★☆ |
| **信用评估后决策** | 先进行客户信用调查，再根据结果决定付款条款 | ★★★☆☆ |
| **第三方托管** | 使用托管账户或第三方支付平台，降低双方风险 | ★★★☆☆ |

## 必须确认的问题

**在考虑任何付款安排前，需与客户确认以下信息：**

1. **客户信用与资信证明**
   - 是否可提供企业营业执照、财务报表或银行资信证明？
   - 是否有与我方或行业内其他供应商的合作案例？

2. **项目范围与需求**
   - 项目的具体范围、功能需求和验收标准能否形成书面文档？
   - 是否有明确的实施时间表和上线计划？

3. **成本与预算**
   - 客户对项目的总预算上限是多少？
   - 是否认可上述成本构成（人员、现场、第三方工具）？

4. **付款意愿与能力**
   - 客户对上述哪种替代付款方案有接受意向？
   - 是否愿意在合同中明确约定违约责任条款？
