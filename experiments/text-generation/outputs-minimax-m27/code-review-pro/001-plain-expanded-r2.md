## Code Review: `get_customer_notes`

### Overall Judgment
**严重安全问题 - 不建议合并**

---

### 主要发现

#### 1. [严重] SQL 注入漏洞
```python
query = f"SELECT * FROM notes WHERE customer_id = {customer_id}"
```
**问题**：直接拼接用户输入到 SQL 语句。

**影响**：
- `customer_id` 来自 HTTP query 参数，攻击者可注入恶意 SQL
- 示例攻击：`customer_id=1; DROP TABLE notes;--`
- 可导致数据泄露、篡改或完全丧失

**必需修复**：使用参数化查询
```python
query = "SELECT * FROM notes WHERE customer_id = %s"  # 或 ?
rows = db.execute(query, (customer_id,))
```

---

#### 2. [高] 缺少权限校验
**问题**：无验证请求者是否有权访问该客户备注。

**影响**：未经授权用户可能通过遍历 `customer_id` 获取任意客户数据（水平越权）。

**建议**：在查询前验证当前用户与 `customer_id` 的从属关系。

---

#### 3. [中] 无输入校验
**问题**：
- 无类型检查（应为整数）
- 无边界校验
- 异常输入可能导致数据库报错泄露信息

---

#### 4. [低] 错误处理缺失
**问题**：数据库异常无捕获，可能导致未处理的 500 错误。

---

### 测试建议

| 测试场景 | 目的 |
|---------|------|
| SQL 注入尝试：`1 OR 1=1`, `1; DROP TABLE` | 验证参数化查询有效性 |
| 非数字输入：`abc`, `NULL` | 验证输入校验 |
| 未授权访问：用户 A 查询用户 B 的备注 | 验证权限控制 |
| 超长输入 | 验证边界处理 |
| 数据库连接失败 | 验证错误处理 |

---

### 风险影响总结

| 维度 | 等级 | 说明 |
|------|------|------|
| 安全 | **严重** | SQL 注入可被利用获取/破坏全部数据 |
| 正确性 | **高** | 缺少权限校验导致越权访问 |
| 隐私 | **高** | 客户备注可能含敏感信息 |
