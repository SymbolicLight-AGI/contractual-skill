## 代码审查风险报告

### 总体判断

该 PR 存在**高风险**质量问题。主要风险集中在**后端权限校验缺失**，导致任何登录用户均可导出完整用户数据（包括手机号、邮箱等敏感信息），违反最小权限原则和数据保护要求。

---

### 主要问题

#### 1. 权限绕过风险（严重）

| 项目 | 内容 |
|------|------|
| **发现** | 后端 API 未实现角色校验，前端按钮的权限判断可被直接调用 API 绕过 |
| **代码依据** | 变更摘要明确指出"只有前端按钮做了管理员判断" |
| **影响** | 任意已登录用户可导出所有用户敏感数据 |

#### 2. 敏感数据导出风险（严重）

| 项目 | 内容 |
|------|------|
| **发现** | 导出字段包含手机号、邮箱等 PII（个人身份信息） |
| **影响** | 数据泄露范围覆盖姓名+联系方式+消费能力 |

#### 3. 审计日志缺失（高）

| 项目 | 内容 |
|------|------|
| **发现** | 无导出操作的审计记录 |
| **影响** | 无法追溯数据导出行为，违反合规要求 |

#### 4. 导出数量限制缺失（高）

| 项目 | 内容 |
|------|------|
| **发现** | 无导出数量上限控制 |
| **影响** | 可能被滥用进行批量数据拉取 |

---

### 风险与影响

| 风险等级 | 风险描述 | 潜在影响 |
|----------|----------|----------|
| **严重** | 未授权用户导出他人数据 | 违反 GDPR/个人信息保护法，可能导致监管处罚 |
| **严重** | 权限校验仅在客户端 | 任何懂 API 调用者均可绕过 |
| **高** | 无导出限制 | 资源滥用、批量数据泄露 |
| **高** | 无审计日志 | 合规审计失败、事后追溯困难 |
| **中** | 数据字段暴露 | 手机号+邮箱可用于社工攻击 |

---

### 测试建议

| 测试类型 | 具体建议 |
|----------|----------|
| **权限测试** | 验证非管理员角色调用导出 API 应返回 403 |
| **边界测试** | 测试导出数量达到上限时的行为 |
| **安全测试** | 验证直接调用 API 能否绕过前端权限 |
| **回归测试** | 确认导出操作被正确记录到审计日志 |
