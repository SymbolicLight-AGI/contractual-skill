**风险提示**  
- 配置中 `api_key` 已硬编码在代码库，属于密钥泄露风险。  

**建议措施**  

- **阻断合并**：在 CI 流水线中加入 secret 检测（如 git‑secrets、TruffleHog），对包含 `api_key` 的提交/分支直接阻止合并。  
- **轮换密钥**：立即在支付平台后台重新生成 `api_key`，并在新密钥生成后立即替换旧密钥。  
- **改用环境变量**：将 `api_key` 从配置文件中移除，改为从环境变量或密钥管理服务（如 Vault、AWS Secrets Manager）读取，例如 `env: PAYMENT_API_KEY`。  
- **补充测试/扫描**：  
  - 添加单元/集成测试，验证配置中不存在硬编码的敏感信息。  
  - 引入定期或每次提交的 secret 扫描任务，确保新提交的密钥能被及时发现。  
- **清理历史**：如果该密钥已经在仓库历史中，需要使用 `git filter‑branch` 或 `BFG Repo‑Cleaner` 等工具将其从所有提交中彻底删除。
