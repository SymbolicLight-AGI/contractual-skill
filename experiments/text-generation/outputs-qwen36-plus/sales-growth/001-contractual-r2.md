# 客户画像
- **行业/区域**：华东某离散制造企业
- **评估方向**：企业知识库与售后服务 Agent
- **核心干系人**：信息化负责人（技术/集成把关）、售后负责人（业务痛点方/积极推动者）、生产计划经理（潜在协同/数据源方）

# 已知事实
- [Known fact] 售后知识当前分散在微信群、Excel 和老系统中，导致重复问答多。
- [Known fact] 新员工培训周期长，客户明确提出需将标准问答和工单经验沉淀下来。
- [Known fact] 客户对数据安全和权限隔离高度敏感。
- [Known fact] 售后负责人态度积极；信息化负责人重点关注系统集成风险。

# 推断与不确定性

- [Inference] 生产计划经理参会，暗示知识库未来可能需与生产排程或工单系统联动。

# 痛点与机会
- **痛点**：知识资产碎片化导致服务效率低、人力成本高；新人培养依赖口口相传，标准化程度低；数据安全与权限管控要求严格。
- **机会**：知识库+Agent 方案可精准匹配“经验沉淀+智能问答”需求；售后负责人的积极态度有利于快速推进 POC；可借此建立信任，后续横向拓展至生产协同或全渠道智能客服。

# 风险信号
- [Risk signal] 信息化负责人对系统集成风险存在顾虑，若技术架构无法平滑对接现有 IT 环境，可能触发技术否决。
- [Risk signal] 数据安全与权限隔离要求高，若方案无法满足企业级合规或私有化部署标准，项目可能停滞。
- [Risk signal] 历史非结构化数据（微信群、Excel）清洗与迁移工作量易被低估，可能影响交付周期与成本。
- [Risk signal] 预算、决策链与时间表缺失，存在需求悬置或竞品截胡风险。

# 推荐下一步
1. **技术对齐会议**：邀请我方售前架构师，针对系统集成路径、数据安全架构与权限模型进行专项演示与答疑，化解信息化负责人顾虑。
2. **轻量级 POC 验证**：提供沙箱环境或标准模板，让售后团队实际体验知识导入、权限配置与 Agent 问答流程，验证业务价值。
3. **需求澄清访谈**：向客户发送标准化调研清单，明确预算区间、决策审批流、期望上线节点及竞品对比情况。
4. **内部方案预研**：拉齐售前与交付团队，初步评估老系统对接可行性与数据治理工作量，形成内部基线。

# 交接建议
- **售前/架构师**：负责输出系统集成可行性评估、数据安全与权限隔离架构草案，准备技术答疑材料。
- **交付/实施 PM**：评估非结构化历史数据清洗、迁移与知识库构建的工作量边界，明确交付前置条件。
- **风险/合规评审**：审核客户数据安全要求，确认方案是否符合企业级合规标准，规避无限定制或数据泄露风险。
