# 客户画像
- **行业/类型**：装备制造集团
- **核心诉求**：售后知识库建设、工单自动分类、与现有CRM及备件系统打通、管理层要求验证试点ROI
- **决策特征**：技术/业务侧倾向“先全量集成后验证”，商务侧希望打包后续所有二次开发成本，管理层高度关注投资回报与效果可见性

# 已知事实
- [Known fact] 客户明确提出建设售后知识库与工单自动分类功能。
- [Known fact] 要求与现有CRM系统和备件系统实现数据/流程打通。
- [Known fact] 管理层要求提供试点ROI评估。
- [Known fact] 客户提出实施路径偏好：“先把所有系统都接上，再看效果”。
- [Known fact] 客户商务诉求：报价需包含后续所有二次开发。
- [Known fact] 数据权限架构需覆盖集团、分公司、服务商三方。

# 推断与不确定性
- [Inference] 客户当前售后协同可能存在工单流转慢、备件匹配效率低或知识沉淀不足的问题（基于功能需求反推）。
- [Inference] 管理层对ROI的期待可能包含响应时效提升、备件周转率优化或人工成本下降等指标。
- [Missing Information] 预算范围、最终决策人/审批链路、项目启动与验收时间表、现有CRM与备件系统的接口开放程度及数据质量、ROI测算的具体基线指标与口径。以上信息缺失，需在后续沟通中明确。

# 痛点与机会
- **痛点**：多系统数据孤岛导致售后协同效率低；三方权限架构复杂，易引发越权访问或数据合规风险；“先全量集成”路径若缺乏收敛，易导致实施周期拉长、试点效果被稀释。
- **机会**：知识库与工单分类属标准化能力，具备快速落地与横向复制潜力；若试点ROI达标，易在集团内形成规模化采购；明确的权限与集成诉求为方案边界划定提供清晰输入。

# 风险信号
- [Risk signal] **范围蔓延风险**：“先接所有系统”与“报价含所有后续二开”属于无边界需求，极易导致交付成本失控、验收标准模糊及合同纠纷。
- [Risk signal] **集成与数据风险**：三方权限模型若未提前定义隔离策略与接口标准，可能引发安全合规问题及实施阻塞。
- [Risk signal] **商业承诺风险**：固定总价覆盖无限定制违反常规交付逻辑，需严格限定合同范围与变更机制。

# 推荐下一步
- **销售可继续推进**：
  1. 梳理客户现有系统清单、接口文档现状与数据质量基线。
  2. 准备标准化产品演示（聚焦知识库检索与工单分类准确率）。
  3. 与管理层对齐ROI测算框架（明确评估周期、对比基线、核心指标）。
- **禁止承诺**：不承诺全量系统一次性打通、不承诺报价包含无限次二开、不承诺具体上线时间与最终ROI数值。

# 交接建议
- **售前方案团队**：评估CRM与备件系统对接的技术可行性、接口开放度及三方数据权限架构设计；输出标准集成方案、定制化边界清单与试点收敛建议（建议首期聚焦1-2个核心场景）。
- **风险审查/法务合同**：针对“报价含所有后续二开”及“先全量集成”条款进行商业与合规评估；拟定范围限定条款、需求变更管理机制、验收标准与免责边界。
- **交付项目经理**：评估三方权限落地方案、试点范围界定、数据清洗与迁移工作量；制定分阶段上线与数据就绪检查清单。
- **财务/商务合同**：基于售前边界与风险审查意见，输出阶梯式报价模型（基础产品授权+标准集成实施+按需二开人天包），避免固定总价包死。
