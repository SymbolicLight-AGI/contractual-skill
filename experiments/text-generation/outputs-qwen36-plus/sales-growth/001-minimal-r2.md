**客户画像**
- 企业类型：华东地区离散制造企业
- 核心干系人：信息化负责人（关注系统集成与安全合规）、售后负责人（关注效率提升与知识沉淀，态度积极）、生产计划经理（参会，关注业务连续性）
- 当前阶段：企业知识库与售后服务 Agent 方案评估期

**明确事实**
- 知识资产分散于微信群、Excel 及老系统，导致重复问答率高。
- 新员工培训周期长，急需将标准问答与历史工单经验结构化沉淀。
- 对数据安全与权限隔离机制高度敏感。
- 预算范围、决策时间表、竞品对比情况均未明确（信息不足，存在不确定性）。

**推断机会**
- 业务价值：通过 AI 知识库与 Agent 可显著降低重复答疑成本，缩短新员工上岗周期，提升售后响应标准化水平。
- 推进路径：以售后负责人为内部倡导者切入 → 开展高频场景 PoC 验证 → 联合信息化团队完成安全与集成评估 → 进入商务与立项流程。

**风险信号**
- 系统集成风险：信息化负责人明确关注老系统对接与数据打通复杂度，需警惕“全量系统快速上线”预期。
- 合规与架构风险：数据安全与权限隔离要求严格，可能拉长技术评审与合规审批周期。
- 信息缺失风险：预算、决策时间表及竞品状态未明确，存在项目延期或流失的不确定性。
- 承诺红线：严禁承诺最终价格、折扣、付款条款、交付时间及固定实施范围。

**下一步动作**
1. 输出针对离散制造场景的权限隔离架构与数据安全白皮书/案例，定向对齐信息化负责人。
2. 联合技术团队规划小范围 PoC，聚焦高频售后问答场景，验证准确率与工单流转效率。
3. 与售后负责人沟通内部立项流程，探询预算区间与决策时间表，同步收集竞品对比信息。
4. 明确项目边界，仅承诺 PoC 验证范围，不承诺全量交付与最终商务条款。

**需要交接的事项**
- 移交售前团队：系统集成架构假设、权限隔离设计方案、初步实施范围边界，进行技术可行性评审。
- 移交风险评审团队：潜在定价模型假设、交付周期预估、数据安全合规要求及客户对“快速上线/固定低价”的潜在预期，进行风险备案。
- 内部同步：严格遵循不承诺最终价格、折扣、付款条款、交付时间及实施范围的销售纪律。
