# 客户画像

某装备制造集团，业务涉及多层级组织（集团、分公司、服务商）。需求聚焦于售后环节的数字化与智能化，希望构建统一的知识库、自动化工单分类，并实现与现有 CRM、备件系统的全链路打通，管理层关注试点 ROI 的可验证性。

# 已知事实

- **客户行业**：装备制造集团。  
- **需求明确表达**：  
  - 售后知识库；  
  - 工单自动分类；  
  - 与现有 CRM 和备件系统打通；  
  - 管理层希望看到试点 ROI。  
- **风险信号（客户原话或态度）**：  
  - 客户要求“先把所有系统都接上，再看效果”；  
  - 客户希望报价里包含后续所有二开；  
  - 数据权限涉及集团、分公司、服务商三方。

# 推断与不确定性

- **推断**：客户可能对单一功能试点的价值感知较弱，倾向于用全系统集成的效果来说服内部决策层。  
- **推断**：客户可能缺少内部 IT 规范或长期预算规划，将“后续所有二开”视作风险转移手段。  
- **不确定性（缺失信息）**：  
  - 预算范围与审批路径；  
  - 最终决策人及关键影响者；  
  - 试点时间线与整体上线里程碑；  
  - 现有 CRM 与备件系统的厂商、版本、接口方式、数据模型；  
  - 三方数据权限的具体隔离要求、合规政策；  
  - 知识库语料的来源、格式与质量。

# 痛点与机会

- **痛点**：售后知识分散，工单处理依赖人工经验，跨系统数据割裂，难以量化改进效果。  
- **机会信号**：管理层关注 ROI，表明存在可量化的试点场景（例如先解决工单分类效率提升），可作为销售切入的支点。  
- **机会信号**：三方数据权限需求说明客户有真实的业务协作痛点，若能提出合规的权限方案，可形成差异化竞争力。

# 风险信号

| 风险信号 | 类型 | 可能影响 |
|---|---|---|
| “先把所有系统都接上，再看效果” | 范围蔓延、交付风险 | 项目变成全系统集成一次性实施，范围不可控，失败率高，验收标准模糊 |
| 报价里包含后续所有二开 | 固定价格、无限范围 | 严重财务风险，二开成本无法预估，且会形成长期隐性承诺 |
| 数据权限涉及集团、分公司、服务商三方 | 数据合规、权限冲突 | 需明确数据隔离规则，否则可能导致隐私泄露或合规问题，并影响系统架构 |

上述三个风险信号均已超出普通销售可自行消化的范围，必须由专业角色介入评估。

# 推荐下一步

**销售可以继续推进的事项**  
1. 澄清业务场景，引导客户聚焦一个可量化 ROI 的试点模块（如工单自动分类），并确定试点所需的有限集成范围。  
2. 引导客户明确后续二开的定义与边界（是否仅修bug还是功能新增），设法将“含所有二开”调整为按需项目制或年度维护包。  
3. 进一步挖掘决策链和预算归属，了解 ROl 试点批准需要谁签字。

**必须移交的事项**  
以下事项销售不得自行承诺或推进，须交由对应角色完成评估与方案设计，获得确认后方可形成客户沟通材料：

- **售前方案团队**：  
  - 评估与现有 CRM、备件系统打通的技术可行性与工作成本；  
  - 产出推荐的试点集成范围与全系统集成的分阶段路线图；  
  - 就“工单自动分类”和“知识库”所需的数据、模型、部署环境给出技术假设。

- **风险审查**：  
  - 审查客户“全系统对接再看效果”对合同 SOW 和验收条款的风险；  
  - 审查“含所有二开”的报价模式是否可接受，设计替代性计费与限制条款；  
  - 评估三方数据权限架构的合规边界，明确责任归属。

- **交付项目经理**：  
  - 评估数据权限三方隔离的落地复杂度和工期影响；  
  - 评估试点数据就绪程度、与周边系统的对接节奏；  
  - 明确试点阶段的验收标准和 ROI 测量方式。

# 交接建议

根据以上分析，建议**立即启动售前方案、风险审查、交付经理三方的内部同步会议**，销售暂不向客户提供任何技术或价格承诺。具体交接材料如下：

**交接给售前方案**  
- 任务：基于一个试点场景（工单自动分类与 CRM 对接）输出技术方案草案、集成范围边界、实施假设与限制条件。  
- 需指明：无法给出“全部系统一次性对接”的方案，仅提供分阶段演进路线。

**交接给风险审查**  
- 风险点：客户要求固定价格含无限二开，且意图一次性全系统集成。  
- 需明确：合同侧必须设定二开范围上限，且全量集成的验收标准不得由客户单方面定义“先接上看”；试点合同应限定范围。  
- 需注意：三方数据权限可能涉及数据法务条款，需提前征询。

**交接给交付经理**  
- 需产出：试点项目的大致实施周期估算（基于最简可交付范围）、数据权限隔离的方案草案、所需客户配合事项清单。  
- 提示：若客户坚持“所有系统全接”，交付风险极高，建议作为内部红线。
