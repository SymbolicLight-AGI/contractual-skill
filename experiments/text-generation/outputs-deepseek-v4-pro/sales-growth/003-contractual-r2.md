# 推断与不确定性

1. **推断**：客户可能希望快速看到原型或可用成果，因此关心价格和上线时间。但这不一定等于客户接受低质量或全风险固定范围。
2. **不确定性（关键缺失信息）**：
   - **数据规模**：知识库覆盖的异常类型、质检标准、历史案例数量不明确。
   - **系统对接**：是否需要对接 MES、ERP 或质检系统未确认。
   - **实时性要求**：是否涉及生产设备实时数据未确认。
   - **部署方式**：是否需要现场部署或纯 SaaS/私有化部署未确认。
   - **合同与验收标准**：范围边界、交付定义、验收指标未定义。

# 痛点与机会

- **痛点信号**：
  - 生产异常知识分散，查询效率低，质检经验依赖人工，一致性难保证。
  - 客户寻求快速上线，可能受限于内部效率压力或合规/质量事件推动。
- **机会信号**：
  - 食品行业质检与异常管理是高频刚需，一旦方案可靠，可形成标杆。
  - 若数据基础较好、场景封闭，可能较容易定义最小可行范围。

# 风险信号

1. **价格风险**：在未明确数据规模、系统对接和部署方式的情况下，“10万以内全包”属于高风险固定价格承诺，极易导致范围蔓延和亏损。
2. **上线时间风险**：未评估数据准备、集成和验证工作即承诺“2周上线”，大概率不可实现，将直接损害信任。
3. **范围风险**：未界定“生产异常知识库”和“质检问答”的具体功能边界、语料来源、更新机制、准确率承诺，轻易承诺等同于无限责任。
4. **集成风险**：若需要对接 MES/ERP/质检系统，两周内完成接口开发、联调、数据清洗几乎不可能。
5. **交付风险**：现场部署需求未确认，若需在工厂侧部署，网络、硬件、合规要求会进一步延长周期。
6. **合规与质量风险**：食品加工涉及工艺合规与质量追溯，Agent回答错误可能造成生产或合规事故，必须由风险审核介入。

# 推荐下一步

## 绝不可发送给客户的语句（基于给定约束，以下语句被禁止）

> ~~“我们可以 10 万以内全包，保证 2 周上线。”~~

**原因**：违反 Skill 关于“不得承诺最终价格、折扣、付款条件、交付时间表、实施范围或保证结果”的硬性约束，且属高风险不可控承诺。

## 可以安全发送给客户的替代表述（内部建议措辞）

```
关于您关心的投入和上线节奏，我们非常理解您希望快速落地并控制成本。
目前项目还处于早期，我们建议先一起明确以下几件事，这样才能给出合理的范围评估和靠谱的时间预估：
1. 知识库和质检问答具体覆盖哪些异常/质检项？是否有现成的文档、SOP或历史案例可作种子数据？
2. 是否需要与现有 MES、ERP 或质检系统对接？系统接口是否可开放？
3. 是否要求现场部署，还是可接受云端方案？
4. 对于回答的准确率和覆盖范围，您初步的期望是什么？

在厘清以上信息后，我们可以快速设计一个最小可行方案，并匹配分阶段的交付计划和预算区间，避免一开始就锁定不合理的总价和时间。我们最快可以在[澄清会议后X个工作日]内提供一个原型范围和阶段化预算供您决策。
```

# 交接建议

- **必须立即移交售前**：进行解决方案范围界定、技术可行性分析和最小可行范围（MVP）设计。
- **必须立即移交风控/法务 & 交付PM**：评估“全包固定价”和“两周上线”的合同条款风险。任何固定工期、固定总价且范围开放的承诺都必须经风险审核并设计严格的变更控制机制。
- **建议提前准备交付 PM**：在收到客户数据与对接需求后，给出数据就绪度评估和真实交付排期，避免后续被动承诺。
