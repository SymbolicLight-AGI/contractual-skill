# 已知事实

- 客户要求“下周给我一个方案”。

# 推断与不确定性

- **推断**：客户可能对快速见效有期待，但对 AI Agent 的适用场景、所需数据和集成复杂度缺乏认知。
- **推断**：客户内部可能存在某种效率提升或自动化诉求，但未明确——可能是客服、内部协作、数据分析等。
- **不确定性**：
  - 业务场景未说明，无法判断 Agent 匹配度。
  - 预算、决策人、上线时间、数据来源、系统环境均缺失。
  - “一周上线”的询问可能表明客户对实施周期有不合理预期。
  - 没有明确需解决的问题或痛点，兴趣可能是试探性了解。
  - 缺乏会议纪要进一步验证客户真实意图。

# 痛点与机会

- **潜在痛点**（需待确认）：经销商可能在订单处理、库存查询、客户询价、维修派单、报表生成等环节存在效率瓶颈，希望通过 AI Agent 实现自动化或辅助。
- **机会信号**：客户主动要求方案，表现出初步意愿，可借此机会进行需求挖掘和方案教育。
- **机会局限**：无具体场景、无预算、无决策人信息，尚无法评估项目成熟度，现阶段不宜视为重点机会。

# 风险信号

- **“一周上线”诉求**：隐含对实施复杂度的低估，可能导致后续范围失控或验收争议。
- **无场景、无数据、无系统环境**：可能为泛泛的兴趣，存在“要了方案就再无下文”的风险。
- **决策人未知**：无法判断给出方案的接对象是否具备预算权限。
- **固定价格、快速交付风险**：若后续客户期望低价格且一周交付，极易导致范围蔓延或交付失败。

# 推荐下一步

- **暂不将该机会标记为重点推进**，仅列入培育线索；直至至少明确了业务场景、预算和决策人。
- **下周方案提供前**，先安排一次简短的业务需求沟通（哪怕10分钟电话），询问：
  - 希望 AI Agent 首先解决什么具体业务问题？
  - 目前该问题如何解决？用哪些系统或手工流程？
  - 需要涉及哪些数据来源（ERP、Excel、微信、邮件等）？
  - 决策流程和预期节奏？
  - 预算范围或审批路径？
- **方案输出**：基于沟通内容，仅输出通用场景介绍和需要客户准备的条件（如数据、系统接口），不承诺具体功能范围、价格、上线时间。
- **内部对齐**：提前告知销售和售前，客户有“一周上线”的期望，需在沟通中管理预期，避免售前在任何书面材料中默认可实现。

# 交接建议

- **售前/方案团队**：待业务场景初步明确后介入，输出技术可行性和所需条件，不提供固定交付计划。
- **风险审核**：若客户坚持“一周上线固定报价包干”的模式，必须提交风险审核，避免做出无保障承诺。
- **交付PM**：暂不涉及，待明确场景和数据环境后评估。
