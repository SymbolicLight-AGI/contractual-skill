# 结论摘要

当前信息不支持“先做后付”。缺少客户信用、项目范围、成本估算及合同框架，直接承诺将引发回款、范围蔓延及成本超支等不可控风险。建议暂停该回复，并向客户提供分期付款或里程碑付款等替代方案，待必要信息核实后由相关团队评审决定。

# 缺失信息

- 项目范围与验收标准
- 详细成本估算（人力、差旅、第三方工具）
- 客户信用报告与付款历史
- 付款触发条件与里程碑定义
- 合同条款（包括违约责任、知识产权、终止条件）
- 内部成本审批与定价策略
- 客户要求的“先做后付”具体范围与期限

# 财务与合同风险

- **回款风险**：客户信用未知，无付款承诺，可能延迟支付或拒付，影响现金流。
- **成本风险**：成本未量化，若先投入后无法收回，将直接造成亏损。
- **范围蔓延风险**：无正式方案，客户可能在过程中不断追加需求，导致成本失控。
- **法律责任风险**：无合同约束，发生纠纷时我方权益无保障。
- **验收风险**：无验收标准，完工后客户可能以不满足要求为由拒绝支付。
- **执行风险**：未明确项目边界与交付成果，易引发双方预期不一致。

# 不得直接承诺事项

- 不得批准“先做后付”的付款方式。
- 不得在无信用评估、无合同的情况下承诺先行投入资源。
- 不得替代销售或法务给出有约束力的付款条件。
- 不得在成本不明时隐含接受无偿服务或无限期垫款。

# 待确认问题

在考虑任何灵活付款安排前，必须由对应团队逐一确认：

1. **客户信用**：是否已进行信用调查？历史交易回款记录如何？
2. **项目范围**：具体交付物、验收标准和排除项有哪些？
3. **成本估算**：人力、差旅、第三方工具等总投入预估多少？
4. **付款方案**：是否可接受前期部分预付款（例如30%启动款）+里程碑付款？
5. **合同条款**：付款节点、违约责任、交付验收机制、知识产权归属是否已草拟？
6. **内部审批**：该特殊付款方式是否已获财务、销售管理层批准？
7. **风险承受**：若客户最终不付款，我方最大可承受损失额度是多少？

# 交接建议

- **财务**：负责评估客户信用、审核付款方案及现金流影响，提供可接受的风险敞口意见。
- **法务**：负责起草或审阅约束性合同条款，明确付款义务、违约责任与验收标准。
- **交付PM**：负责定义项目范围、估时、成本及验收标准，确保可交付性。
- **销售**：负责与客户协商可替代的付款安排（如分阶段付款），并在内部发起审批。
