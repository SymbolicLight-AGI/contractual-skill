**审查结论（Summary Judgment）**  
报价草案存在中高财务与合同风险，不可直接推进签署。主要缺口集中在验收标准缺失、范围模糊、第三方费用归属未定和变更计费机制空白，可能导致成本超支、回款延迟或责任争议。

**已知事实**  
- 项目：售后知识库 Agent 试点  
- 首期报价：120,000 元  
- 付款节点：签约后 50%，试运行通过后 50%  
- 范围：知识库初始化、1 个客服 Agent、2 次远程培训

**财务与合同风险**

| 风险点 | 说明 |
|--------|------|
| **试运行通过标准缺失** | “试运行通过”无客观定义，客户可无限期推迟验收，导致尾款 50% 回流延迟。财务上形成大额应收账款，现金流风险高。 |
| **数据清洗边界未定** | 知识库初始化所需的数据清洗工作量、质量要求和范围未约束，易引发范围蔓延和额外成本，报价可能严重低估。 |
| **第三方系统接口费用未包含** | 若 Agent 需对接 CRM、工单等系统，接口开发、授权或集成成本由谁承担不清，可能直接侵蚀利润。 |
| **变更需求计费方式空白** | 试点期大概率产生增项需求，无变更单价或机制则每次协商陷入拉锯，交付团队无法保护自身成本。 |
| **远程培训次数与范围未定义** | “2 次远程培训”未规定参训人数、时长、教材、是否录制，存在被要求无限加场或扩大范围的风险。 |
| **“首期”暗示后续阶段未锁定** | 未明确后续扩容定价原则，可能被锁定在低价试点框架内，丧失未来议价权。 |
| **知识库初始化交付物未定** | 交付形式（文件、平台、覆盖率、准确性目标）不清晰，易引发主观拒收。 |

**待确认与澄清的问题**

1. 试运行通过的具体标准是什么？（例如：连续多少天无阻断性问题、准确率达到 X%、响应时间满足 Y）  
2. 数据清洗的范围与深度如何界定？是否包含历史数据去重、清洗、格式化？超量数据如何计费？  
3. 第三方系统接口（如 CRM、工单系统）集成费用由谁承担？是否需额外采购 license？  
4. 变更需求如何提报、评估和计费？是否为固定人天单价？是否有变更控制流程？  
5. 2 次远程培训的具体规格：每次时长、最大参训人数、是否可分发录制视频、是否包含教材更新？  
6. 首期后，二期或扩容是否有定价保护或优先约定？  
7. 知识库交付验收标准及维护期如何定义？上线后缺陷修复是否含在本报价内？

**建议审阅与交接方**

- **销售与客户负责人**：与客户协商明确验收标准、集成范围与变更机制，锁定双方预期。  
- **交付/项目经理**：评估数据清洗真实工作量和接口风险，提供内部人天估算。  
- **法务**：起草或补充验收条款、SOW 附件，避免口头承诺。  
- **财务**：根据修正后付款条款重新评估收入确认时点和应收账款风险。  
- **产品/技术负责人**：确认 Agent 所需第三方接口是否依赖外部采购，是否需要额外预算。

**重要提示**  
以上分析不构成对任何条款的批准或最终合同表述。所有模糊项必须在签约前以书面方式明确，避免付款被无限期搁置或成本失控。
