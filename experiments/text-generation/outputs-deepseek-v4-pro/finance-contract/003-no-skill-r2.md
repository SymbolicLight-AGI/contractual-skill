# 风险识别、审批要求及可谈判替代方案

## 一、高风险让利与无限范围承诺识别

| 客户要求 | 风险类型 | 具体风险描述 |
|----------|----------|--------------|
| **七折** | 利润侵蚀 | 标准报价200,000元，七折后为140,000元，降幅达30%。在未确认毛利底线的情况下，可能导致项目亏损或无法覆盖固定成本、人工及管理分摊。 |
| **免费二次开发** | 无限范围承诺 | “二次开发”未界定功能范围、人天投入上限、需求来源（谁提需求）、验收标准。极易演变为无限追加开发、免费补丁升级，严重侵蚀利润并占用开发资源。 |
| **不限次数培训** | 无限责任 | 未限制单次时长、参与人数、培训形式（线上/线下差旅）、内容更新频率。可能导致需承担持续的人力成本、差旅成本及时间成本，且没有终止边界。 |
| **一年内任何问题免费处理** | 无限保修/无限支持 | “任何问题”涵盖范围过宽，可被解释为功能缺陷、新需求变更、第三方系统适配、使用咨询等。未定义响应时效(SLA)、解决时限、责任归属，将形成无底洞式售后义务。 |

> **共同致命点**：上述四项承诺相互叠加，形成“无限责任+无限范围+大幅折让”的高危合同结构，未获管理层审批前签署将产生不可控的履约风险及法律纠纷隐患。

## 二、审批要求

在客户要求写入合同补充条款前，必须完成以下内部确认与审批：

1. **毛利底线确认**：由财务/商务负责人核算项目直接成本与分摊成本，明确七折报价是否在可接受毛利区间内，并书面批复最低可签约价格。
2. **二开范围界定**：由项目经理/产品经理拟定二开功能清单或人天上限（如“不超过10人天”），经技术负责人评审、管理层批准后作为合同附件。
3. **培训方案量化**：明确培训次数（如“不超过3次”）、单次最长时长、参训人数上限、实施方式（线上或线下，差旅费承担方），经交付部门签批。
4. **售后SLA签订**：约定响应时间、解决时限、服务窗口（工作时间/7x24）、问题分类（缺陷、咨询、需求变更）及需求变更按新报价处理等机制，由服务部门与法务会签。
5. **管理层专项审批**：因涉及多维度无限责任让步，必须将该项目的整体风险敞口及所有限制性条款报批至具有签约权限的高级管理层，获得专项书面授权后方可对外承诺。

> **警示**：上述任一项未获批准前，均不得将客户要求直接写入合同补充条款或做出任何书面/口头承诺，否则可能构成表见代理，使公司被迫承担无限义务。

## 三、可谈判替代表述（不可作为合同条款，仅供内部讨论及谈判参考）

以下为向客户沟通时可用于替代高风险承诺的约束性表述思路，需结合审批结果进行调整：

**1. 价格折让替代方案**
- “在签约后X日内一次性付款的前提下，可申请特别折扣至X%。”
- “总价款20万元，如整体方案减配至X模块，价格可调整为14万元。”
- “首年以原价签约，次年续费赠送等价增值服务。”

**2. 二次开发限制性替代**
- “合同包含验收后X个月内的微调开发支持，累计不超过X人天，超出部分按标准人天单价结算。”
- “免费二开范围限定在与本合同功能直接相关、且经双方书面确认的《二次开发需求清单》之内（详见附件），超出清单部分按新项目评估。”

**3. 培训量化替代**
- “免费提供上线前标准操作培训，共计X次，每次不超过X小时，参训人数不限但集中在一个场次。”
- “额外培训可享合同价的X折优惠，年度内不限次数但需提前预约、差旅费用由贵方承担。”

**4. 售后支持限定替代**
- “提供自验收日起12个月的标准维护与技术支持，服务级别为5×8，响应时间2小时，涉及新功能开发或第三方系统集成按变更管理处理并另行计费。”
- “任何因甲方业务变更、接口调整、新增需求等非乙方交付缺陷引发的问题，纳入有偿技术支持范围。”

> **谈判底线**：所有承诺必须带上明确可量化的范围、上限、起止时间及超出部分的计价规则，避免出现“免费”“不限”“任何”等无限义务词语。
