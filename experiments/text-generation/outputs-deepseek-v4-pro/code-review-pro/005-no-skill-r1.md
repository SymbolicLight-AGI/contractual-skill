## 风险识别与处理建议

### 1. 权限风险 (Critical)
- **问题**：管理员权限仅在前端按钮显示层面进行判断，后端 API `/export/users` 未校验用户角色，攻击者可绕过前端直接调用接口导出全量用户敏感信息。
- **影响**：任意登录用户（甚至未登录，若接口无认证）均可导出姓名、手机号、邮箱及订单金额，导致数据泄漏。
- **建议**：后端强制校验调用方角色为管理员，接口层添加角色验证中间件。**需后端负责人和安全工程师立即修复。**

### 2. 隐私与合规风险 (High)
- **问题**：导出字段包含个人身份信息（PII）及财务相关数据（最近订单金额），但无任何脱敏、二次确认或用途声明，且缺少导出数量上限。
- **影响**：违反《个人信息保护法》等法规的最小必要原则；员工恶意导出或误操作可能引发合规事件；大量数据导出可能被用于撞库、诈骗等下游攻击。
- **建议**：
  - **产品负责人**确认导出字段的必要性，默认脱敏手机号、邮箱（如掩码），仅提供必要字段。
  - 增加导出前二次身份验证（如管理员密码确认）。
  - 设置单次导出条数上限（如 5000 条）并限制频率。
  - 明确导出数据的用途声明及留存策略。

### 3. 审计缺失风险 (Medium)
- **问题**：导出操作无任何审计日志，无法追溯谁在什么时间导出了哪些数据。
- **影响**：发生数据泄漏时无法定位责任人和泄漏范围，无法满足合规审查要求。
- **建议**：**安全工程师**与**后端负责人**设计审计日志表，记录操作人ID、IP、时间、导出参数和摘要（如导出 1200 条用户数据），并接入监控告警。

### 4. 功能与测试风险 (Medium)
- **问题**：变更摘要未提及对边界情况（如空数据、超大文件、并发导出）的处理及测试覆盖。
- **影响**：生产环境可能因大量导出导致内存溢出或超时，或导出文件格式错乱。
- **建议**：
  - **测试负责人**补充导出功能的性能测试、并发测试、断点续传测试。
  - **后端负责人**实现异步导出（任务队列）并添加文件有效期，避免即时全量查询。
