**代码审查结果：用户导出功能PR**

**概述**
此PR新增用户数据导出功能，当前实现存在多类高严重度隐患，不应直接合并。

---

### 风险清单（按严重度排序）

1. **权限绕过（严重）**
   - 仅前端做管理员角色判断，后端API未校验。
   - 攻击者可直接调用导出接口获取全量用户敏感数据。
   - **影响**：横向越权、数据泄露。

2. **隐私与合规（严重）**
   - 导出字段包含姓名、手机号、邮箱等个人身份信息（PII），缺失使用限制、脱敏选项和用户授权机制。
   - 无导出数量上限，可一次性拉取全库数据，违反最小必要原则。
   - **影响**：违反GDPR/PIPL等法规，面临合规风险。

3. **审计缺失（高）**
   - 无审计日志，无法追溯谁、何时、导出了哪些数据。
   - **影响**：事后无法定责，安全事件响应困难。

4. **性能与可用性（中）**
   - 无导出数量限制，大数据量可能打垮数据库和应用内存。
   - 缺少分页、异步导出或速率限制。

5. **测试覆盖（中）**
   - 未提及任何测试（单元/集成/安全测试）。
   - 需补充：权限拒绝、空数据、大量数据、并发导出、审计日志写入等场景。

---

### 需继续确认的责任方

| 角色 | 需确认事项 |
|------|------------|
| **安全工程师** | 实现后端RBAC校验；评审导出API的认证鉴权机制；定义PII脱敏规则；确认审计日志格式与存储。 |
| **后端开发** | 添加角色检查；引入导出数量上限（或分页/异步）；实现审计日志记录；考虑性能优化（流式导出/队列）。 |
| **产品负责人** | 明确导出功能的必要字段、导出限制（如仅导出本人或下属）、是否需要用户授权确认；定义导出频率与数据保留策略。 |
| **法务/合规** | 评估导出功能是否符合数据保护条例，是否需要用户同意或数据保护影响评估（DPIA）。 |

---

**结论**
当前PR存在未授权访问、隐私违规和审计缺失的高危风险，必须由安全、后端及产品负责人完成上述确认与修改后，重新提交审查。不要仅依赖前端控制。
