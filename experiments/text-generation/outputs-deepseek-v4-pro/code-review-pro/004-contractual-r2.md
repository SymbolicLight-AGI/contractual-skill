- Finding: 配置文件中包含明文支付API密钥（`api_key`），直接暴露在仓库中。该密钥前缀 `FAKE_PAYMENT_TOKEN_FOR_SYNTHETIC_TEST_ONLY` 表明属于生产环境活跃密钥，泄露后果严重。

- Risk: 密钥泄露可导致攻击者直接调用支付API，造成资金盗刷、数据窃取、业务中断及合规问题。项目无 secret 扫描，进一步扩大了攻击窗口期。

- Uncertainty: 无法确认密钥当前是否仍有效、权限范围、以及是否已被外部访问。需要与 backend owner 及安全团队核实。

  - 添加集成测试，确保支付调用使用轮换后的新密钥并能正确处理鉴权错误。

## 主要问题
1. **敏感信息硬编码**：`api_key` 值明文写入版本控制，违反安全基线。
2. **缺乏 Secret 管理机制**：未使用环境变量、外部密钥管理器或加密配置。
3. **无自动化扫描**：无 secret 扫描，无法阻止相似问题再次引入。

## 风险与影响
- **严重**：生产密钥泄露可被恶意利用执行未授权交易。
- **数据完整性**：攻击者可篡改支付金额或收款方。
- **合规**：可能违反 PCI DSS 或数据保护法规，导致罚款。
