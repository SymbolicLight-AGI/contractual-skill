## 客户画像

| 维度 | 描述 |
|:---|:---|
| **企业类型** | 华东离散制造企业，传统制造业数字化升级阶段 |
| **参会角色特征** | 信息化负责人（技术把关/风险规避）、售后负责人（业务痛点驱动/积极）、生产计划经理（潜在关联用户/观察角色） |
| **核心诉求** | 售后知识管理 + 员工赋能，非全面数字化转型 |
| **决策风格** | 谨慎，重视安全合规，需多部门共识 |

---

## 明确事实

1. 售后知识载体分散：微信群、Excel、老系统并存，缺乏统一管理
2. 核心痛点：重复问答消耗人力、新员工培训周期长（经验传承困难）
3. 安全诉求：数据安全与权限隔离为硬性要求
4. 角色态度分化：售后负责人推动力强，信息化负责人关注集成风险
5. 信息缺失：预算范围、决策时间表、竞品接触情况均未透露

---

## 推断机会

| 机会点 | 依据 |
|:---|:---|
| **知识库+Agent组合销售** | 客户同时提及"企业知识库"和"售后服务Agent"，可形成场景化方案而非单点产品 |
| **售后负责人作为内部 champion** | 其积极性可降低我方销售阻力，需重点培育 |
| **生产计划经理的延伸需求** | 离散制造场景下，生产计划与售后返修、物料BOM知识关联度高，存在扩展空间 |
| **老系统替换/对接的集成服务** | 信息化负责人的顾虑可转化为增值服务机会 |

## 风险信号

| 风险 | 等级 |
|:---|:---|
| **决策周期不确定** | ⚠️ 高 — 无时间表，可能处于早期调研阶段，存在长期跟进风险 |
| **预算未锁定** | ⚠️ 高 — 无预算确认，项目可能停留在设想层面 |
| **竞品信息空白** | ⚠️ 中 — 客户可能已接触或即将接触其他厂商，我方位置不明 |
| **信息化负责人的集成焦虑** | ⚠️ 中 — 若不能有效回应，可能成为技术否决点 |
| **多角色诉求平衡难度** | ⚠️ 中 — 售后要效率、IT要稳定、生产要协同，方案需兼顾 |

## 下一步动作

| 优先级 | 动作 | 责任人 |
|:---|:---|:---|
| P0 | 向售后负责人补充了解：当前售后团队规模、月均工单量、知识条目估算量（量化ROI基础） | 销售 |
| P0 | 向信息化负责人提供：**同类型制造企业集成案例** + **权限隔离架构白皮书** + **本地化/私有化部署选项** | 售前架构师 |
| P0 | 试探决策流程：询问"此类项目通常需哪些部门审批""是否需要纳入明年预算规划" | 销售 |
| P1 | 了解竞品接触情况："贵司目前是否接触过类似方案，评估维度有哪些" | 销售 |
| P1 | 邀请生产计划经理参与下一轮方案演示，挖掘生产侧知识管理需求 | 销售 |
| P2 | 准备**安全合规专项材料**（等保、数据分级、权限模型）作为差异化筹码 | 售前 |

## 需要交接的事项

| 事项 | 说明 |
|:---|:---|
| **客户决策链待补全** | 是否需总经理/财务副总审批？生产计划经理的实际影响力权重？ |
| **现有IT架构信息** | 老系统品牌、数据库类型、接口开放程度，需售前技术评估 |
| **预算窗口期** | 是否赶得上客户财年预算申报，需明确以调整销售节奏 |
| **竞品动态** | 客户是否已发RFP/已演示，需持续跟踪 |
