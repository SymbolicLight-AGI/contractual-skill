## 推断与不确定性

| 类别 | 内容 | 置信度 |
|:---|:---|:---|
| **推断** | 客户内部对"试点"定义可能存在分歧（管理层要 ROI vs. 执行层要全量上线） | 高 |
| **推断** | 现有 CRM 和备件系统可能为不同厂商/版本，接口复杂度待评估 | 高 |
| **推断** | "后续所有二开"可能隐含对当前系统功能的不完全认可 | 中 |
| **不确定性** | 现有系统品牌、版本、接口文档是否完备 | **缺失** |
| **不确定性** | 三方数据权限的具体合规要求（等保、数据出境、行业监管） | **缺失** |
| **不确定性** | 试点范围、预算上限、决策链及最终审批人 | **缺失** |
| **不确定性** | ROI 计算标准（效率提升？成本降低？客户满意度？） | **缺失** |

## 痛点与机会

| 痛点/机会 | 类型 | 说明 |
|:---|:---|:---|
| 售后知识分散，依赖人工经验 | 痛点 | 知识库需求明确 |
| 工单分类效率低 | 痛点 | 自动分类可量化效率提升 |
| 系统孤岛，数据未打通 | 痛点 | 集成需求 = 方案增值空间 |
| 管理层需 ROI 证明 | 机会信号 | 可设计分阶段验证路径，绑定预算释放 |
| "全量接入再看效果" | **风险型机会** | 若引导为可控试点，可缩短周期；若妥协，则放大交付风险 |

---

## 风险信号

| 风险类别 | 具体表现 | 严重程度 |
|:---|:---|:---:|
| **范围蔓延** | "所有系统都接上"后再看效果，试点边界失控 | 🔴 高 |
| **定价陷阱** | "包含后续所有二开" = 无限责任，固定价格+无限范围 | 🔴 高 |
| **数据合规风险** | 三方数据权限（集团/分公司/服务商），涉及数据归属、跨境/跨主体流转 | 🔴 高 |
| **集成可行性** | 现有 CRM/备件系统接口未知，技术假设未验证 | 🟡 中高 |
| **验收争议** | "效果"未定义，ROI 计算标准缺失，易引发验收纠纷 | 🟡 中 |

## 推荐下一步

### 销售可继续推进（无需额外审批）

| 事项 | 动作 |
|:---|:---|
| 客户关系维护 | 与管理层确认"试点"与"全量"的真实意图，探查是否存在内部意见分歧 |
| 信息补全 | 索要现有系统清单（CRM/备件系统品牌、版本、接口文档） |
| 预算探询 | 以"便于设计合适方案"为由，了解预算区间和审批流程 |
| 决策链梳理 | 确认除管理层外，IT 负责人、法务/合规负责人的参与程度 |

### 必须停止并移交的事项

| 事项 | 移交对象 | 原因 |
|:---|:---|:---|
| 系统集成可行性评估 | **售前方案（Presales）** | 需技术团队评估 CRM/备件系统接口、数据映射、集成架构 |
| "所有二开包含在报价内"的回应 | **风险审查（Risk Review）** | 涉及无限范围承诺，需法务/风控界定责任边界 |
| 三方数据权限的合规架构 | **风险审查（Risk Review）** | 数据归属、访问控制、审计留痕需合规确认 |
| 试点范围与全量上线的边界定义 | **售前方案 + 交付经理（Delivery PM）** | 需共同设计可交付、可验收的试点闭环 |
| 试点 ROI 的计算模型与基线数据 | **交付经理（Delivery PM）** | 需确认数据可获取性、指标可量化性 |

## 交接建议

### 致：售前方案（Presales）

| 项目 | 内容 |
|:---|:---|
| **交接材料** | 客户现有系统清单（待销售补全后转交）、集成需求清单 |
| **待确认问题** | ① CRM/备件系统接口能力与对接周期；② 是否支持渐进式集成而非一次性全量；③ 知识库与工单系统的 AI 能力匹配度 |
| **输出要求** | 技术可行性评估 + 分阶段集成方案（建议：试点期仅接核心系统，验证后再扩展） |

### 致：风险审查（Risk Review）

| 项目 | 内容 |
|:---|:---|
| **交接材料** | 客户"包含所有二开"的报价要求原文、三方数据权限描述 |
| **待确认问题** | ① "所有二开"是否可转化为有限范围的变更条款；② 三方数据权限下的最小必要数据原则适用性；③ 是否需要单独签署数据处理协议 |
| **输出要求** | 合同条款红线 + 风险缓释方案（如：人天包干 vs. 固定总价、数据隔离架构要求） |

### 致：交付经理（Delivery PM）

| 项目 | 内容 |
|:---|:---|
| **交接材料** | 管理层 ROI 诉求、试点/全量争议背景 |
| **待确认问题** | ① 试点 ROI 的量化指标建议（如：工单分类准确率、平均处理时长、知识库调用率）；② 基线数据获取方式；③ 最短可验证闭环周期 |
| **输出要求** | 试点验收标准草案 + 数据基线采集方案 |
