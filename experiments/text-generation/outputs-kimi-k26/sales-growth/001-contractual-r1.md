- 客户画像
  - 行业：华东离散制造
  - 规模：中大型企业（多部门参与）
  - 当前阶段：方案评估/早期调研
  - 关键角色：
    - 售后负责人：态度积极，业务痛点直接相关方
    - 信息化负责人：技术把关，关注集成风险
    - 生产计划经理：参与评估，具体诉求待明确
  - 核心诉求：售后知识管理 + 售后服务 Agent
  - 敏感点：数据安全、权限隔离

- 已知事实
  - 售后知识分散在微信群、Excel、老系统中，重复问答多（客户明确表达）
  - 新员工培训周期长，希望沉淀标准问答和工单经验（客户明确表达）
  - 对数据安全和权限隔离比较敏感（客户明确表达）
  - 售后负责人态度积极（我方观察）
  - 信息化负责人关注系统集成风险（我方观察）

- 推断与不确定性
  - 推断：售后业务量大，Agent 替代人工的 ROI 空间可能较高（无直接数据支撑）
  - 推断：生产计划经理参与可能暗示知识库需求延伸至生产领域，或仅为部门协同评估（不确定）
  - 不确定性：预算范围、决策时间表、竞品对比情况、现有系统清单及接口能力、数据量级、IT 团队规模、安全合规等级要求均缺失

- 痛点与机会
  - 痛点：
    - 知识碎片化导致效率损失
    - 经验传承依赖个人，新人培养成本高
  - 机会：
    - 售后 Agent 可量化降低重复咨询人力投入
    - 知识库建设可扩展至生产、质检等离散制造全链路
    - 安全敏感型需求可匹配私有化/混合云方案，提升客单价

- 风险信号
  - 系统集成风险：信息化负责人已明确关注，若现有系统老旧、接口封闭，实施复杂度可能超预期
  - 权限隔离诉求：多角色、多工厂场景下，细粒度权限设计可能增加定制开发量
  - 数据安全敏感：需确认是否涉及等保、关基等合规要求，影响部署架构和交付周期
  - 决策链不清晰：三部门参与但决策权和预算未明确，存在流程拉长风险
  - 竞品信息缺失：无法判断客户价格预期和方案偏好

- 推荐下一步
  1. 内部准备：整理同行业离散制造售后知识库落地案例（脱敏版），重点标注集成路径和安全架构
  2. 信息补全：通过售后负责人侧面了解预算区间、期望上线时间、是否接触竞品及反馈
  3. 技术预研：由售前评估常见老系统（如用友/金蝶/自研MES）对接可行性和数据迁移方案
  4. 安全方案：准备私有化部署、数据隔离、权限分级的产品能力说明，供信息化负责人专项沟通
  5. 关系深化：单独约访生产计划经理，明确其参与动机和潜在需求边界

- 交接建议
  - 售前：介入系统架构评估，输出集成可行性初步判断及技术假设清单
  - 交付PM：提前介入了解数据现状、现有系统环境，预判实施周期
  - 风险评审：若客户后续要求固定价格、固定工期或结果承诺，触发风险评审
  - 财务/合同：待预算和采购流程明确后，再进入报价和条款谈判

- 完成前自检
  - 未承诺价格、折扣、交付周期或实施范围
  - 未将客户"兴趣"转化为"购买意向"
  - 未推断具体系统名称或技术栈
  - 缺失信息（预算、时间表、竞品、决策链）已标注，未虚构填充
  - 个人联系信息无需额外处理（材料中未提供）
