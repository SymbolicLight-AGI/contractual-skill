## 代码审查结果

### 总体判断
**高风险 — 不予批准上线**

---

### 主要发现

| 优先级 | 类别 | 发现 |
|:---|:---|:---|
| **P0** | 权限绕过 | 后端 API 未校验管理员角色，非管理员可直接调用导出接口 |
| **P0** | 隐私泄露 | 批量导出用户姓名、手机号、邮箱，无数量限制，存在数据过度暴露 |
| **P1** | 审计缺失 | 无导出操作日志，无法追溯谁、何时、导出了哪些数据 |
| **P1** | 前端校验不可信 | 仅前端隐藏按钮，属于"安全通过 obscurity" |

---

### 风险影响

| 风险 | 具体影响 |
|:---|:---|
| **权限绕过 → 数据泄露** | 攻击者构造请求即可批量拉取全量用户敏感信息，违反《个人信息保护法》最小必要原则 |
| **无数量限制 → 性能/滥用** | 全表导出可能导致 DB 慢查询、内存溢出，或被爬虫利用 |
| **无审计 → 合规失败** | 发生泄露后无法定责，监管审计无法通过 |

---

### 测试建议

- 补充接口级权限测试：普通用户/未登录用户直接调用导出 API 应返回 `403`
- 补充速率限制/分页测试：验证单次/单用户导出上限
- 补充审计断言：验证导出行为记录到审计表

---

### 不确定性

- 不清楚该功能是否涉及跨境数据传输、是否需用户单独授权
- 不清楚"最近订单金额"是否含其他用户关联信息（如订单 ID 可反查）

---

### 需要继续处理的责任人

| 负责人 | 事项 |
|:---|:---|
| **安全负责人** | 确认数据分类分级、导出场景是否需 DPIA（数据保护影响评估）、是否触发安全事件响应流程 |
| **后端负责人** | 补全接口鉴权（建议接入统一权限中间件）、增加分页/数量限制、增加审计日志落库 |
| **产品负责人** | 确认导出字段是否全部必要（能否去掉手机号/邮箱）、导出场景的业务合理性、是否需用户授权或告知 |
