## PR 风险 Review

### 一、权限风险（高危）

| 问题 | 影响 |
|------|------|
| 后端 API 未校验角色 | 普通用户/访客可直接调用导出接口，绕过前端管控 |
| 仅前端做管理员判断 | 通过 curl/Postman 可直接攻击，形同虚设 |

**需安全负责人确认：** 接口是否需强制接入 RBAC/权限中间件，最小权限原则如何落实。

---

### 二、隐私风险（高危）

| 问题 | 影响 |
|------|------|
| 导出字段含手机号、邮箱 | 违反《个人信息保护法》最小必要原则 |
| 无脱敏处理 | 明文导出敏感个人信息，泄露风险极高 |
| 无导出数量限制 | 可一次性拉取全量用户数据，数据贩卖风险 |

**需安全负责人 + 产品负责人确认：** 字段范围是否合规、是否需要脱敏、是否需用户授权。

---

### 三、审计风险（中高危）

| 问题 | 影响 |
|------|------|
| 无审计日志 | 无法追溯"谁、何时、导出多少数据" |
| 无操作留痕 | 数据泄露后无法定责，合规检查（等保/ISO27001）不通过 |

**需安全负责人确认：** 审计字段标准（操作人、时间、IP、数据范围、用途）。

---

### 四、测试/其他风险（中危）

| 问题 | 影响 |
|------|------|
| 无导出数量限制 | 大数据量导出导致 OOM/DB 慢查询，影响服务可用性 |
| 无文件下载鉴权/过期机制 | 若生成临时文件，存在未授权访问或残留风险 |
| 未确认导出格式与存储 | CSV 注入攻击、临时文件落盘加密等 |

**需后端负责人确认：** 分页/流式导出、限流、文件安全处理。

---

## 处理分工建议

| 负责人 | 待确认事项 |
|--------|-----------|
| **安全负责人** | 权限校验补全、敏感字段脱敏方案、审计日志规范、等保合规性 |
| **后端负责人** | 接口鉴权中间件、分页/限流、文件生成与清理机制、性能防护 |
| **产品负责人** | 导出字段范围（能否减少）、业务必要性评估、用户告知/授权流程 |
