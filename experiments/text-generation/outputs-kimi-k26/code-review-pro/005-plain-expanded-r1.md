## 审查结果

### 总体判断：**高风险 — 不建议合并**

| 严重级别 | 类别 | 发现项 |
|---------|------|--------|
| **严重** | 权限绕过 | 后端 API 未校验管理员角色，非管理员可通过直接请求 API 导出任意用户数据 |
| **严重** | 隐私泄露 | 导出字段包含手机号、邮箱等 PII，无最小必要原则控制 |
| **高** | 数据泄露范围 | 无导出数量限制，可一次性拉取全量用户数据 |
| **高** | 审计缺失 | 无审计日志，无法追溯谁、何时、导出了什么数据 |
| **中** | 前端校验不可靠 | 仅前端隐藏按钮，属于"安全通过隐匿" |

---

### 主要发现与影响

**1. 垂直权限绕过（严重）**
- 攻击场景：普通用户直接构造请求 `GET /api/users/export` 即可触发导出
- 影响：任意用户可获取全平台用户敏感信息，违反数据最小访问原则

**2. 敏感数据批量泄露（严重）**
- 导出字段：姓名、手机号、邮箱、订单金额均属 PII/财务敏感信息
- 叠加无数量限制，构成大规模数据泄露风险

**3. 完全不可审计（高）**
- 无日志记录 = 无法检测泄露事件、无法满足合规要求（如 GDPR 审计权、等保）

---

### 测试建议

| 测试项 | 验证点 |
|--------|--------|
| 权限测试 | 普通用户/未登录用户直接请求 API，应返回 403 |
| 字段脱敏 | 确认是否所有导出字段均为业务必需，非必要字段应移除 |
| 数量限制 | 验证单次/单日导出上限，及分页机制 |
| 审计日志 | 验证日志包含：操作人、时间、IP、筛选条件、导出条数 |
| 前端绕过 | 确认仅隐藏按钮不足，后端必须强制校验 |

---

### 不确定性

- 不确定：该功能是否涉及跨境数据传输（影响合规基线）
- 不确定：现有用户体系的角色字段名称及校验位置（影响修复方案）
- 不确定：是否需对接数据脱敏平台或 DLP 系统

---

### 必须交接的跟进事项

| 负责人 | 事项 | 原因 |
|--------|------|------|
| **安全负责人** | 确认 PII 处理合规性、数据分类分级、是否需要额外加密或 DLP 管控 | 隐私法规风险 |
| **后端负责人** | 补全 RBAC 校验、审计日志、导出限流/配额、敏感字段脱敏 | 核心漏洞修复 |
| **产品负责人** | 确认导出字段范围是否最小必要、导出场景的业务合理性、是否需用户授权 | 业务合规与用户体验 |

---

### 阻塞性意见

在以下问题解决前**禁止合并**：
1. 后端补充角色校验（非前端）
2. 增加操作审计日志
3. 明确导出数量限制与字段范围（经产品确认）
