## PR 风险 Review

### 权限风险（高危）
- **垂直越权**：后端 API 未校验角色，非管理员可直接请求导出接口
- **前端校验绕过**：仅前端隐藏按钮，通过 curl/Postman 可直接调用
- **需安全团队确认**：当前用户体系是否支持后端角色校验的快速接入

### 隐私风险（高危）
- **敏感字段泄露**：手机号、邮箱、订单金额均属 PII，无脱敏或最小化原则
- **无导出范围限制**：可导出全量用户，存在数据批量泄露风险
- **需产品负责人确认**：导出字段是否合规（GDPR/个人信息保护法），是否需要用户授权

### 审计风险（中高危）
- **无操作日志**：无法追溯"谁、何时、导出了什么数据"
- **无异常监控**：无法检测高频/大量导出行为
- **需安全+后端确认**：审计日志字段标准及存储方案

### 测试/性能风险（中危）
- **无数量限制**：全量导出可能导致 DB 慢查询、内存 OOM、服务降级
- **无文件清理**：导出文件可能滞留服务器造成二次泄露
- **需后端确认**：是否加分页/流式导出/异步队列
