## 复核结论

**不可直接走合同。** 存在重大范围模糊、验收缺失及第三方集成风险，需多角色复核后方可进入合同流程。

## 已知事实

| 项目 | 内容 |
|:---|:---|
| 项目内容 | 集团级知识库 + 多 Agent 工作流 |
| 范围 | 总部知识库、5个业务Agent、对接OA/CRM/售后工单、试点后推广8家分公司 |
| 客户要求 | 单合同覆盖全阶段、报价含全部接口、验收标准后续再定 |

---

## 财务与合同风险

| 风险类别 | 具体风险 |
|:---|:---|
| **范围失控** | "试点后推广8家分公司"——试点成功标准、推广触发条件、额外费用均未约定 |
| **验收缺失** | "验收标准后续再定"= 无验收依据，无法判定履约完成，回款风险极高 |
| **第三方集成** | OA/CRM/售后工单系统涉及客户现有IT架构，接口复杂度、数据权限、改造责任不明 |
| **单合同全阶段** | 未分阶段付款，若试点失败或推广受阻，已收款项与违约责任难以界定 |
| **无限服务隐含** | "所有系统接口"未限定范围，存在免费无限扩展解读空间 |

| 5 | 付款里程碑是否与试点完成、验收通过挂钩？比例如何？ | 财务 |
| 6 | 是否涉及客户第三方供应商（如OA厂商）的授权或配合费用？ | 法务 + 交付 |

## 角色复核要求

| 角色 | 复核事项 |
|:---|:---|
| **售前** | 澄清"单合同全阶段"与客户谈判空间；确认试点→推广的商务条件；提供接口范围边界说明 |
| **交付** | 评估5个Agent + 8家分公司实施可行性、工期、资源；出具第三方集成技术风险评估 |
| **财务** | 审核付款节点与验收挂钩机制；测算范围蔓延成本；拒绝无验收标准的全额预付款 |
| **法务** | 审查单合同覆盖多阶段的法律风险；拟定"验收标准补充协议"模板；明确数据安全与第三方责任条款 |
| **风险/合规** | 评估集团级数据聚合的合规要求；审核知识库内容的知识产权归属 |

## 手off建议

- **当前状态**：材料不足，**禁止直接发起合同审批**
- **下一步动作**：由售前牵头组织交付、财务、法务召开范围澄清会，输出《范围边界确认书》及《验收标准草案》后，方可进入合同条款拟定
- **关键阻塞点**：验收标准未确定前，任何付款承诺均存在重大回款风险
