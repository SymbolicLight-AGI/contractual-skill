## 报价草案审查：售后知识库 Agent 试点

### 审查结论
**不可直接批准，需补充关键条款后方可进入合同签署。**

---

### 已知事实
| 项目 | 内容 |
|:---|:---|
| 项目 | 售后知识库 Agent 试点 |
| 报价 | 首期 120,000 元 |
| 付款节奏 | 合同签署后 50%（60,000 元），试运行通过后 50%（60,000 元） |
| 交付范围 | 知识库初始化、1 个客服 Agent、2 次远程培训 |

---

### 财务与合同风险

| 风险类别 | 具体风险 | 影响程度 |
|:---|:---|:---|
| **付款回收风险** | "试运行通过"标准未定义，第二笔 50% 款项的支付条件存在单方解释空间 | **高** |
| **成本超支风险** | 数据清洗边界不清，可能产生额外工作量；变更需求计费方式未约定，后续变更无法据实收费 | **高** |
| **第三方费用风险** | 未明确是否包含第三方系统接口费用，存在 120,000 元报价未覆盖外部成本的可能 | **中高** |
| **交付范围风险** | "知识库初始化"和"1 个客服 Agent"的具体功能边界、性能基线缺失 | **中** |
| **验收争议风险** | 无明确验收标准和流程，易引发付款拖延或拒付 | **高** |

---

### 待确认条款（禁止代为确认）

1. **试运行通过标准**
   - 具体量化指标是什么？（如：Agent 回答准确率、响应时效、用户满意度阈值）
   - 由谁测试、谁签字确认？

2. **数据清洗边界**
   - 原始数据提供方及质量责任？
   - 清洗工作量上限或超出部分的计费方式？

3. **第三方系统接口费用**
   - 是否包含在 120,000 元内？若涉及外部 API、授权费、集成开发费，由哪方承担？

4. **变更需求计费方式**
   - 变更的认定标准？
   - 工时单价或功能点单价？
   - 变更审批流程及时限？

5. **验收与付款衔接**
   - 试运行周期多长？
   - 试运行通过后付款的期限？（如：验收确认后 N 个工作日内）
   - 试运行未通过时的整改期限及费用承担？
