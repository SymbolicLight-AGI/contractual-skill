# 结论摘要

**不可直接走合同。** 该项目存在跨阶段覆盖、未定义验收标准、未拆分成本假设等重大缺口，需经多角色复核后方可推进。

---

# 已知事实

| 项目 | 内容 |
|:---|:---|
| 项目内容 | 集团级知识库 + 多 Agent 工作流 |
| 范围 | 集团总部知识库、5 个业务 Agent、对接 OA/CRM/售后工单系统 |
| 推广计划 | 试点后推广至 8 家分公司 |
| 客户要求 | 单合同覆盖全部阶段；报价含全部系统接口；验收标准后续再定 |

---

# 缺失信息

| 缺失项 | 影响 |
|:---|:---|
| 验收标准 | **关键阻塞项**：未定义交付完成边界，无法触发付款 |
| 各阶段交付物清单 | 试点 vs. 推广阶段的范围分界不清 |
| 系统接口的对接工作量与责任方 | 客户 IT 配合度、接口文档完备度未知 |
| 8 家分公司的实施顺序与时间表 | 影响收入确认和现金流预测 |
| 知识库的数据迁移范围与数据治理责任 | 可能产生额外成本 |
| 报价明细与成本假设 | 无法评估折扣空间与利润 |
| 客户信用与付款条件 | 影响账期和坏账风险 |
| 变更管理机制 | 范围蔓延无约束 |

---

# 财务与合同风险

| 风险类别 | 具体描述 | 等级 |
|:---|:---|:---|
| **跨阶段合同风险** | 单合同覆盖试点+推广，若试点失败或客户中途变更，已承诺义务无法收回 | 🔴 高 |
| **验收标准缺失** | "后续再定" = 付款触发条件悬空，收入确认和回款无保障 | 🔴 高 |
| **接口范围模糊** | "包含所有系统接口"未限定变更责任，客户新增系统或接口变更将转嫁成本 | 🟡 中高 |
| **推广阶段不确定** | 8 家分公司无时间表、无独立报价，成本假设失效 | 🟡 中高 |
| **无限服务风险** | Agent 调优、知识库运营可能隐含持续服务预期 | 🟡 中 |
| **数据责任不清** | 知识库内容准确性、数据迁移质量无责任边界 | 🟡 中 |

# 不得直接承诺事项

- ❌ 单合同覆盖全部阶段
- ❌ 验收标准后续补充（必须前置）
- ❌ 报价锁定全部接口（未限定变更范围）
- ❌ 试点自动过渡至推广阶段
- ❌ 任何折扣或账期优惠

# 待确认问题

1. **验收标准**：谁定义、何时定义、客户签字确认流程？
2. **阶段拆分**：试点与推广能否分签？若必须单合同，是否设置阶段退出条款？
3. **接口清单**：OA/CRM/售后工单的接口数量、复杂度、客户配合责任？
4. **推广约束**：8 家分公司是义务还是意向？有无最低承诺？
5. **付款里程碑**：与阶段交付挂钩的具体比例和时间？
6. **变更上限**：接口变更、Agent 增删的免费容忍度？
