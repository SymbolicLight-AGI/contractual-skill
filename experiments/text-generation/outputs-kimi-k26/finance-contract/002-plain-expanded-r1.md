## 审查结论

**不支持直接承诺"先做后付"**。在信用、范围、成本均缺失的情况下，该提议构成重大财务与合同风险，需升级审批。

## 财务与合同风险

| 风险类别 | 具体描述 |
|:---|:---|
| **回款风险** | 信用未知，"先做后付"无保障，可能形成坏账 |
| **成本失控** | 范围未定，实施、调研、第三方成本均无法锁定，存在无限投入敞口 |
| **定价失效** | 无范围即无报价，后期议价能力丧失 |
| **交付争议** | 无合同、无验收标准，"完成"难以界定，易触发拒付 |
| **第三方垫资** | 若第三方工具需预付，我方现金流承压 |

---

## 必须确认的问题

1. **客户信用**：是否已完成资信调查？有无历史交易记录或银行授信？
2. **项目范围**：何时完成正式方案？能否先签SOW或框架协议锁定边界？
3. **成本测算**：第三方工具是否必须？现场调研是否触发差旅及人天成本？
4. **付款结构**：客户"先做后付"的具体诉求是什么（如验收后付、分期、延期）？
5. **内部授权**：该付款模式是否需财务/法务/高管专项审批？

## 可替代方案（供内部评估）

| 方案 | 适用场景 |
|:---|:---|
| **分阶段付款** | 按里程碑（签约、调研完成、上线、验收）支付 |
| **小额预付款+验收尾款** | 降低前期投入，保留尾款激励 |
| **框架协议+单项目结算** | 先签框架锁定条款，具体项目逐笔约定 |
| **信用证/银行保函** | 若客户坚持后付，要求其提供银行信用支持 |
