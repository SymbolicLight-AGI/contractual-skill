## 判断结论

**不能直接走合同。** 存在多项重大风险，需多角色复核后方可推进。

---

## 必须复核事项

### 交付 / 售前复核
| 风险点 | 说明 |
|--------|------|
| 范围模糊 | "5个业务Agent"未定义功能边界，易引发范围蔓延 |
| 试点→推广机制不明 | "试点后推广到8家分公司"未约定试点成功标准、推广触发条件及费用是否另计 |
| 系统接口范围 | "所有系统接口"未明确OA/CRM/售后工单的接口数量、复杂度及对接责任方 |
| 验收标准缺失 | "验收标准后续再定"构成重大交付风险，无验收依据 |

### 法务复核
| 风险点 | 说明 |
|--------|------|
| 合同覆盖阶段不清 | "全部阶段"未界定，是否包含试点、推广、运维？ |
| 验收条款缺失 | 验收标准、验收流程、整改期限均未约定，存在无法闭环风险 |
| 集团-分公司法律关系 | 8家分公司是否为合同主体？权利义务如何约束？ |

### 财务复核
| 风险点 | 说明 |
|--------|------|
| 付款里程碑缺失 | 无验收标准则无法绑定付款节点，回款风险极高 |
| 推广阶段费用 | 8家分公司推广是否另行收费？总价是否锁定？ |
| 折扣与定价 | 报价结构及折扣是否经审批？ |

### 风险审查
| 风险点 | 说明 |
|--------|------|
| 无限服务风险 | 范围、阶段、验收均开放，存在重大履约风险 |
| 回款周期不可控 | 验收标准未定→验收时间不可控→应收账款逾期风险 |

---

## 建议下一步

1. **暂停合同签署**，要求客户或售前补充：5个Agent功能清单、试点成功标准、推广触发机制、验收标准草案
2. **交付负责人**：确认技术可行性及人天估算
3. **法务**：审核集团-分公司法律架构及验收条款模板
4. **财务**：确认报价拆分、付款节点及推广阶段定价策略
5. **风险审查**：评估整体履约风险，决定是否需分期签约或附加条件
