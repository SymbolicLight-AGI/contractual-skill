# 结论摘要

**不可直接走合同。** 本项目存在跨阶段覆盖、未定义验收标准、未拆分交付范围等重大缺失，需经多角色复核后方可推进。

---

# 已知事实

| 项目 | 内容 |
|:---|:---|
| 项目名称 | 集团级知识库和多 Agent 工作流 |
| 交付范围 | 集团总部知识库 + 5 个业务 Agent + 对接 OA/CRM/售后工单系统 |
| 推广计划 | 试点后推广至 8 家分公司 |
| 客户要求 | 单份合同覆盖全部阶段；报价含所有系统接口；验收标准后续再定 |

---

# 缺失信息

| 缺失项 | 影响 |
|:---|:---|
| 验收标准 | **关键缺失**——无法判定交付完成，构成收款触发障碍 |
| 各阶段交付物清单 | 试点 vs. 推广阶段的范围边界不明 |
| 系统接口的技术规格与数据 readiness | 影响工时估算与接口费用定价 |
| 8 家分公司的实施顺序、时间、责任方 | 跨阶段合同范围膨胀风险 |
| 报价明细及成本假设 | 无法评估折扣空间与利润 |
| 客户信用及付款里程碑 | 现金流风险未知 |
| 变更控制机制 | 范围蔓延无约束 |
| SLA（响应时间、可用性、故障处理） | 运维责任未界定 |

---

# 财务与合同风险

| 风险类别 | 具体描述 |
|:---|:---|
| **跨阶段合同风险** | 单合同覆盖试点+8家分公司推广，属于"cross-phase contract"，一旦前期交付受阻，整体收款链断裂 |
| **验收标准缺失风险** | "后续再定"等于未约定付款触发条件，形成**无限期应收账款** |
| **范围膨胀风险** | 5 个 Agent + 3 套系统接口 × 8 家分公司，实际工作量可能数倍于总部试点，报价基础存疑 |
| **接口成本风险** | "所有系统接口"未限定变更次数、数据格式责任方、第三方配合义务 |
| **现金流风险** | 无付款里程碑约定，无法评估 receivables 周期 |

---

# 不得直接承诺事项

- ❌ 单份合同覆盖全部阶段
- ❌ 报价包含"所有"系统接口（未限定边界）
- ❌ 验收标准后续再定
- ❌ 试点到推广的自动扩展机制
- ❌ 任何折扣或账期优惠

# 待确认问题

1. 验收标准能否在签约前确定？若不能，是否接受分阶段签约？
2. 8 家分公司的推广是否为可选权利还是必做义务？是否单独计价？
3. 系统接口的现有文档、数据质量、第三方配合度如何？
4. 客户是否接受"试点单独签约，推广另行协商"的模式？
5. 报价是否基于"人天"还是"固定总价"？成本假设是否经交付 PM 评审？

# 交接建议

| 角色 | 必须复核事项 |
|:---|:---|
| **售前** | 拆分试点与推广阶段的报价策略；明确"所有接口"的边界与例外；客户信用初评 |

| **风险审查** | 跨阶段合同结构、无限范围风险、高让步评估（如客户坚持单合同覆盖全部） |
