## 合同与财务风险识别

### 一、合同风险

| 风险点 | 具体说明 |
|--------|---------|
| **试运行通过标准缺失** | "试运行通过"无定义，双方理解易分歧，尾款支付条件存在重大不确定性 |
| **数据清洗边界不清** | 知识库初始化范围模糊，可能引发工作量争议或无限扩大 |
| **变更需求计费空白** | 无变更控制机制，项目易范围蔓延，成本不可控 |
| **第三方接口费用未约定** | 若涉及系统对接，费用承担主体不明，易产生额外支出 |
| **培训交付标准缺失** | "2次远程培训"无时长、人数、效果验收标准 |

### 二、财务风险

| 风险点 | 具体说明 |
|--------|---------|
| **回款周期不确定** | 试运行时长未限定，尾款60000元回收时间无法预估 |
| **现金流风险** | 首期50%（60000元）可能不足以覆盖前期投入成本 |
| **坏账风险** | 试运行标准缺失导致尾款争议概率高 |

## 待确认条款清单

1. **试运行通过标准**（量化指标：响应准确率/客户满意度/无故障运行时长等）
2. **试运行期限上限**（最长不超过多少工作日）
3. **数据清洗范围**（数据源数量、格式、清洗深度、验收标准）
4. **第三方系统接口**（是否涉及、接口清单、费用承担方）
5. **变更需求计价规则**（人天单价/功能点评分/变更审批流程）
6. **培训具体方案**（单次时长、参训人数、课件交付物、考核标准）
7. **知识库初始化交付标准**（条目量、更新机制、准确率要求）
8. **逾期付款/验收的违约责任**（双方对等条款）
