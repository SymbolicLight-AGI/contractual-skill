## 审查结论

**不支持直接承诺"先做后付"**。在信用状况、项目范围、成本边界均未确认的情况下，该付款方式存在重大财务暴露，需完成多部门评审后方可评估可行性。

## 财务与合同风险

| 风险类别 | 具体暴露 |
|:---|:---|
| **回款风险** | 信用未知，"先做后付"无追索保障，可能形成坏账 |
| **成本失控** | 范围未定义，现场调研、第三方工具成本可能无限膨胀 |
| **交付争议** | 无正式方案及验收标准，"完成"难以界定，付款触发条件缺失 |
| **资金占用** | 实施周期内全垫资，影响现金流 |
| **合同效力** | 口头承诺可能被认定为要约，后续谈判陷入被动 |

---

## 必须确认的问题

**关于客户**
- 该客户是否在我方CRM中有历史交易记录？账期表现如何？
- 是否可获取对方近年度财报或银行授信证明？

**关于项目**
- "先做后付"的具体含义：是全部验收后付款，还是分阶段后付？
- 客户是否接受里程碑节点与付款挂钩？

**关于成本**
- 第三方工具成本是否必须由我方垫付？有无预付款或信用账期？
- 现场调研的人天预估及差旅预算上限？

## 可替代付款方案（供谈判参考）

| 方案 | 要点 |
|:---|:---|
| **预付款+里程碑** | 合同签署后支付30%，关键里程碑各付30%，验收后10% |
| **信用证/银行保函** | 若客户坚持后付，要求其开立不可撤销信用证或银行付款保函 |
| **第三方工具代采** | 工具费用客户直付或客户自行采购，我方仅收实施费 |
| **有限试点后付** | 极小范围POC，明确人天上限与验收标准，超支即停止 |
