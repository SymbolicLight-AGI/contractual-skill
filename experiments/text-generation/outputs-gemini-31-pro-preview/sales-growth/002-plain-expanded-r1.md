**Customer Profile**
某区域设备经销商。

**Known Facts**
- 客户口头表达了对产品的初步兴趣：“你们这个 AI Agent 看起来不错，下周给我一个方案”。
- 客户曾询问过“能不能一周上线”。
- 目前无会议纪要。
- 缺失关键信息：未说明具体业务场景、预算、决策人、上线时间、数据来源和系统环境。

**Inferred Opportunities**
- 客户可能在寻找提升运营效率或客户交互的自动化工具。
- **不确定性标注**：目前客户仅停留在模糊的兴趣表达阶段，缺乏实质性业务痛点支撑，**不能视为确定的购买意向**。

**Risk Signals**
- **交付时间风险**：客户曾询问“一周上线”，属于要求快速上线的高风险信号，极易导致交付延期或预期管理失控。
- **方案范围风险**：在完全没有明确业务场景、数据来源和系统环境的情况下要求“下周给方案”，存在盲目承诺固定范围和无效售前投入的风险。
- **商机质量风险**：预算、决策人、真实需求均不明确，盲目作为重点推进可能浪费销售资源。

**Recommended Next Steps**
- **机会判断**：当前信息严重不足，暂不建议直接将其作为重点商机推进，也不建议在下周直接提供定制化解决方案。
- **行动建议**：向客户提议在下周出具方案前，先进行一次简短的需求沟通会议（Discovery Call），以对齐业务目标。可先提供标准化的产品介绍或行业通用案例作为回应。
- **待确认问题（Missing Information）**：
  1. 客户希望 AI Agent 解决的具体业务场景或核心痛点是什么？
  2. 期望“一周上线”的背后是否有特定的业务节点或外部压力？
  3. 现有的系统环境是什么？AI Agent 需要对接哪些数据源？
  4. 该项目的预算规划和内部决策流程是怎样的？

**Handoff Suggestions**
- **售前与风险审查移交**：在收集到具体的业务场景和系统环境信息后，必须将“一周上线”的可行性评估、解决方案范围及系统集成需求移交至**售前团队（Presales）**和**风险审查团队（Risk Review）**进行评估。销售人员不得向客户承诺最终的实施范围、集成可行性或交付时间表。
