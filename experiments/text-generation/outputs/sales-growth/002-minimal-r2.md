## 机会判断

当前不建议直接判定为重点推进机会，只能判断为“早期兴趣线索，需快速做资格确认”。  
客户仅表达“AI Agent 看起来不错，下周给方案”，但尚未明确业务场景、预算、决策链、上线目标、数据与系统条件，不能将其视为确定购买意向。

可采取“轻量跟进 + 快速资格评估”的方式推进：先安排一次需求澄清会议，再决定是否投入售前资源做正式方案。

## 可能痛点

当前无法确认真实痛点，仅可做谨慎假设，需进一步验证：

1. 可能希望通过 AI Agent 提升销售、客服、售后或内部运营效率。
2. 可能对自动化处理客户咨询、设备资料查询、报价辅助、售后跟进等场景感兴趣。
3. 曾询问“一周上线”，可能存在快速试用、快速展示或短期业务压力。
4. 也可能只是泛泛了解 AI Agent，并未形成明确采购需求。

以上均为待确认假设，不能作为确定需求处理。

## 成交路径判断

当前机会处于非常早期阶段，尚未进入明确商机阶段。

建议成交路径：

1. **需求澄清**
   - 明确客户想解决的具体业务问题。
   - 确认 AI Agent 用于销售、客服、售后、知识库、内部流程还是其他场景。

2. **资格确认**
   - 确认预算范围。
   - 确认决策人、使用部门、技术接口人。
   - 确认是否有明确上线目标或试点计划。

3. **可行性评估**
   - 了解客户数据来源、知识库资料、系统环境。
   - 判断是否涉及 CRM、ERP、微信、官网、小程序、设备系统等集成。

4. **方案分层**
   - 如需求清晰，可提供初步方案方向。
   - 如需求不清，不建议直接承诺完整方案、价格、周期或上线时间。

5. **售前与风险评审**
   - 涉及方案范围、交付周期、系统集成、价格假设时，应交由售前和风险评审确认。

## 待确认问题

建议下次沟通优先确认以下问题：

1. 您希望 AI Agent 主要解决哪个业务场景？
   - 销售获客？
   - 客户咨询？
   - 售后服务？
   - 设备知识问答？
   - 内部员工助手？
   - 报价或订单辅助？

2. 当前这个场景的主要问题是什么？
   - 人工响应慢？
   - 客户流失？
   - 售后压力大？
   - 产品资料复杂？
   - 经销商管理难？
   - 销售转化率低？

3. 是否已有明确的试点部门或负责人？

4. 目前是否有预算规划？

5. 决策流程是怎样的？
   - 谁使用？
   - 谁评估？
   - 谁拍板？
   - 是否需要老板、业务负责人、IT 或财务参与？

6. 客户希望“下周给方案”是用于内部汇报、预算申请、项目立项，还是只是了解产品？

7. 曾提到“一周上线”，具体是指：
   - 一周内做演示？
   - 一周内试点？
   - 一周内正式上线？
   - 还是希望快速看到效果？

8. 客户现有数据在哪里？
   - 产品资料、设备手册、报价表、FAQ、客户记录、售后工单等是否完整？

9. 是否需要对接现有系统？
   - CRM、ERP、企微、公众号、小程序、官网、客服系统、工单系统等。

10. 对效果是否有明确指标？
   - 响应速度、转化率、人工节省、客户满意度、售后处理时长等。

## 下一步建议

1. **不要直接提交完整正式方案**
   - 当前信息不足，直接出方案容易偏离真实需求，也可能造成交付和价格风险。

2. **先约 30-60 分钟需求澄清会**
   - 目标是确认场景、价值、决策链、预算和上线预期。

3. **提供一页式初步方向，而不是承诺型方案**
   - 可给客户一个“AI Agent 可选应用场景清单 + 需确认问题 + 试点思路”。
   - 不承诺最终价格、交付周期、实施范围和效果。

4. **判断是否值得重点推进的标准**
   - 若客户能明确核心场景、业务负责人、试点目标、预算意向和决策路径，可升级为重点商机。
   - 若客户仍停留在泛泛兴趣，且坚持“一周上线”但无法提供数据、场景和负责人，则不建议重点投入。

5. **引入售前前置评估**
   - 一旦涉及方案范围、系统对接、上线时间或报价，应交由售前进行可行性评估，并进入风险 review。

## 风险信号

1. **需求不清**
   - 客户未说明具体业务场景，无法判断 AI Agent 是否匹配真实问题。

2. **预算不明**
   - 未确认预算，可能只是了解产品或比价。

3. **决策链不明**
   - 未确认决策人、使用部门和技术负责人，推进路径不清晰。

4. **上线预期风险**
   - 客户曾问“能不能一周上线”，属于快速上线风险信号。
   - 需明确是一周演示、一周试点，还是一周正式上线，不能直接承诺。

5. **数据与系统环境不明**
   - 未确认知识库、业务数据、系统接口和数据质量，可能影响方案可行性。

6. **方案范围风险**
   - 客户要求“下周给方案”，但范围尚未明确，若直接给完整方案，容易引发范围、价格和交付争议。

7. **购买意向不确定**
   - 当前只有兴趣表达，不能视为确定采购意向。
