## 三、推断机会

1. **售后知识库建设机会明确**
   - 客户已有大量分散知识资产，存在整合需求。
   - 可围绕“微信群/Excel/老系统知识统一沉淀”设计解决方案。

2. **售后 Agent 应用场景较匹配**
   - 重复问答多，适合通过 Agent 辅助一线售后人员快速查询标准答案。
   - 可优先切入常见问题问答、工单经验推荐、标准处理流程推荐等场景。

3. **新员工培训场景具备价值**
   - 客户明确提到培训周期长，可将知识库和 Agent 包装为“新人训练助手”或“售后经验陪练工具”。

4. **售后负责人可能是业务推动者**
   - 售后负责人积极，可能成为内部业务 sponsor。
   - 后续应重点协助其量化业务价值，例如减少重复问答、缩短新人上手周期、提升工单处理效率。

5. **信息化负责人是关键影响人**
   - 其关注集成、安全、权限，说明方案需要具备较强的 IT 可控性说明。
   - 如果能提前提供架构、安全、权限和集成方案，有助于降低阻力。

## 四、风险信号

1. **预算不明确**
   - 当前无法判断客户是否已有正式项目预算。
   - 可能仍处于早期调研阶段。

2. **决策时间表不明确**
   - 尚不清楚客户是否有明确上线节点、采购计划或内部立项节奏。

3. **竞品情况不明确**
   - 不确定客户是否已接触其他知识库、客服系统、AI Agent 或大模型厂商。
   - 需要尽快识别竞争态势。

4. **数据安全和权限要求可能影响推进**
   - 客户对数据安全和权限隔离敏感，若不能充分说明权限模型、数据边界、部署方式，可能影响信任建立。

5. **系统集成风险被重点关注**
   - 老系统、Excel、微信群等数据来源复杂，若集成方案不清晰，可能导致信息化负责人产生顾虑。

6. **业务价值尚需量化**
   - 虽然痛点明确，但目前尚未看到客户给出具体指标，例如重复问答数量、新人培训周期、工单处理时长等。
   - 若不量化 ROI，后续推动预算可能困难。

## 五、下一步动作

1. **安排一次业务场景深挖会议**
   - 重点邀请售后负责人、信息化负责人参加。
   - 目标是确认高频售后问题、知识来源、工单流程和优先落地场景。

2. **补充确认项目关键信息**
   - 是否已有预算或预算范围。
   - 是否已有立项计划。
   - 期望上线时间。
   - 是否正在对比竞品。
   - 谁是最终决策人、采购负责人和技术评审人。

3. **准备针对性方案**
   - 方案重点突出：
     - 售后知识统一归集；
     - 标准问答沉淀；
     - 工单经验复用；
     - 新员工培训辅助；
     - Agent 辅助售后问答；
     - 权限隔离；
     - 数据安全；
     - 与老系统集成方式。

4. **提供安全与权限说明材料**
   - 包括：
     - 用户权限分层；
     - 部门/角色隔离；
     - 数据访问控制；
     - 日志审计；
     - 敏感数据处理机制；
     - 可选部署方式，如客户有要求需进一步确认。

5. **设计小范围试点方案**
   - 建议从售后高频问答或典型工单经验库开始。
   - 试点范围可控制在一个售后团队、一个产品线或一类高频问题。
   - 试点目标建议包括：
     - 重复问答减少；
     - 新人查询效率提升；
     - 工单处理经验复用；
     - 知识沉淀数量。

6. **协助客户定义评估指标**
   - 可建议客户记录：
     - 当前重复问题数量；
     - 新员工独立处理问题所需时间；
     - 售后工单平均处理时长；
     - 知识查询命中率；
     - Agent 答案采纳率；
     - 人工转接率。

7. **推动售后负责人内部牵引**
   - 与售后负责人共同梳理业务痛点和收益。
   - 帮助其形成内部汇报材料，争取信息化和管理层支持。
