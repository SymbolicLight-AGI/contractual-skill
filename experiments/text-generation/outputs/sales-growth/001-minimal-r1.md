## 3. 推断机会

- **售后知识库建设机会**：客户已有明确知识分散、重复问答、新人培训周期长等痛点，适合从售后知识沉淀场景切入。
- **售后 Agent 辅助问答机会**：客户存在大量标准问答和工单经验，具备构建售后问答助手的潜在基础。
- **知识治理机会**：客户资料分布在微信群、Excel、老系统，可能需要梳理知识来源、分类、权限和更新机制。
- **权限与安全能力展示机会**：客户对数据安全和权限隔离敏感，后续沟通中应重点回应权限模型、数据边界、访问控制、审计等问题。
- **售后部门可能是业务推动方**：售后负责人积极，可作为需求共创和内部推动的关键联系人。
- **信息化部门是关键把关方**：信息化负责人关注集成风险，后续需尽早纳入技术沟通，避免后期因安全、集成、架构问题阻塞。

## 4. 风险信号

- **预算未明确**：无法判断客户采购能力、项目优先级和商业推进节奏。
- **决策时间表未明确**：当前机会阶段不清晰，可能停留在调研或初步评估阶段。
- **竞品情况未知**：不确定客户是否已有对比对象，也不清楚我方差异化定位。
- **系统集成风险**：客户已有老系统，且信息化负责人明确关注集成风险；涉及系统对接、数据同步、权限打通时需谨慎评估。
- **数据安全与权限隔离敏感**：若方案不能清晰说明安全边界和权限控制，可能影响推进。
- **知识来源复杂**：微信群、Excel、老系统中的知识质量、结构化程度、历史数据可用性不确定，可能影响后续方案范围和实施复杂度。
- **项目范围存在扩大的可能**：若客户期望覆盖所有知识来源、所有系统集成、快速上线或固定低价交付，应作为风险处理。
- **生产计划经理诉求不明**：生产计划角色参会但需求未显性表达，可能存在跨部门扩展诉求，也可能影响项目边界。

## 5. 下一步动作

1. **确认业务目标与优先场景**
   - 与售后负责人进一步确认最急迫的场景：
     - 重复问答减少；
     - 新员工培训；
     - 工单经验沉淀；
     - 售后人员辅助查询；
     - 客户自助服务是否需要覆盖。
   - 明确第一阶段希望解决的核心问题，避免范围过大。

2. **补充决策与采购信息**
   - 询问客户是否已有预算规划。
   - 了解决策链条：业务部门、信息化部门、采购、管理层分别承担什么角色。
   - 确认是否存在目标上线时间或内部节点。
   - 了解是否已接触或比较其他厂商。

3. **组织技术与安全专题沟通**
   - 针对信息化负责人关注点，安排一次安全与集成专题交流。
   - 重点收集：
     - 老系统类型；
     - 数据存储位置；
     - 是否需要与工单系统、账号体系、权限体系对接；
     - 数据安全要求；
     - 权限隔离规则；
     - 是否有私有化、内网、审计等要求。

4. **开展售后知识现状盘点**
   - 与售后负责人共同梳理知识来源：
     - 微信群历史问答；
     - Excel 文档；
     - 老系统工单；
     - 标准服务手册；
     - 常见故障处理经验。
   - 初步判断哪些内容适合沉淀为标准知识，哪些需要清洗、审核或结构化。

5. **设计小范围验证路径**
   - 建议先围绕一个明确售后场景进行验证，例如常见问题问答或典型工单经验检索。
   - 验证目标应聚焦在业务可感知指标，但不承诺最终效果。
   - 验证范围、数据边界、系统对接需求需交由售前进一步评估。

6. **推动形成客户内部共识**
   - 售后负责人侧重业务价值。
   - 信息化负责人侧重安全、权限、集成可控。
   - 如生产计划经理有相关诉求，需明确是否纳入本阶段，避免扩大项目范围。

## 6. 需要交接的事项

### 交接给售前/解决方案团队

- 客户正在评估企业知识库和售后服务 Agent。
- 当前核心痛点：
  - 售后知识分散；
  - 重复问答多；
  - 新员工培训周期长；
  - 工单经验未充分沉淀；
  - 数据安全和权限隔离敏感。
- 需重点评估：
  - 知识库建设范围；
  - 售后 Agent 可覆盖场景；
  - 知识清洗与治理工作量；
  - 老系统集成可行性；
  - 权限隔离方案；
  - 数据安全方案；
  - 是否需要账号体系或工单系统对接。
- 不确定项：
  - 客户预算；
  - 决策周期；
  - 竞品情况；
  - 老系统类型和接口能力；
  - 数据量和数据质量；
  - 生产计划经理具体诉求。

### 交接给风险评审/项目管理相关团队

- 客户关注系统集成风险，后续如涉及老系统对接需提前评估。
- 客户对数据安全和权限隔离敏感，需避免在未评估前承诺安全能力边界。
- 若客户提出全系统集成、快速上线、固定低价范围或保证效果，需进入风险评审。
- 项目范围、价格、交付周期、实施边界均需售前和风险团队评估后再对外沟通。

### 销售跟进重点

- 优先绑定售后负责人，推动业务场景明确化。
- 同步稳住信息化负责人，降低其对安全和集成风险的顾虑。
- 尽快补齐预算、决策链、时间表、竞品情况四类关键信息。
- 避免直接承诺价格、交付周期、上线时间和最终效果。
