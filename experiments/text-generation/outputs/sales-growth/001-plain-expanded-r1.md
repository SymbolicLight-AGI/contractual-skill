## 2. 客户明确事实

客户已明确表达：

1. **售后知识分散**
   - 当前知识分布在微信群、Excel 和老系统中。
   - 存在知识查找困难、信息不统一的可能性。

2. **重复问答较多**
   - 售后团队存在大量重复性问题处理。
   - 客户希望减少重复沟通和重复解答成本。

3. **新员工培训周期长**
   - 客户希望将标准问答和工单经验沉淀下来。
   - 目标是帮助新员工更快掌握售后知识。

4. **重视数据安全和权限隔离**
   - 客户对数据访问控制、权限边界较敏感。
   - 后续方案中需要重点说明安全机制和权限设计。

## 3. 推断机会

以下为基于拜访记录的销售推断，尚未得到客户确认：

1. **企业知识库建设机会**
   - 客户已有多源知识沉淀需求，可能需要统一知识采集、整理、审核、发布和检索能力。

2. **售后服务 Agent 试点机会**
   - 重复问答多、新员工培训周期长，可能适合从售后标准问答、常见故障处理、工单经验检索等场景切入。

3. **知识治理与流程规范机会**
   - 客户知识分散在微信群、Excel 和老系统，可能需要知识分类、标签体系、版本管理、权限管理和内容审核机制。

4. **集成型方案机会**
   - 由于涉及老系统和工单经验，后续可能存在与既有系统对接的需求，但具体系统范围和可行性尚不明确。

5. **分阶段落地机会**
   - 可建议先选取售后高频问题或典型产品线做小范围验证，再评估扩展到更多知识源和业务部门。

## 4. 风险信号

1. **预算未知**
   - 客户未明确预算范围，暂不能判断采购成熟度和商业推进节奏。

2. **决策时间表未知**
   - 客户未说明立项、评估、采购或上线时间要求，需进一步确认项目紧迫性。

3. **竞品情况未知**
   - 客户未明确是否已比较其他厂商，存在竞品介入或内部自研方案的可能。

4. **信息化负责人关注集成风险**
   - 后续若涉及微信群、Excel、老系统、工单系统等多源数据接入，可能带来数据清洗、接口权限、系统兼容、责任边界等风险。

5. **数据安全和权限隔离要求高**
   - 方案不能简单承诺“完全安全”或“无风险接入”，需要由售前/安全/技术团队评估具体权限模型、部署方式和数据边界。

6. **需求范围可能扩大**
   - 从知识库延伸到售后 Agent、工单经验、培训、系统集成，存在范围蔓延风险，需要尽早界定一期目标和验收口径。

## 5. 推荐下一步动作

1. **安排需求澄清会**
   - 参会建议：售后负责人、信息化负责人、关键售后主管、我方销售、售前顾问。
   - 重点确认：
     - 当前售后知识量级和主要类型。
     - 高频问答场景。
     - 工单经验沉淀方式。
     - 老系统类型及是否支持接口。
     - 权限隔离要求。
     - 是否有试点部门或试点产品线。

2. **推动售后场景优先试点**
   - 建议以售后负责人积极关注的场景切入。
   - 可讨论一期聚焦：
     - 高频标准问答。
     - 常见问题检索。
     - 工单经验查询。
     - 新员工培训知识库。
   - 不承诺最终范围和上线周期，需售前评估后确定。

3. **准备安全与权限专题沟通**
   - 针对信息化负责人关注点，提前准备：
     - 数据权限隔离思路。
     - 角色权限模型。
     - 知识访问控制机制。
     - 部署方式选项。
     - 日志审计和数据边界说明。
   - 具体安全方案需技术/安全团队确认。

4. **补齐商务信息**
   - 下次沟通需确认：
     - 是否已有预算。
     - 项目是否已立项。
     - 采购流程和关键决策人。
     - 期望决策时间。
     - 是否已有竞品或内部方案。
     - 客户更关注降本、提效、培训还是服务质量提升。

5. **识别决策链**
   - 当前参会角色覆盖业务和信息化，但尚不清楚最终决策人、采购部门和财务审批角色。
   - 建议销售进一步确认决策链和影响人。

## 6. 需要交接的事项

### 交接给售前团队

1. **售后知识库与 Agent 场景方案**
   - 围绕售后标准问答、工单经验沉淀、新员工培训设计初步方案框架。
   - 明确可做能力、需评估能力和不确定边界。

2. **系统集成初步评估**
   - 需确认客户老系统类型、接口能力、数据格式、权限机制。
   - 不提前承诺集成可行性和实施周期。

3. **数据安全与权限隔离方案**
   - 准备权限分层、数据隔离、访问控制、审计等说明。
   - 如涉及敏感数据或私有化部署，需要进一步技术评估。

### 交接给销售负责人/商机管理

1. **商机阶段判断**
   - 当前属于需求探索/方案评估阶段，尚不能判断为明确采购。
   - 预算、时间表、竞品情况和决策链均需补齐。

2. **关键推进对象**
   - 售后负责人：业务痛点明确且态度积极，可作为业务推动人。
   - 信息化负责人：关注安全与集成，是技术可行性和风险控制关键人。

3. **风险复核**
   - 若客户后续提出固定价格、明确上线时间、广泛系统集成或效果保证，需先进入售前评估和风险 review，不应由销售直接承诺。
