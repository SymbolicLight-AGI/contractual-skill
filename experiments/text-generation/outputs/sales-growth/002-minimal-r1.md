## 机会判断：暂不建议列为重点推进机会

当前仅能判断为“初步兴趣线索”，不能判断为明确购买意向或高优先级商机。  
原因是客户只表达“AI Agent 看起来不错，下周给我一个方案”，但缺少业务场景、预算、决策链、上线目标、数据与系统环境等关键信息，且曾询问“能不能一周上线”，存在对交付周期和范围预期不清的风险。

建议：先作为需快速澄清的培育型机会推进，不宜直接投入大量方案、报价或承诺交付。

## 可能痛点

当前无法确认真实痛点，仅可推测存在以下可能方向，需进一步验证：

1. 可能希望通过 AI Agent 提升销售、客服或售后效率。
2. 可能关注设备经销业务中的客户跟进、询价响应、售后问答或内部知识检索。
3. 可能对快速上线有期待，但是否有清晰业务目标未知。
4. 可能只是对 AI 概念感兴趣，尚未形成明确项目需求。

以上均为待确认信息，不能作为确定需求处理。

## 待确认问题

### 业务场景
1. 希望 AI Agent 解决哪个具体业务问题？
2. 使用对象是谁：销售、客服、售后、管理层，还是客户？
3. 当前流程中最耗时或最影响业绩的环节是什么？
4. 是否已有明确的业务指标，例如提升响应效率、减少人工成本、提升线索转化等？

### 决策与预算
1. 这次需求是业务部门调研，还是公司已有立项？
2. 谁是最终决策人？谁参与评估？
3. 是否有预算范围或采购计划？
4. 是否有竞品或其他方案在比较？

### 数据与系统
1. AI Agent 需要接入哪些数据？
2. 是否已有客户资料、产品资料、报价资料、售后知识库等可用内容？
3. 是否需要对接 CRM、ERP、工单系统、企业微信、微信公众号或其他系统？
4. 数据权限、安全要求和合规要求是什么？

### 时间与交付预期
1. 客户提到“一周上线”是希望试用、演示，还是正式生产上线？
2. 期望下周拿到的是初步方案、演示方案，还是可执行项目方案？
3. 是否接受先做场景澄清和小范围验证？

## 成交路径建议

1. **先做需求澄清，不直接给完整方案**
   - 安排一次 30-60 分钟需求访谈。
   - 明确业务场景、目标用户、现有流程、数据条件、系统环境和决策链。

2. **将“下周方案”定位为初步方向方案**
   - 不承诺最终范围、价格、上线时间或效果。
   - 方案应以“假设前提 + 待确认项 + 分阶段建议”为主。

3. **优先识别是否存在真实业务价值**
   - 如果客户能明确高频、刚需、可量化场景，可进入重点跟进。
   - 如果客户只是泛泛了解 AI Agent，继续作为培育线索，不建议重点投入。

4. **需要售前介入**
   - 涉及 AI Agent 能力边界、系统集成、数据接入、上线周期等问题，应交由售前评估。
   - 不由销售直接承诺实施范围和交付时间。

## 下一步建议

1. 立即约客户进行一次需求澄清会。
2. 会前发送简短问题清单，要求客户明确：
   - 目标场景
   - 使用对象
   - 现有系统
   - 数据来源
   - 期望上线形式
   - 决策人和预算情况
3. 下周输出“初步方案框架”，而不是完整项目方案。
4. 若客户坚持“一周上线”或要求固定低价全范围交付，应进入风险评审。
5. 根据澄清结果再判断是否升级为重点商机。

## 风险信号

1. **需求不清**
   - 客户未说明具体业务场景，当前无法判断真实需求。

2. **购买意向不明确**
   - “看起来不错”只能代表兴趣，不能视为确定采购意向。

3. **预算未知**
   - 未确认预算，可能只是调研或比价。

4. **决策链未知**
   - 不清楚是否接触到决策人，推进路径不明。

5. **交付预期风险**
   - 客户曾询问“能不能一周上线”，存在对交付周期预期过快的风险。

6. **系统与数据风险**
   - 数据来源、系统环境、是否需要集成均未确认。
   - 如涉及多系统集成，应作为风险交由售前和风险评审处理。

7. **方案承诺风险**
   - 当前不应承诺最终方案范围、价格、上线周期或效果。
   - 任何实施范围、报价和交付假设都应交由售前与风险评审确认。
