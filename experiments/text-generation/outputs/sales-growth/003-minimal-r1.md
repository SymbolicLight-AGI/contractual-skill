## 客户痛点

1. 生产异常知识可能分散，缺少统一沉淀和复用机制。  
2. 质检问题可能依赖人工经验，查询和响应效率有提升空间。  
3. 希望通过知识库和 Agent 提升生产、质检相关问题的处理效率。  
4. 可能希望快速看到效果，但实际交付复杂度取决于数据、系统和部署条件。  

---

## 成交路径建议

1. 先确认核心场景：生产异常知识库、质检问答 Agent 的优先级和首期目标。  
2. 售前介入调研：明确数据来源、数据规模、系统对接、部署方式和验收标准。  
3. 输出分阶段方案：建议先做 MVP 或一期核心场景验证，再扩展系统对接和复杂能力。  
4. 风控评估：涉及价格、范围、上线时间、系统集成、现场部署等内容，需统一评审后再对外承诺。  
5. 正式报价与排期：在需求边界清晰后，由售前、交付和商务共同确认。

## 下一步动作

1. 约客户进行一次需求澄清会。  
2. 请售前参与，梳理业务流程、知识来源和问答场景。  
3. 收集客户现有资料样例，例如质检标准、异常处理记录、知识文档、历史问答等。  
4. 判断是否需要对接 MES、ERP、质检系统或生产设备实时数据。  
5. 明确部署方式：云端、私有化、现场部署或混合部署。  
6. 明确验收标准：问答准确率、知识覆盖范围、异常场景数量、用户角色、使用频次等。  
7. 在售前和风控确认后，再输出预算、周期和实施范围。  

---

## 需要售前和风控确认的问题

1. 数据规模有多大？包括文档数量、格式、历史记录量、更新频率。  
2. 知识来源有哪些？是否包括质检标准、SOP、异常处理单、设备日志、人工经验文档等。  
3. 是否需要对接 MES、ERP 或质检系统？如需要，对接方式和接口条件是什么。  
4. 是否涉及生产设备实时数据？是否需要实时告警或异常诊断。  
5. 是否需要现场部署或私有化部署？客户 IT 环境和安全要求是什么。  
6. 首期范围是否只做知识库和问答，还是包含流程闭环、工单、报表、权限、审计等能力。  
7. Agent 的目标用户是谁？质检员、生产主管、设备人员还是管理层。  
8. 验收标准如何定义？例如准确率、响应时间、覆盖场景、上线用户数等。  
9. 是否有明确上线时间要求？如果有，是否可接受分阶段上线。  
10. 合同范围、交付边界、维护责任和后续扩展方式是否清晰。

## 风险信号

1. “10 万以内全包”属于价格和范围承诺，当前信息不足，不能直接承诺。  
2. “保证 2 周上线”属于交付周期和结果承诺，当前数据、系统、部署和验收条件未确认，风险较高。  
3. 如需对接 MES、ERP、质检系统，属于系统集成风险，需要售前和交付评估。  
4. 如涉及生产设备实时数据，复杂度和安全要求可能显著提高。  
5. 如需要现场部署或私有化部署，环境准备、部署周期和验收风险需单独评估。  
6. 合同范围和验收标准未确认，存在范围蔓延和交付争议风险。  
7. “全包”“快速上线”“固定低价”同时出现，属于高风险销售承诺，需提交售前和风控评审。
