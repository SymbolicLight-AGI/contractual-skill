## 机会判断：暂不建议作为重点机会推进

当前只能判断客户对 AI Agent 有初步兴趣，但不能判断其具备明确购买意向或高优先级成交可能。

原因是关键信息严重不足：客户尚未说明具体业务场景、预算、决策人、上线时间、数据来源和系统环境。客户说“下周给我一个方案”，更像是初步了解或方案探索，不应直接等同于采购意向。

销售备注中客户曾问过“能不能一周上线”，说明其可能关注交付速度，但无法确认这是紧急项目需求、随口询问，还是对产品能力的试探。因此该机会目前应评估为：有兴趣，但未验证需求与成交条件，需谨慎推进。

## 待确认关键问题

1. **业务场景**
   - 客户希望 AI Agent 解决什么具体问题？
   - 是销售、客服、售后、渠道管理、库存、报价，还是内部流程自动化？
   - 当前痛点是什么？是否已经影响业务结果？

2. **目标与价值**
   - 客户希望通过 AI Agent 达到什么效果？
   - 是降本、提效、提升成交率、减少人工，还是改善客户响应？
   - 是否有明确的衡量指标？

3. **上线时间**
   - 之前询问“一周上线”是否代表真实项目时间要求？
   - 如果不能一周上线，客户是否仍有兴趣？
   - 是否有必须赶上的业务节点？

4. **预算情况**
   - 是否已有预算？
   - 预算大致范围是多少？
   - 是试点预算还是正式采购预算？

5. **决策链路**
   - 当前沟通人是否为决策人？
   - 谁负责业务拍板、技术评估和采购审批？
   - 是否需要老板、IT、财务或业务负责人参与？

6. **数据与系统环境**
   - 客户是否已有可用数据？
   - 数据存放在哪里？
   - 是否需要对接现有 CRM、ERP、客服系统、微信、企业微信或其他业务系统？
   - 是否存在数据安全、权限或部署要求？

7. **方案范围**
   - 客户希望看到的是产品介绍、场景方案、报价，还是可落地的实施计划？
   - 是否接受先做小范围 PoC 或试点？

## 下一步建议

1. **不要直接提交完整正式方案**
   - 在需求未明确前，不建议投入大量资源做深度定制方案。
   - 可先提供一版“探索型方案框架”，避免承诺过度。

2. **尽快安排一次需求澄清会议**
   - 会议目标不是推产品，而是确认客户场景、痛点、预算、决策人和时间要求。
   - 建议邀请客户业务负责人及可能的技术/系统负责人参加。

3. **用低成本方式验证机会质量**
   - 可准备 2-3 个适合设备经销商的 AI Agent 场景假设，例如：
     - 售前咨询与报价辅助
     - 售后工单与知识库问答
     - 销售跟进与客户线索管理
   - 在会议中让客户选择最接近的场景，确认是否有真实痛点。

4. **明确“一周上线”的真实含义**
   - 如果客户只是希望快速看到效果，可建议先做轻量试点。
   - 如果客户要求正式上线，则需要确认数据、系统、流程和验收标准，否则风险较高。

5. **设置推进门槛**
   - 若客户能明确场景、业务负责人、试点范围和时间计划，可升级为重点跟进机会。
   - 若客户仍无法说明需求、预算和决策链路，则建议保持普通培育，不作为重点销售机会投入。
