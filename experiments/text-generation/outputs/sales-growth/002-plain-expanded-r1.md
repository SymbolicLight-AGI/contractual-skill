## 机会判断：暂不建议直接列为重点推进，建议先做快速资格确认

客户仅表达“AI Agent 看起来不错，下周给我一个方案”，目前只能判断为**初步兴趣**，不能视为明确购买意向或高确定性机会。  
在缺少业务场景、预算、决策链、上线时间、数据与系统环境等关键信息的情况下，建议将该机会列为**待验证机会**，先安排一次需求澄清，再决定是否重点投入售前资源。

- 当前状态：对 AI Agent 有初步兴趣
- 业务背景：未提供具体经营规模、客户群体、销售/售后流程、现有系统情况
- 当前机会阶段：早期线索 / 初步兴趣阶段

- 客户尚未说明数据来源。
- 客户尚未说明现有系统环境。
- 当前没有会议纪要。

## 可推断的潜在机会，但需验证

以下仅为推断，不应作为客户已确认需求：

- 客户可能希望了解 AI Agent 在设备经销业务中的应用价值，例如销售辅助、客户跟进、售后咨询、内部知识问答等。
- 客户询问“一周上线”，可能存在对快速落地的期待。
- 客户要求“下周给方案”，可能希望看到初步解决路径或演示型方案，但不代表已经进入采购决策。

## 风险信号

1. **需求不清**
   - 未明确要解决的业务问题，无法判断方案范围和价值点。

2. **上线预期风险**
   - 客户曾问“能不能一周上线”，存在对交付周期过短的预期风险。
   - 不能承诺一周上线，应由售前评估范围、数据和系统条件后再判断。

3. **方案范围风险**
   - 未明确是否涉及系统集成、数据接入、权限控制、知识库建设、流程改造等内容。
   - 不宜直接给固定范围方案。

4. **商机质量不明**
   - 未确认预算、决策人、采购流程和优先级，无法判断是否值得重点推进。

5. **信息记录不足**
   - 没有会议纪要，容易造成销售、售前与客户理解不一致。

## 待确认问题

建议在下次沟通中重点确认：

1. 客户希望 AI Agent 解决什么具体问题？
   - 销售获客、客户跟进、售后服务、内部知识查询、订单/库存咨询，还是其他场景？

2. 当前业务痛点是什么？
   - 人工成本高、响应慢、客户流失、销售转化低、售后重复问题多，还是管理效率问题？

3. 使用对象是谁？
   - 内部销售、客服、售后人员、管理层，还是终端客户？

4. 是否已有数据基础？
   - 产品资料、报价规则、客户资料、售后知识库、历史对话、工单数据等是否可用？

5. 是否需要对接现有系统？
   - CRM、ERP、库存系统、订单系统、客服系统、企业微信/微信、小程序等。

6. 客户期望的上线节奏是什么？
   - 是希望先做 Demo / POC，还是直接生产上线？

7. 是否有预算范围和采购计划？
   - 今年预算、项目预算、试点预算是否明确？

8. 谁参与决策？
   - 业务负责人、老板、IT、财务、采购是否需要参与？

9. “下周给方案”的具体期望是什么？
   - 是概念方案、场景建议、产品演示、试点方案，还是报价方案？

## 推荐下一步

1. **先安排一次 30–45 分钟需求澄清会**
   - 目标不是直接报价，而是明确业务场景、优先级、上线边界和决策流程。

2. **给客户提供“初步场景建议版方案”，避免承诺固定交付**
   - 可输出方向性方案，例如：
     - 适用场景假设
     - 可能的 AI Agent 能力模块
     - 试点路径
     - 所需客户配合资料
     - 待评估事项
   - 不建议直接给最终价格、确定周期或固定实施范围。

3. **将“一周上线”改为“是否可做轻量试点需评估”**
   - 明确需确认数据、系统、使用场景和验收标准后再判断可行性。

4. **内部先做机会资格评估**
   - 若客户能明确业务痛点、预算、决策人和试点范围，可升级为重点机会。
   - 若客户仅泛泛了解且无预算/负责人，则维持低成本跟进。

5. **补充会议纪要**
   - 后续每次沟通需记录客户原话、确认事项、未确认事项和下一步责任人。

---

## 内部交接建议

- **销售负责**
  - 约需求澄清会。
  - 确认预算、决策人、采购节奏、项目优先级。
  - 不将客户兴趣表述为确定购买意向。

- **售前负责**
  - 参与需求澄清。
  - 判断场景适配度、方案边界、数据要求和系统集成复杂度。
  - 输出非承诺型初步方案。

- **风险/交付评估建议介入条件**
  - 客户继续要求“一周上线”。
  - 客户要求固定报价、固定范围或保证效果。
  - 涉及多个系统集成、敏感数据接入或复杂业务流程改造。

---

## 结论

该机会目前**有跟进价值，但不建议直接作为重点机会推进**。  
建议先低成本快速验证客户真实需求、预算、决策链和可落地场景。只有在关键问题明确后，才适合投入更多售前资源并升级为重点商机。
