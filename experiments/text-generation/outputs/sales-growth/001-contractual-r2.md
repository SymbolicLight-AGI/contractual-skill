- **风险信号**：客户对数据安全和权限隔离敏感，若权限边界、数据访问、日志审计、知识隔离机制不清晰，可能影响推进。

- **风险信号**：决策时间表未明确，销售节奏和资源投入优先级存在不确定性。

## 推荐下一步

1. **组织需求澄清会**
   - 参会建议：售后负责人、信息化负责人、生产计划经理、我方销售、售前。
   - 目标：明确首期业务场景、知识范围、用户角色、权限要求、数据来源和系统边界。
   - 注意：不承诺最终范围、上线周期、集成效果或业务结果。

2. **梳理首期场景优先级**
   - 建议聚焦：
     - 售后标准问答沉淀。
     - 工单经验知识化。
     - 新员工培训知识库。
     - 售后一线检索和问答辅助。
   - 输出：场景清单、优先级、涉及数据源、使用角色、预期验证指标。
   - 指标仅作为评估方向，不承诺最终达成结果。

3. **补充关键缺失信息**
   - 预算范围或预算审批方式。
   - 决策人、评估人、使用部门、IT 审批角色。
   - 决策时间表和采购流程。
   - 是否存在竞品或内部方案。
   - 老系统名称、接口能力、数据格式、权限体系。
   - 数据安全要求：部署方式、访问控制、审计、数据脱敏、知识隔离。
   - 首期希望覆盖的知识规模、用户范围、部门范围。

4. **准备方案交流材料**
   - 面向售后负责人：突出知识沉淀、标准问答、工单经验复用、新人培训。
   - 面向信息化负责人：突出权限隔离、数据安全、系统集成边界、接口假设和风险控制。
   - 面向管理层：突出业务问题、分阶段建设路径、评估指标和资源投入要求。
   - 所有材料应避免承诺最终价格、上线时间和确定收益。

5. **建议采用分阶段推进路径**
   - 阶段一：需求澄清与数据盘点。
   - 阶段二：选取售后知识库小范围场景验证。
   - 阶段三：评估 Agent 问答、工单辅助、权限隔离和系统集成能力。
   - 阶段四：根据验证结果讨论采购、实施范围和推广计划。
   - 以上为推进建议，不代表已确认交付范围或时间承诺。

## 交接建议

- **售前交接**
  - 需要售前参与需求澄清和方案设计。
  - 重点评估：
    - 企业知识库与售后 Agent 的首期可行场景。
    - 微信群、Excel、老系统知识导入方式。
    - 权限隔离、角色访问、知识分类设计。
    - 老系统集成边界和接口前提。
  - 禁止在未评估前承诺“全部系统打通”或“自动完成知识治理”。

- **交付 PM 交接**
  - 如客户进入试点或实施讨论，需要交付 PM 评估：
    - 数据准备情况。
    - 知识清洗和归类工作量。
    - 试点范围。
    - 验收标准。
    - 推广节奏。
  - 暂不承诺上线周期或实施范围。

- **风险评审交接**
  - 如客户提出以下诉求，应触发风险评审：
    - 全部老系统集成。
    - 短周期上线。
    - 固定低价覆盖不确定范围。
    - 无限知识清洗或无限定制。
    - 对售后效率提升、新人培训周期缩短作结果保证。
  - 当前已有风险关注点：数据安全、权限隔离、系统集成复杂度。

- **财务 / 合同交接**
  - 当前预算、付款条件、采购方式均未明确，暂不进入价格承诺。
  - 如客户询价，可先收集预算和采购流程，再由销售、财务或合同团队确认可对外口径。
  - 不应在当前阶段承诺折扣、付款条件或最终报价。
