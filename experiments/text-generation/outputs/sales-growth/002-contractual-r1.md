## 推断与不确定性

- **推断**：客户可能对 AI Agent 有初步好奇或探索意愿，但尚不能判断是否存在明确采购需求。
- **推断**：客户要求“下周给方案”，可能希望快速了解可行性、场景、价格或落地路径，但具体诉求不明确。
- **推断**：客户曾问“一周上线”，可能存在对快速落地的期待，但不能据此承诺上线周期。
- **不确定性**：
  - 具体业务场景不明确。
  - 是否有真实业务痛点不明确。
  - 是否有预算不明确。
  - 决策人和采购流程不明确。
  - 是否已有竞品或内部方案不明确。
  - 数据是否可用、是否合规、是否结构化不明确。
  - 是否需要对接客户现有系统不明确。
  - 客户期望的 AI Agent 能力边界不明确。
  - 客户所谓“方案”是概念方案、技术方案、报价方案还是实施方案不明确。

## 痛点与机会

- **机会信号**
  - **已知事实**：客户主动表达对 AI Agent 的兴趣。
  - **已知事实**：客户要求下周提供方案，说明愿意继续沟通。
  - **推断**：设备经销商可能存在销售跟进、客户服务、设备知识问答、售后工单、备件咨询、内部运营提效等潜在 AI Agent 应用场景，但需客户确认。

- **潜在痛点方向，需确认**
  - 是否希望提升销售线索跟进效率。
  - 是否希望减少售后客服或技术支持压力。
  - 是否希望让业务人员快速查询产品、库存、价格、维修知识等信息。
  - 是否希望统一管理客户咨询、设备资料、工单或知识库。
  - 是否希望将 AI Agent 接入现有 CRM、ERP、工单、库存或客服系统。

- **当前机会价值判断**
  - 目前只能判断为**有兴趣信号**，不能判断为**明确购买意向**。
  - 若客户能明确业务场景、试点范围、数据来源、预算区间和决策人，则可升级为重点推进机会。
  - 若客户只是泛泛了解 AI Agent，且要求一周上线但无法提供数据和系统条件，则应谨慎投入，避免过早承诺方案深度和交付周期。

## 风险信号

- **风险信号：需求不清**
  - 客户未说明具体业务场景，容易导致方案泛化，难以形成有效推进路径。

- **风险信号：上线周期可能不现实**
  - 客户曾询问“能不能一周上线”。在数据、系统、集成范围、验收标准均未知的情况下，不能承诺一周上线。

- **风险信号：预算未知**
  - 未确认预算，可能出现方案投入与客户支付意愿不匹配。

- **风险信号：决策链未知**
  - 未确认决策人、使用部门、IT/数据负责人、采购流程，推进效率和成交概率无法判断。

- **风险信号：数据与系统环境未知**
  - 未确认数据来源、数据质量、权限、合规要求和系统接口，可能影响技术可行性、交付范围和实施成本。

- **风险信号：方案边界不清**
  - 若客户期待“什么都能做”的 AI Agent，容易引发无限定制、固定低价、快速上线等交付风险。

## 推荐下一步

1. **先做资格确认，不直接按重点机会重投入**
   - 建议销售在 1 次 30–45 分钟沟通内完成关键问题确认。
   - 沟通后再判断是否升级为重点推进机会。

2. **下周方案建议定位为“初步场景探索方案”**
   - 不建议直接提交最终报价、最终实施范围或上线承诺。
   - 可提供：
     - AI Agent 可应用场景假设；
     - 典型试点路径；
     - 所需客户配合事项；
     - 待确认问题清单；
     - 可选 PoC / 试点思路。
   - 明确标注：具体范围、周期、价格需基于业务场景、数据和系统环境确认后评估。

3. **必须向客户确认的问题**
   - 业务场景：
     - 您希望 AI Agent 首先解决哪个具体问题？
     - 面向内部员工、销售、售后人员，还是终端客户？
   - 目标与成功标准：
     - 希望提升什么指标？例如响应效率、线索转化、客服效率、知识查询准确性等。
     - 如何判断试点成功？
   - 数据来源：
     - 是否已有产品资料、FAQ、工单记录、客户咨询记录、设备手册等数据？
     - 数据格式、质量、权限和更新机制如何？
   - 系统环境：
     - 是否需要接入 CRM、ERP、库存、工单、客服系统或企业微信等？
     - 是否有接口、权限和 IT 配合？
   - 时间要求：
     - “一周上线”指的是演示 Demo、内部试点，还是正式生产上线？
   - 预算与采购：
     - 是否已有预算区间？
     - 本次是调研、试点，还是正式采购？
   - 决策链：
     - 哪些部门参与评估？
     - 谁是业务负责人、技术负责人和最终决策人？

4. **机会优先级建议**
   - 当前建议评级：**中低优先级 / 待资格确认**。
   - 升级为重点推进的条件：
     - 客户明确首个落地场景；
     - 有业务负责人参与；
     - 有可用数据或愿意提供样例数据；
     - 明确试点目标和时间窗口；
     - 确认预算或采购路径；
     - 技术集成范围可控。
   - 降级或暂缓的条件：
     - 客户只要求泛泛方案；
     - 无具体场景、无预算、无决策人；
     - 坚持一周正式上线；
     - 要求低价固定范围或无限制功能承诺。

## 交接建议

- **销售负责人**
  - 负责完成客户资格确认。
  - 避免将“感兴趣”表述为“有明确采购意向”。
  - 不承诺价格、上线周期、交付范围或效果。

- **售前 Presales**
  - 建议在客户确认初步场景后介入。
  - 负责判断 AI Agent 场景匹配度、技术路径、数据要求和集成假设。
  - 若客户要求下周方案，可由售前支持输出“初步方案框架”，但需保留假设条件。

- **交付 PM**
  - 如客户继续关注“一周上线”，需交付 PM 评估数据准备、配置、测试、验收和上线条件。
  - 不应在未评估前承诺正式上线周期。

- **风险 Review**
  - 如客户要求“保证一周上线”“固定低价全包”“所有系统都接入”“效果保证”，需进入风险评审。

- **财务 / 合同**
  - 当前不建议进入正式报价或合同阶段。
  - 如客户明确预算和采购意向后，再评估价格、付款条件和合同条款。
