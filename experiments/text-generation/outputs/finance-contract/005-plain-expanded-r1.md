## 结论：不建议直接走合同

该项目范围大、阶段多、接口多，且客户要求“一份合同覆盖全部阶段、报价包含所有系统接口、验收标准后续再定”。目前存在明显的范围、成本、交付、验收和责任风险，不能直接进入合同签署流程。应先完成售前、交付、财务、法务及风险审查后，再决定合同结构和报价边界。

## 已知事实

- 项目内容：集团级知识库和多 Agent 工作流。
- 初始范围包括：
  - 集团总部知识库；
  - 5 个业务 Agent；
  - 对接 OA、CRM、售后工单系统；
  - 试点后推广到 8 家分公司。
- 客户要求：
  - 一份合同覆盖全部阶段；
  - 报价包含所有系统接口；
  - 验收标准后续再定。

## 主要风险

### 1. 范围风险

- “集团级知识库”和“多 Agent 工作流”范围较宽，需明确知识库数量、数据来源、权限体系、更新机制、Agent 能力边界等。
- 5 个业务 Agent 的具体场景、功能、流程、交互方式未明确。
- 试点后推广到 8 家分公司，可能涉及大量差异化配置、培训、数据迁移和本地化支持，不能默认包含在原报价内。
- “所有系统接口”表述过宽，存在无限扩展风险。

### 2. 接口与第三方系统风险

- OA、CRM、售后工单系统的厂商、版本、接口开放程度、认证方式、数据权限、开发配合责任均未明确。
- 若第三方系统不开放接口或接口能力不足，可能导致延期、改造费用或交付失败风险。
- 需明确接口数量、字段范围、调用频率、数据同步方式、异常处理和安全要求。

### 3. 验收风险

- “验收标准后续再定”不可接受作为直接签约基础。
- 缺少验收标准会导致交付完成标准、付款触发条件、返工边界不清。
- 应至少在合同或附件中明确阶段性验收标准、验收流程、验收期限、客户反馈机制和默认验收规则。

### 4. 报价与成本风险

- 报价包含全部阶段和所有接口，可能低估实施、集成、推广、培训、运维和变更成本。
- 8 家分公司推广成本未拆分，容易形成免费扩展或无限服务承诺。
- 如需一份合同覆盖全部阶段，应拆分阶段、里程碑、付款节点和变更机制。

### 5. 合同与责任风险

- 一份合同覆盖试点、集成、推广全部阶段，若边界不清，会放大违约、延期和质量责任。
- 第三方系统配合、客户数据质量、客户内部流程确认等责任需写清。
- 后续再定验收标准可能导致合同关键条款缺失。

## 需确认的问题

1. 5 个业务 Agent 分别对应哪些业务场景、功能范围和使用角色？
2. 集团总部知识库的数据来源、数据量、权限体系、更新频率和安全要求是什么？
3. OA、CRM、售后工单系统的具体厂商、版本、接口文档、开放权限和客户配合人是否已确认？
4. “所有系统接口”是否仅指当前列明的 3 类系统？是否包含后续新增系统？
5. 试点范围是哪一个总部部门或哪几家单位？试点成功后推广到 8 家分公司的具体交付内容是什么？
6. 是否需要数据迁移、历史数据清洗、权限梳理、系统培训、驻场支持、运维服务？
7. 验收标准、验收周期、验收材料、失败重试机制和默认验收规则是否可在合同签署前明确？
8. 客户是否接受按阶段付款、阶段验收和范围变更机制？
9. 若第三方系统接口不具备条件，责任和费用由谁承担？

## 必须复核角色与事项

### 售前复核

- 明确项目方案、功能边界和交付清单。
- 拆分 5 个业务 Agent 的场景、能力、限制和不包含事项。
- 明确知识库建设范围、数据来源、权限要求和实施假设。
- 将“所有系统接口”改为明确的接口清单、数量和边界。
- 明确试点与 8 家分公司推广的阶段划分。

### 交付复核

- 评估 OA、CRM、售后工单系统集成可行性。
- 评估第三方系统接口依赖、客户配合要求和技术风险。
- 评估总部试点到 8 家分公司推广的人力、周期、培训和实施成本。
- 输出可执行的里程碑、交付物、验收条件和变更流程建议。
- 判断是否需要分阶段合同或主合同加订单/附件模式。

### 财务复核

- 审核报价是否覆盖接口开发、推广实施、培训、运维、项目管理和风险成本。
- 评估一份合同覆盖全部阶段的付款节点和回款风险。
- 要求设置阶段付款、阶段验收和变更收费机制。
- 评估是否存在免费扩展、无限接口、无限培训或无限支持风险。
- 审核试点失败、延期或客户配合不足时的成本承担安排。

### 法务复核

- 审查合同是否能在验收标准未明确前签署。
- 明确范围边界、客户配合义务、第三方系统责任、知识产权、数据安全和保密责任。
- 设置变更控制、延期责任、验收流程、违约责任和责任限制。
- 避免“一份合同覆盖全部阶段”导致不明确或无限义务。
- 审核是否需要主合同加阶段附件、SOW 或订单机制。

### 风险审查

- 评估集团级、多分公司推广、多系统集成带来的整体交付风险。
- 评估客户要求“报价包含所有系统接口”和“验收标准后续再定”的高风险程度。
- 判断是否需要立项风险评审或管理层审批。
- 对范围不清、接口不可控、验收缺失、回款不确定等事项提出前置门槛。

## 建议处理方式

- 暂不直接走合同。
- 先完成范围澄清、接口清单、阶段拆分、验收标准和付款节点设计。
- 建议采用“主合同 + 阶段性 SOW/附件/订单”的方式，而不是无边界地一份合同覆盖全部阶段。
- 合同签署前必须完成售前、交付、财务、法务和风险审查。
