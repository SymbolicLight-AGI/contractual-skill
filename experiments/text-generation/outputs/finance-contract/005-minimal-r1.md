## 必须复核事项

### 1. 售前复核
- 明确“集团级知识库”和“5 个业务 Agent”的具体功能边界。
- 明确 5 个 Agent 的业务场景、能力范围、是否包含训练、调优、运营支持。
- 明确 OA、CRM、售后工单系统的接口数量、接口方式、数据范围、权限要求和第三方配合责任。
- 明确试点范围与推广到 8 家分公司的实施边界。
- 评估“报价包含所有系统接口”是否存在范围过大或接口数量不封顶风险。
- 明确是否包含后续新增系统、新增 Agent、新增分公司需求。

### 2. 交付复核
- 评估总部知识库、5 个 Agent、3 类系统集成、8 家分公司推广的实施工作量和排期。
- 明确试点阶段、推广阶段的交付里程碑。
- 明确客户侧数据、账号、接口文档、测试环境、业务人员配合等前置条件。
- 明确验收标准不能“后续再定”，至少需在合同前形成阶段性验收口径。
- 评估多系统对接、跨分公司推广带来的项目管理和交付风险。
- 判断是否需要分阶段交付、分阶段验收、分阶段付款。

### 3. 财务复核
- 审查当前报价是否足以覆盖全部阶段、全部接口、8 家分公司推广成本。
- 评估“一份合同覆盖全部阶段”是否导致收款周期过长或回款风险。
- 明确付款节点、预付款比例、试点验收款、推广款、尾款比例。
- 评估是否存在未封顶服务、接口无限制、后续需求无限包含的成本风险。
- 如存在大额折扣、打包价或包含所有接口，应进行财务测算和审批。

### 4. 法务复核
- 审查“一份合同覆盖全部阶段”的合同结构是否可控。
- 要求合同中明确范围边界、变更机制、客户配合义务、第三方系统责任。
- 明确验收标准、验收流程、逾期验收处理机制。
- 明确数据安全、权限、保密、系统对接责任边界。
- 审查是否涉及集团总部及 8 家分公司的主体、授权和履约责任问题。
- 避免写入“包含所有系统接口”“验收标准后续再定”等高风险模糊条款。

### 5. 风险审查
- 范围风险：总部知识库、Agent、系统集成、分公司推广范围较大，当前描述不够清晰。
- 接口风险：客户要求报价包含所有系统接口，但接口数量、复杂度和第三方配合不明确。
- 验收风险：验收标准后续再定，可能导致无法确认交付完成。
- 回款风险：若合同覆盖全部阶段但付款节点不清晰，可能造成长期垫资。
- 交付风险：涉及集团总部和 8 家分公司，组织协调复杂。
- 变更风险：后续推广和新增需求可能被客户理解为已包含在原合同内。

## 建议下一步

1. 暂不直接走合同。
2. 先组织售前、交付、财务、法务和风险审查联合评审。
3. 要求补充确认：
   - 详细范围清单；
   - 接口清单；
   - 分阶段交付计划；
   - 验收标准；
   - 付款里程碑；
   - 变更机制；
   - 客户侧配合责任。
4. 建议采用“总体框架 + 阶段范围/里程碑/付款/验收”方式推进，避免无限范围和无限接口承诺。
