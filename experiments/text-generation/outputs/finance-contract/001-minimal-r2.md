## 报价草案财务与合同风险检查结果

### 一、主要合同风险

1. **范围边界不清**
   - “知识库初始化”未明确包含的数据量、资料格式、数据清洗深度、标签/分类建设、知识审核责任等。
   - “1 个客服 Agent”未明确功能边界、支持渠道、并发能力、对话轮次、知识库调用范围、权限配置等。
   - “2 次远程培训”未明确每次时长、参训人数、培训内容、是否包含录播或补训。

2. **数据清洗边界缺失**
   - 未说明是否包含原始资料整理、去重、格式转换、结构化处理、问答生成、人工校对等工作。
   - 若客户提供的数据质量较差，可能导致交付工作量显著增加。

3. **试运行通过标准缺失**
   - 付款节点为“试运行通过后 50%”，但未定义通过标准。
   - 存在客户以主观体验、业务效果或持续优化为由延迟确认的风险。
   - 未明确试运行周期、验收流程、反馈次数、问题修复范围。

4. **第三方系统接口费用未明确**
   - 未说明是否包含 CRM、工单系统、客服系统、企业微信、钉钉、官网等第三方接口对接。
   - 如涉及接口开发、API 调用费、第三方平台服务费、账号费、私有化部署费，可能产生额外成本争议。

5. **变更需求计费方式缺失**
   - 未明确超出当前范围的新增需求如何报价、审批和排期。
   - 客户可能将新增功能、额外培训、额外数据处理、接口集成等要求视为已包含在首期报价中。

6. **验收与交付物不明确**
   - 未列明最终交付物，例如知识库文件/配置、Agent 配置、培训材料、试运行报告等。
   - 未明确客户应配合提供的数据、系统权限、业务联系人和反馈时限。

---

### 二、主要财务风险

1. **尾款回收风险**
   - 50% 尾款绑定“试运行通过”，但通过标准不明确，可能导致尾款支付周期拉长。
   - 若客户迟迟不确认试运行结果，可能形成应收账款风险。

2. **成本超支风险**
   - 数据清洗、接口对接、反复调优、额外培训等边界不清，可能导致实际投入超过 120000 元报价覆盖范围。

3. **第三方费用承担风险**
   - 未明确第三方接口、平台、API、账号、云资源等费用由谁承担。
   - 若由我方承担，可能压缩项目利润。

4. **变更无偿交付风险**
   - 未建立变更确认和计费机制，后续新增需求可能被要求免费完成。

5. **付款条件偏主观**
   - “试运行通过”如果不绑定客观标准、时间限制和默认确认机制，财务收款可控性较弱。

---

### 三、待确认条款

1. **知识库初始化范围**
   - 包含多少份资料、多少条知识、什么格式的数据？
   - 是否包含数据清洗、结构化、问答生成、人工审核？
   - 客户与我方各自负责哪些工作？

2. **客服 Agent 范围**
   - Agent 的功能边界是什么？
   - 是否仅基于知识库问答，还是包含工单创建、客户信息查询、业务流程处理？
   - 是否涉及第三方系统集成？

3. **试运行与验收标准**
   - 试运行周期多长？
   - 通过标准是什么？
   - 验收由谁确认？
   - 客户反馈问题的次数和范围如何限定？
   - 若客户逾期不反馈，是否进入下一付款节点需进一步确认。

4. **第三方系统和费用**
   - 是否包含接口开发？
   - 第三方系统 API、账号、平台、云资源、短信或模型调用等费用由谁承担？
   - 如接口不可用或第三方配合延迟，是否影响项目周期和验收？

5. **变更需求机制**
   - 超出报价范围的需求如何提交、评估、报价和审批？
   - 新增 Agent、额外培训、额外知识库处理、接口开发是否另行计费？

6. **付款与发票**
   - 50% 首付款和 50% 尾款对应金额分别为 60000 元。
   - 发票类型、开票时间、付款期限、付款条件需进一步明确。
   - 尾款支付是否与书面验收单或试运行报告绑定需确认。

---

### 四、建议下一步

1. 在报价或合同附件中补充明确的**项目范围说明**和**不包含事项**。
2. 明确“试运行通过”的客观标准、周期、验收流程和客户反馈时限。
3. 单独列出第三方系统接口、平台费用、API 费用、账号费用的承担方式。
4. 增加变更需求的审批和计费流程，避免免费扩范围。
5. 将交付物、客户配合义务、培训次数及时长、数据提交要求写清。
6. 建议提交法务、财务和交付团队进一步审核后，再确认最终报价和合同条款。
