## 总体判断

该 PR 存在高风险问题，当前不建议上线。新增用户导出功能涉及个人信息与订单金额数据，但权限控制仅在前端完成，后端 API 未做角色校验，同时缺少审计日志和导出数量限制，存在未授权导出、隐私泄露、合规与追责困难等风险。

## 主要问题

1. **Finding：后端 API 未检查角色**
   - 导出功能只依赖前端按钮判断管理员身份。
   - 后端没有进行管理员权限校验。

2. **Finding：导出内容包含敏感/个人信息**
   - 导出字段包括姓名、手机号、邮箱、最近订单金额。
   - 这些数据属于个人信息及可能涉及业务敏感信息。

3. **Finding：缺少审计日志**
   - 当前变更摘要中明确没有审计日志。
   - 无法记录谁在什么时间导出了哪些范围的数据。

4. **Finding：没有导出数量限制**
   - 当前导出功能没有数量上限。
   - 可能一次性导出大量用户数据。

5. **Finding：缺少明确测试覆盖信息**
   - 摘要未提到权限测试、隐私导出范围测试、审计测试、导出上限测试。

## 风险与影响

1. **Risk：未授权访问与数据泄露**
   - 仅前端隐藏按钮不能作为权限控制。
   - 非管理员可能通过直接调用后端 API、构造请求或复用接口访问导出能力。
   - 影响范围包括用户姓名、手机号、邮箱、最近订单金额。

2. **Risk：隐私与合规风险**
   - 导出手机号、邮箱、订单金额等信息需要明确业务目的、最小化原则和访问范围。
   - 如果缺少产品/合规确认，可能导出超出必要范围的数据。

3. **Risk：无法追责与安全事件响应困难**
   - 没有审计日志时，无法确认导出发起人、导出时间、导出字段、导出数量和导出结果。
   - 一旦发生数据泄露，难以排查和取证。

4. **Risk：批量数据外泄风险扩大**
   - 没有导出数量限制可能导致一次请求导出全量用户数据。
   - 如果接口被滥用或账号被盗，影响面会显著扩大。

5. **Risk：数据完整性和业务边界不清**
   - “最近订单金额”的定义、币种、退款/取消订单是否包含等规则不明确。
   - 可能造成导出数据与业务预期不一致。

## 测试建议

1. **权限测试**
   - 非登录用户调用导出 API 应失败。
   - 普通用户调用导出 API 应失败。
   - 非管理员即使知道接口地址也不能导出。
   - 管理员调用导出 API 才允许成功。
   - 前端隐藏按钮之外，必须验证后端强制鉴权。

2. **隐私字段测试**
   - 验证导出字段是否仅包含产品确认允许的字段。
   - 验证手机号、邮箱、订单金额是否符合最小化导出原则。
   - 如存在脱敏要求，应测试脱敏格式和完整导出权限边界。

3. **审计日志测试**
   - 成功导出时应记录操作者、时间、导出条件、导出数量、导出字段、请求来源等必要信息。
   - 失败导出或权限拒绝也应记录安全相关事件。
   - 验证审计日志不包含不必要的完整敏感数据。

4. **数量限制与滥用测试**
   - 测试单次导出数量上限。
   - 测试分页、时间范围或筛选条件限制。
   - 测试超出限制时的错误响应。
   - 测试高频请求或重复导出是否有速率限制或风控策略。

5. **业务规则测试**
   - 验证“最近订单金额”的计算规则。
   - 覆盖无订单、退款订单、取消订单、多币种订单等场景。
   - 验证导出结果与后台订单数据一致。

## 不确定性

- **Uncertainty：** 未说明导出文件格式、存储方式和下载链接有效期，无法判断是否还有文件泄露风险。
- **Uncertainty：** 未说明导出对象范围，例如是否支持全量用户、按条件筛选或仅当前租户/组织。
- **Uncertainty：** 未说明是否有现有审计平台或安全日志规范。
- **Uncertainty：** 未说明该功能的产品需求、合规要求和数据最小化标准。

## 交接建议

1. **Security 负责人**
   - 确认该导出功能的权限模型。
   - 审查是否存在未授权访问、批量数据泄露、隐私合规风险。
   - 明确审计日志、安全告警、速率限制和数据保护要求。

2. **Backend 负责人**
   - 在后端 API 实现强制管理员角色校验，不能依赖前端按钮。
   - 增加导出数量限制、请求参数校验、错误处理和审计日志。
   - 确认导出查询范围、租户/组织隔离、订单金额计算逻辑。

3. **Product 负责人**
   - 确认导出字段是否必要，尤其是手机号、邮箱、最近订单金额。
   - 明确哪些角色可以导出、可导出的用户范围、导出目的和验收标准。
   - 确认是否需要脱敏、审批流程或导出提示。

4. **QA/Test 负责人**
   - 补充权限、隐私字段、审计日志、导出上限、异常路径和业务规则测试。
   - 覆盖非管理员直接调用 API 的回归测试。
   - 验证失败场景不会返回成功或部分敏感数据。
