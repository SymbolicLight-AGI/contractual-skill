## 总体判断

该 PR 存在高风险问题，不建议直接发布。  
新增的用户导出功能涉及姓名、手机号、邮箱、最近订单金额等敏感/个人信息，但当前摘要显示权限控制仅在前端完成，后端 API 未校验角色，也没有审计日志和导出数量限制。这会带来明显的未授权访问、隐私泄露、合规审计缺失和批量数据导出风险。

## 主要问题

### 1. 后端 API 未检查角色

- **Finding:** 只有前端按钮做了管理员判断，后端 API 未检查角色。
- **Risk:** 非管理员用户可能绕过前端限制，直接调用后端导出接口获取用户数据。
- **影响范围:** 涉及用户姓名、手机号、邮箱、最近订单金额等敏感数据，属于高风险权限与隐私问题。
- **建议:** 后端必须进行强制角色/权限校验，不能依赖前端控制。

### 2. 导出内容包含敏感/个人信息

- **Finding:** 导出字段包括姓名、手机号、邮箱、最近订单金额。
- **Risk:** 这些字段可能构成个人信息或敏感业务信息，一旦被未授权导出，会造成隐私泄露、客户信任损失和潜在合规风险。
- **建议:** 明确导出字段是否必要，是否需要脱敏、最小化字段范围，是否需要按业务角色细分权限。

### 3. 缺少审计日志

- **Finding:** 没有审计日志。
- **Risk:** 无法追踪谁在什么时间导出了哪些数据、导出范围是多少、是否存在异常导出行为。
- **影响:** 发生数据泄露或用户投诉时，难以调查和追责。
- **建议:** 增加导出审计日志，至少记录操作者、角色、时间、导出条件、导出数量、请求来源、结果状态等信息。避免在日志中完整记录敏感字段内容。

### 4. 没有导出数量限制

- **Finding:** 没有导出数量限制。
- **Risk:** 可能被一次性导出大量用户数据，扩大泄露影响；也可能导致后端性能压力、超时或资源耗尽。
- **建议:** 增加单次导出上限、分页/异步任务机制、频率限制、权限分级限制和异常导出告警。

### 5. 缺少明确的产品和合规边界

- **Finding:** 当前摘要未说明导出功能的目标用户、业务场景、导出范围、数据保留策略和审批流程。
- **Risk:** 功能可能超出实际业务需要，导致过度收集/导出。
- **建议:** 产品负责人需要确认导出对象、字段必要性、访问角色、导出频次、是否需要审批或二次确认。

## 风险与影响

- **权限风险:** 后端无角色校验会导致前端限制失效，存在越权导出风险。
- **隐私风险:** 姓名、手机号、邮箱和订单金额被批量导出，可能造成个人信息泄露。
- **审计风险:** 无审计日志会导致事后无法追踪导出行为，影响安全调查和合规证明。
- **数据完整性/业务风险:** 最近订单金额属于业务敏感数据，未经授权导出可能泄露客户消费信息。
- **可用性风险:** 无数量限制可能导致大批量导出拖垮接口或影响正常服务。
- **滥用风险:** 如果接口可被脚本直接调用，可能被批量抓取用户数据。

## 测试建议

### 权限测试

- 非管理员用户直接调用导出 API，应被拒绝。
- 未登录用户调用导出 API，应被拒绝。
- 普通管理员、超级管理员、运营角色等不同角色调用导出 API，应符合权限矩阵。
- 绕过前端按钮，直接请求后端接口，确认后端仍强制校验权限。

### 隐私与字段测试

- 验证导出字段是否仅包含产品确认允许的字段。
- 如设计要求脱敏，验证手机号、邮箱等字段脱敏规则是否正确。
- 验证最近订单金额是否只对有权限角色可见。

### 审计日志测试

- 成功导出时应记录审计日志。
- 失败导出、权限拒绝、超出数量限制等场景也应有必要的安全事件记录。
- 审计日志不应完整记录手机号、邮箱等敏感数据。
- 验证审计日志包含操作者、时间、导出条件、数量、结果等必要信息。

### 数量限制与性能测试

- 超过单次导出上限时应失败或进入异步审批/任务流程。
- 大数据量导出不应导致接口超时、内存异常或影响其他请求。
- 高频导出请求应被限流或告警。
- 验证分页、异步导出或任务队列场景下的失败恢复能力。

### 回归测试

- 管理员正常导出路径。
- 非管理员不可见按钮且无法调用接口。
- 权限变更后即时生效。
- 导出失败时前端不应显示成功状态。
