## PR 风险评审结论

该 PR 风险较高，不建议直接合并。当前实现涉及用户个人信息导出，但权限控制、隐私保护、审计和滥用防护均不完整，需要安全、后端和产品负责人继续确认后再推进。

---

## 高优先级问题

### 1. 权限控制风险：后端 API 未校验管理员角色

**严重程度：高**

当前仅前端按钮做管理员判断，后端 API 未检查角色。  
这意味着只要知道接口地址，非管理员用户可能直接调用导出 API，绕过前端限制。

**风险：**

- 非授权用户导出用户姓名、手机号、邮箱、订单金额等敏感信息。
- 前端权限判断可被绕过，不应作为安全边界。
- 可能造成严重数据泄露。

**建议处理方：后端负责人、安全负责人**

需要确认：

- 后端是否有统一鉴权/授权中间件。
- 导出接口是否必须强制校验管理员或特定权限。
- 是否需要更细粒度权限，例如 `user:export`、`pii:export`。
- 非授权访问时是否返回明确的拒绝响应并记录安全事件。

---

### 2. 隐私风险：导出内容包含敏感个人信息

**严重程度：高**

导出字段包括：

- 姓名
- 手机号
- 邮箱
- 最近订单金额

这些字段涉及个人身份信息和交易相关信息。当前变更摘要未说明是否有脱敏、最小化导出、用途限制或审批机制。

**风险：**

- 过度导出用户隐私数据。
- 数据被下载后难以追踪和撤回。
- 可能不符合内部隐私规范或合规要求。
- 最近订单金额可能属于敏感交易信息，应确认是否有导出必要。

**建议处理方：产品负责人、安全负责人**

需要确认：

- 导出这些字段是否有明确业务必要性。
- 是否可以减少导出字段，例如默认不导出手机号或订单金额。
- 是否需要手机号、邮箱脱敏。
- 是否需要导出前审批或二次确认。
- 是否需要在界面上提示导出包含敏感数据。
- 导出文件是否需要加密、限时下载或访问控制。

---

### 3. 缺少审计日志

**严重程度：高**

用户导出是高风险操作，应记录审计日志。当前没有审计日志。

**风险：**

- 无法追踪谁在什么时间导出了哪些数据。
- 出现数据泄露后无法溯源。
- 无法满足安全审计或合规追踪要求。
- 无法发现异常导出行为。

**建议处理方：后端负责人、安全负责人**

需要确认并补充审计内容：

- 操作人 ID / 角色
- 导出时间
- 导出条件
- 导出字段
- 导出数量
- 请求来源 IP / 设备信息，如系统已有能力
- 导出结果：成功 / 失败
- 文件标识或任务 ID，如采用异步导出

---

### 4. 没有导出数量限制

**严重程度：高**

当前没有导出数量限制，可能导致一次性导出大量用户数据。

**风险：**

- 数据泄露影响面扩大。
- 接口可能被滥用批量抓取数据。
- 大量导出可能造成数据库和服务性能压力。
- 可能导致超时、内存占用过高或服务不可用。

**建议处理方：后端负责人、产品负责人、安全负责人**

需要确认：

- 单次最大导出数量。
- 每日 / 每小时导出频率限制。
- 是否需要分页导出或异步导出。
- 是否需要导出任务队列。
- 是否需要对大批量导出增加审批。
- 是否需要按角色限制可导出范围。

---

## 中优先级问题

### 5. 前后端权限逻辑不一致，存在维护风险

**严重程度：中**

前端做了管理员判断，但后端没有对应授权逻辑，容易造成误解：开发或测试人员可能以为功能已经受保护。

**风险：**

- 后续维护时继续依赖前端控制。
- 权限模型不清晰。
- 测试覆盖容易遗漏直接调用 API 的场景。

**建议处理方：后端负责人、前端负责人**

需要确认：

- 前端按钮隐藏仅作为用户体验优化，不作为安全控制。
- 后端必须作为最终权限判断点。
- 权限规则需要在接口文档或权限配置中明确。

---

### 6. 缺少异常与失败处理信息

**严重程度：中**

摘要中未说明导出失败、权限失败、数据量过大、生成文件失败等场景如何处理。

**风险：**

- 用户体验不明确。
- 后端错误可能暴露内部信息。
- 重试机制不当可能导致重复导出或资源浪费。

**建议处理方：后端负责人、产品负责人**

需要确认：

- 无权限时的错误码和提示。
- 超出导出限制时的提示。
- 导出失败是否记录日志。
- 是否允许重试。
- 异步导出时任务状态如何展示。

---

## 测试风险与建议

当前摘要未体现相关测试，存在明显测试缺口。

### 必须补充的测试

#### 权限测试

需要覆盖：

- 非登录用户调用导出 API，应被拒绝。
- 普通用户直接调用导出 API，应被拒绝。
- 非管理员即使前端没有按钮，直接请求接口也应失败。
- 管理员或具备导出权限的用户可以成功导出。
- 角色变更后权限立即生效。

#### 隐私与字段测试

需要覆盖：

- 导出字段是否符合产品确认范围。
- 不应导出的敏感字段不会出现在文件中。
- 如要求脱敏，手机号、邮箱等字段必须按规则脱敏。
- 最近订单金额是否只导出被允许的范围。

#### 审计日志测试

需要覆盖：

- 成功导出会记录审计日志。
- 失败导出会记录审计日志。
- 无权限访问尝试会被记录，至少作为安全事件或访问拒绝日志。
- 审计日志包含操作人、时间、导出数量、导出条件等必要字段。

#### 导出数量与频率限制测试

需要覆盖：

- 超过单次导出上限时被拒绝或拆分处理。
- 高频调用导出接口时触发限流。
- 大数据量导出不会导致接口超时或服务异常。
- 并发导出不会造成资源耗尽。

#### 安全测试

需要覆盖：

- 直接请求后端 API 不能绕过权限。
- 修改请求参数不能导出超出权限范围的数据。
- 导出文件下载链接不能被未授权用户访问。
- 如果有文件临时存储，文件过期和访问控制有效。
