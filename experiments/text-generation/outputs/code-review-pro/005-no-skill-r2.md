## PR 风险 Review 结果

该 PR 风险较高，不建议直接合并上线。

### 1. 权限风险

- 当前仅在前端按钮判断管理员权限，后端 API 未做角色校验。
- 用户可以绕过前端，直接调用导出接口获取数据。
- 该问题属于严重权限控制缺失，可能导致非授权用户导出敏感用户数据。

**需要后端负责人处理：**

- 在后端导出 API 增加强制鉴权与角色校验。
- 明确只有具备指定权限的管理员角色才能调用。
- 增加越权访问测试用例。

**需要安全负责人确认：**

- 当前导出接口是否涉及敏感数据访问控制要求。
- 是否需要更细粒度权限，例如“用户数据导出权限”而非普通管理员权限。

---

### 2. 隐私与数据合规风险

- 导出字段包含姓名、手机号、邮箱，属于个人敏感/隐私相关信息。
- “最近订单金额”也可能属于用户交易相关数据。
- 未说明是否经过脱敏、最小化导出原则评估。
- 未说明导出用途、适用场景、数据保留或下载后的管控方式。

**需要产品负责人确认：**

- 这些字段是否都是业务必要字段。
- 是否可以减少导出字段，或按场景配置导出范围。
- 是否需要对手机号、邮箱等字段做脱敏处理。
- 是否需要用户范围筛选，避免一次性导出全量数据。

**需要安全负责人确认：**

- 是否满足隐私与数据保护要求。
- 是否需要增加数据导出审批或二次确认机制。

---

### 3. 审计风险

- 当前没有审计日志。
- 无法追踪谁在什么时间导出了哪些用户数据。
- 发生数据泄露或误操作时，无法进行追责和排查。

**需要后端负责人处理：**

- 增加导出审计日志，至少记录：
  - 操作人
  - 操作时间
  - 导出条件
  - 导出字段
  - 导出数量
  - 请求来源/IP
  - 导出结果状态

**需要安全负责人确认：**

- 审计日志字段是否满足安全审计要求。
- 日志保留周期与访问权限要求。

---

### 4. 导出数量与稳定性风险

- 当前没有导出数量限制。
- 可能导致：
  - 一次性导出大量个人数据，增加数据泄露影响面。
  - 数据库压力过大。
  - 接口超时或服务不可用。
  - 被恶意调用造成资源消耗。

**需要后端负责人处理：**

- 增加导出数量上限。
- 增加分页、异步任务或限流机制。
- 对导出接口增加频率限制。
- 对大批量导出设置后台任务和状态查询。

**需要产品负责人确认：**

- 单次导出的合理数量上限。
- 是否需要按筛选条件导出，而不是默认全量导出。
- 是否需要导出前展示风险提示或二次确认。

---

### 5. 测试风险

当前摘要未体现相关测试覆盖，存在以下测试缺口：

- 非管理员直接调用后端 API 是否会被拒绝。
- 普通管理员与具备导出权限的管理员是否权限区分清楚。
- 导出字段是否符合产品和隐私要求。
- 大数据量导出时是否稳定。
- 超过数量限制时是否正确拦截。
- 审计日志是否完整记录。
- 导出失败时是否记录异常并返回合理错误。
- 接口是否有重复请求、并发请求、频率限制相关测试。

**需要后端负责人补充：**

- 权限校验单元测试/集成测试。
- 导出数量限制测试。
- 审计日志测试。
- 异常和并发场景测试。

**需要安全负责人参与：**

- 越权访问测试。
- 敏感数据导出风险验证。
- 审计与合规检查。

---

## 需要继续处理的人

### 安全负责人

需要确认：

- 导出姓名、手机号、邮箱、最近订单金额是否符合隐私与安全要求。
- 是否需要字段脱敏、审批、二次确认或更细粒度权限。
- 审计日志字段与保留策略是否满足要求。
- 是否存在越权访问和数据泄露风险。

### 后端负责人

需要处理：

- 后端 API 必须增加角色/权限校验。
- 增加审计日志。
- 增加导出数量限制、限流或异步导出机制。
- 补充权限、审计、异常、大数据量和安全相关测试。

### 产品负责人

需要确认：

- 导出功能的业务场景和必要性。
- 导出字段是否都必须提供。
- 单次导出数量上限。
- 是否需要筛选条件、脱敏展示、二次确认或审批流程。

## 结论

该 PR 当前存在明显的权限、隐私、审计和稳定性风险。尤其是后端 API 未做角色校验，会导致前端权限判断被绕过，属于高风险问题。建议阻塞合并，待安全、后端和产品负责人确认并完成整改后再继续推进。
