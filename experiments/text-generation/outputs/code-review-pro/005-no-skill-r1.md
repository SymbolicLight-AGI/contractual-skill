## PR 风险评审结论

该 PR 风险较高，不建议直接合入。用户导出功能涉及个人信息与订单数据，目前在权限控制、隐私保护、审计追踪、导出范围控制和测试覆盖方面存在明显缺口。

## 主要风险

### 1. 权限风险

- 当前仅前端按钮判断管理员权限。
- 后端 API 未做角色校验，存在越权调用风险。
- 非管理员用户可能通过直接请求接口、抓包、脚本调用等方式导出用户数据。
- 权限控制不应依赖前端，必须在后端进行强制校验。

**风险等级：高**

### 2. 隐私与数据安全风险

导出字段包含：

- 姓名
- 手机号
- 邮箱
- 最近订单金额

这些属于敏感或隐私相关数据。当前没有说明：

- 是否有数据脱敏策略；
- 是否所有管理员都应该看到完整手机号、邮箱；
- 是否需要按角色区分可导出字段；
- 最近订单金额是否属于可导出的业务敏感信息；
- 导出文件是否有有效期、加密或访问控制。

**风险等级：高**

### 3. 审计风险

- 当前没有审计日志。
- 无法追踪是谁在什么时间导出了哪些用户数据。
- 出现数据泄露时无法定位责任人或排查影响范围。
- 对涉及个人信息的批量导出操作，应至少记录：
  - 操作人；
  - 操作时间；
  - 导出条件；
  - 导出数量；
  - 导出字段；
  - 请求来源；
  - 成功或失败状态。

**风险等级：高**

### 4. 导出数量与滥用风险

- 当前没有导出数量限制。
- 可能一次性导出全部用户数据。
- 存在数据批量泄露、接口滥用、性能冲击风险。
- 应考虑：
  - 单次导出上限；
  - 每日导出次数限制；
  - 分页或异步任务；
  - 大批量导出审批；
  - 频率限制。

**风险等级：高**

### 5. 测试风险

目前摘要中未体现相关测试，至少需要补充：

- 后端权限校验测试；
- 非管理员访问导出 API 的拒绝测试；
- 管理员正常导出测试；
- 导出字段正确性测试；
- 导出数量限制测试；
- 审计日志生成测试；
- 敏感字段脱敏或权限差异测试；
- 大数据量导出性能或超时测试。

**风险等级：中到高**

## 需要继续处理的人和事项

### 1. 安全负责人

需要确认：

- 该导出功能是否符合公司数据安全要求；
- 导出字段是否涉及敏感信息或需要脱敏；
- 是否需要导出审批机制；
- 是否需要下载链接过期、文件加密、访问控制；
- 审计日志应记录哪些字段；
- 是否需要频率限制和异常导出告警。

### 2. 后端负责人

需要处理：

- 在后端 API 增加管理员或更细粒度角色校验；
- 禁止仅依赖前端权限判断；
- 增加导出数量限制；
- 增加导出频率限制或防滥用机制；
- 增加审计日志；
- 补充权限、审计、导出限制相关测试；
- 明确接口在无权限、超出限制、导出失败时的错误处理。

### 3. 产品负责人

需要确认：

- 哪些角色可以使用用户导出功能；
- 是否所有字段都必须导出；
- 手机号、邮箱、订单金额是否需要脱敏；
- 单次导出最大数量；
- 是否需要导出原因、审批流程；
- 是否允许导出全部用户；
- 导出功能的业务使用场景和合规边界。

## 建议结论

当前 PR 不建议合入。

建议阻塞该 PR，待安全、后端、产品负责人分别确认并完成以下最低要求后再进入合并流程：

1. 后端增加权限校验；
2. 增加审计日志；
3. 增加导出数量限制；
4. 明确敏感字段处理策略；
5. 补充相关测试；
6. 由产品确认导出范围和角色权限。
