## 主要发现与严重级别

### 1. 高危：后端导出 API 未做角色/权限校验
**问题**：变更摘要说明“只有前端按钮做了管理员判断，后端 API 未检查角色”。这意味着只要用户能直接调用 API，就可能绕过前端限制导出用户数据。

**影响**：
- 非管理员可能导出姓名、手机号、邮箱、最近订单金额等敏感/个人信息。
- 存在越权访问和隐私泄露风险。
- 前端权限判断不能作为安全边界。

**建议**：
- 后端必须基于服务端身份与角色进行授权校验。
- 明确哪些角色允许导出、允许导出哪些字段、是否需要额外审批。
- 未授权请求应返回明确的拒绝响应，而不是依赖前端隐藏按钮。

**负责人**：后端负责人、安全负责人。

---

### 2. 高危：导出内容包含个人敏感信息，隐私风险未充分控制
**问题**：导出字段包括姓名、手机号、邮箱、最近订单金额。这些属于个人信息/敏感业务信息组合，泄露后风险较高。

**影响**：
- 可能造成用户隐私泄露。
- 最近订单金额可能暴露消费能力或业务敏感数据。
- 如果导出范围过大，单次泄露影响面会显著扩大。

**建议**：
- 产品和安全负责人需要确认导出字段是否必要，是否符合最小化原则。
- 考虑对手机号、邮箱做脱敏，或按角色控制字段粒度。
- 明确导出用途、使用范围、数据保留和下载有效期。
- 如业务允许，增加二次确认、审批或水印追踪。

**负责人**：产品负责人、安全负责人。

---

### 3. 高危：没有导出数量限制，存在大规模数据泄露与性能风险
**问题**：没有导出数量限制，可能允许一次性导出大量甚至全部用户数据。

**影响**：
- 一旦权限被绕过或账号被盗，可导致大规模数据泄露。
- 大数据量导出可能拖垮数据库、接口服务或文件生成服务。
- 缺少速率限制也可能被滥用为批量爬取接口。

**建议**：
- 增加导出条数上限、分页/异步任务限制、时间范围限制。
- 增加用户级、角色级、IP/租户级频率限制。
- 对大批量导出使用异步任务队列，避免阻塞主业务接口。
- 对超限操作要求审批或更高级权限。

**负责人**：后端负责人、安全负责人、产品负责人。

---

### 4. 中高危：没有审计日志，无法追踪数据导出行为
**问题**：导出操作没有审计日志。

**影响**：
- 发生数据泄露后无法确认是谁、何时、导出了哪些数据。
- 无法满足内部风控、合规或安全调查需要。
- 难以及时发现异常导出行为。

**建议**：
- 记录导出操作审计日志，包括操作者 ID、角色、时间、筛选条件、导出字段、导出数量、请求来源、任务 ID、结果状态等。
- 避免在日志中直接记录完整手机号、邮箱等敏感数据。
- 对异常导出量、异常频率、非工作时间导出等建立告警。

**负责人**：后端负责人、安全负责人。

---

### 5. 中危：缺少权限、隐私和导出边界相关测试
**问题**：摘要未提到相关测试，且当前设计存在明显风险点。

**建议补充测试**：
- 非管理员直接调用后端导出 API，应被拒绝。
- 普通用户即使知道接口路径和参数，也不能导出数据。
- 管理员可导出，但只能导出被授权字段。
- 导出数量超过限制时应失败或进入审批流程。
- 审计日志应在成功、失败、被拒绝等场景下正确记录。
- 导出筛选条件、分页/数量限制、速率限制应有覆盖。
- 验证日志中不包含完整手机号、邮箱等敏感信息。
- 大数据量导出场景下验证不会造成接口超时或数据库压力异常。

**负责人**：测试负责人、后端负责人、安全负责人。

---

## 总体判断

当前 PR 不建议直接上线。  
该导出功能涉及用户个人信息和业务敏感数据，但目前仅依赖前端管理员判断，后端没有权限校验，也没有审计日志和导出数量限制，存在明显的越权访问、隐私泄露、大规模数据导出和事后不可追踪风险。

上线前至少需要完成：
1. 后端强制权限校验；
2. 导出字段和范围的产品/安全确认；
3. 导出数量与频率限制；
4. 审计日志与必要告警；
5. 权限、隐私、审计、边界和异常场景测试。

## 需要交给谁继续处理

### 安全负责人
需要确认：
- 导出用户个人信息的权限模型是否合理；
- 是否需要审批、脱敏、水印、告警；
- 审计日志字段是否满足安全追踪要求；
- 是否存在合规或隐私要求。

### 后端负责人
需要处理：
- 在后端 API 增加强制角色/权限校验；
- 增加导出数量限制、频率限制和异常处理；
- 增加审计日志；
- 避免日志或错误信息泄露敏感数据；
- 补齐服务端测试。

### 产品负责人
需要确认：
- 哪些角色确实需要导出功能；
- 是否必须导出姓名、手机号、邮箱、最近订单金额这些字段；
- 是否需要脱敏展示或分级授权；
- 单次导出的合理数量、时间范围和业务流程；
- 是否需要审批流程或导出记录可见性。

### 测试负责人
需要补充验证：
- 权限绕过场景；
- 非管理员直接调用 API 的拒绝逻辑；
- 管理员正常导出；
- 字段脱敏/字段权限；
- 数量限制、频率限制；
- 审计日志完整性；
- 大批量导出的性能和失败恢复场景。
