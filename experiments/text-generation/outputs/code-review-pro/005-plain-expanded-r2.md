## Review 结论：不建议合并，存在高风险安全与隐私问题

### 主要发现与严重程度

#### 1. **严重：后端 API 未做角色/权限校验**
- 当前仅前端按钮判断管理员身份，后端 API 没有检查角色。
- 这意味着非管理员用户可能通过直接调用 API、抓包重放、脚本请求等方式导出用户数据。
- 前端权限控制不能作为安全边界。

**影响：**
- 未授权访问用户个人信息。
- 可能造成大规模数据泄露。
- 存在合规风险，尤其涉及手机号、邮箱等个人敏感信息。

**建议：**
- 后端必须增加管理员权限校验。
- 权限判断应基于服务端可信身份与角色信息。
- 非授权用户应返回明确的 403/401。
- 需要补充权限绕过测试。

**负责人：后端负责人、安全负责人。**

---

#### 2. **严重：导出内容包含个人敏感信息，隐私风险高**
- 导出字段包括姓名、手机号、邮箱、最近订单金额。
- 这些字段属于个人信息或敏感业务数据。
- 当前摘要未说明是否有字段最小化、脱敏、用途说明、访问审批或数据保留策略。

**影响：**
- 数据泄露后可直接用于骚扰、钓鱼、诈骗或用户画像。
- 最近订单金额可能暴露用户消费能力或业务敏感信息。
- 可能违反内部隐私规范或外部合规要求。

**建议：**
- 产品与安全确认导出字段是否必要。
- 评估是否需要脱敏，例如手机号、邮箱部分隐藏。
- 明确导出用途、可见范围、下载有效期、文件保存策略。
- 若业务不强依赖完整手机号/邮箱，不应默认导出完整值。

**负责人：产品负责人、安全负责人。**

---

#### 3. **高：缺少审计日志**
- 导出用户数据是高风险操作，但当前没有审计日志。
- 无法追踪是谁、何时、导出了哪些范围的数据。

**影响：**
- 发生数据泄露时无法追责或调查。
- 无法满足安全审计、合规审计要求。
- 内部滥用无法被发现。

**建议：**
- 记录导出操作审计日志，至少包括：
  - 操作人 ID；
  - 操作时间；
  - 导出条件或数据范围；
  - 导出记录数量；
  - 请求来源/IP/客户端信息；
  - 操作结果成功或失败。
- 审计日志应避免记录完整敏感数据内容。
- 审计日志应具备查询与告警能力。

**负责人：后端负责人、安全负责人。**

---

#### 4. **高：没有导出数量限制，存在批量泄露和性能风险**
- 当前没有导出数量限制。
- 一旦接口被滥用，可能一次性导出大量用户数据。

**影响：**
- 数据泄露规模不可控。
- 可能造成数据库压力、接口超时、服务降级。
- 恶意或误操作会放大影响范围。

**建议：**
- 增加导出数量上限。
- 支持分页、异步任务、审批流或限流。
- 对高风险、大批量导出增加二次确认或审批。
- 增加服务端速率限制与异常导出告警。

**负责人：后端负责人、产品负责人、安全负责人。**

---

### 测试建议

需要补充以下测试后再考虑合并：

1. **权限测试**
   - 非登录用户调用导出 API 应失败。
   - 普通用户调用导出 API 应失败。
   - 管理员调用导出 API 才允许成功。
   - 绕过前端按钮直接请求 API 时仍应被后端拦截。

2. **隐私字段测试**
   - 确认导出字段符合产品和安全确认范围。
   - 如果有脱敏要求，测试手机号、邮箱、金额字段是否按规则处理。
   - 确认未额外导出未授权字段。

3. **审计测试**
   - 成功导出应产生审计日志。
   - 失败或未授权导出尝试也应产生安全审计记录。
   - 审计日志不应包含完整敏感数据。

4. **数量限制与滥用测试**
   - 超过导出上限时应失败或进入审批/异步流程。
   - 高频请求应被限流。
   - 大批量导出不应导致服务不可用。

5. **异常与边界测试**
   - 空结果导出。
   - 大结果集导出。
   - 导出任务失败时不应返回成功状态。
   - 文件生成、下载链接、过期机制如有实现也需测试。

### 不确定性

基于当前变更摘要，无法确认以下事项：

- 是否已有统一的网关层权限校验。
- 是否已有全局审计系统但未在摘要中提及。
- 导出文件是否落盘、是否有下载链接、链接是否可被猜测或复用。
- 是否已有数据脱敏、审批、限流或告警机制。
- 最近订单金额是否属于必须导出的业务字段。

这些点需要 PR 作者或相关负责人补充说明，但在未确认前不应默认安全。
