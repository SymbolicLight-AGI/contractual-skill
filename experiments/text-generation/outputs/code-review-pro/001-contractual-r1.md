## 风险与影响

1. **Risk：SQL 注入**
   攻击者可能通过 HTTP query 参数构造恶意 `customer_id`，影响查询逻辑，读取非授权数据，甚至在某些数据库配置下造成数据破坏或更严重后果。

2. **Risk：客户备注泄露**
   CRM 客户备注可能包含敏感业务信息或个人信息。若缺少权限校验或注入防护，可能导致跨客户、跨用户的数据泄露。

3. **Risk：数据最小化不足**
   `SELECT *` 可能返回调用方不需要的字段，例如内部标记、审计字段、敏感备注元数据等，增加隐私和合规风险。

4. **Risk：错误输入导致运行时异常或非预期查询**
   如果 `customer_id` 为空、非数字、包含特殊字符，可能导致 SQL 语法错误、异常返回，或被利用改变查询条件。

5. **Risk：缺少测试导致回归不可控**
   当前没有测试，后续即使修复参数化查询、输入校验或权限逻辑，也缺少自动化验证防止回归。
